# Phase 16 Design: rosetta-rml-gen v2

## Goal

Replace the legacy `MappingDecision`-driven RML Turtle generator with a pipeline that:

1. Reads approved SSSOM mappings + source LinkML schema + master LinkML schema.
2. Generates a **linkml-map `TransformationSpecification`** YAML as the canonical intermediate representation.
3. Compiles the TransformSpec to **YARRRML** via a new `YarrrmlCompiler` contributed to a fork of `linkml-map`.
4. Executes the YARRRML via `morph-kgc` against source data (JSON/CSV/XML) → N-Triples.
5. Frames the N-Triples as **JSON-LD** conforming to the master ontology, using a JSON-LD context generated by `linkml gen-jsonld-context`.

## Architectural Choice: Option B (all-in on linkml-map)

Accepted trade-off: short-term stability risk (linkml-map flags its data model as "not yet fully stable") in exchange for:

- Reusable IR: `mapping.transform.yaml` is usable by `linkml-tr` for other transforms (data migration, schema derivation, profiling).
- Free unit conversion via linkml-map's `unit_conversion` on `SlotDerivation`.
- Community contribution path: `YarrrmlCompiler` sits alongside linkml-map's forthcoming `r2rml` compiler.
- Clean separation: TransformSpec is data-format-agnostic; all JSONPath/XPath/CSV logic lives in the compiler.

**rosetta-cli is a proof-of-concept at this stage** — architectural alignment with the LinkML ecosystem beats long-term stability hedging.

## Plan split

### Plan 16-00 — SSSOM audit-log schema extension + ingest collision lint (rosetta-cli)

Prerequisite for 16-01. Adds composite-mapping support to SSSOM per the
[composite entity expression](https://github.com/mapping-commons/sssom/blob/master/examples/schema/composite-entities.sssom.tsv)
pattern documented by mapping-commons, and enforces unique schema prefixes at ingest time.

- Extend `SSSOMRow` (`rosetta/core/models.py`) with four optional fields:
  - `subject_type: str | None = None` (e.g., `"composed entity expression"`)
  - `object_type: str | None = None`
  - `mapping_group_id: str | None = None` (shared by rows composing one logical mapping)
  - `composition_expr: str | None = None` (Python/GREL expression used by linkml-map `SlotDerivation.expr`)
- Audit log grows from 9 to 13 columns; suggest TSV grows from 11 to 15 columns.
- `rosetta-suggest` writes empty strings for new columns (auto-suggestions are never composite).
- `rosetta-accredit._parse_sssom_row` reads new columns with `.get()` defaults (backward-compat with existing 9-column logs).
- README audit-log and suggest-TSV column tables updated.
- Unit: composite round-trip test verifying a 2-row `mapping_group_id` survives append + load.
- **Schema-as-registry enforcement:** `rosetta-ingest` refuses to write a LinkML output whose `default_prefix` or `id` collides with an existing `*.linkml.yaml` sibling. Supports the multi-schema audit-log filter in 16-01.

### Plan 16-01 — SSSOM → linkml-map TransformSpec (rosetta-cli)

- New module: `rosetta/core/transform_builder.py`
- Inputs: `--sssom`, `--source-schema`, `--master-schema`
- Output: `mapping.transform.yaml` validated by linkml-map's Pydantic models
- Algorithm:
  1. Merge CURIE map from `rosetta.toml` + SSSOM header + both schemas (schemas take precedence).
  2. Filter SSSOM rows to `mapping_justification` in `{semapv:ManualMappingCuration, semapv:HumanCuration}` (unless `--include-manual`).
  3. **Grouping pass:** rows with `mapping_group_id` are collected first; each group becomes one composite SlotDerivation with `expr = row.composition_expr`. Non-grouped rows pass through as simple 1:1 derivations.
  4. Classify each row/group: CURIE resolves to class → ClassDerivation `populated_from`; CURIE resolves to slot → SlotDerivation `populated_from` (or `expr` for composites) under the owning class's ClassDerivation.
  5. Flag datatype mismatches: if `subject_datatype != object_datatype`, and both are quantity-kind, mark `unit_conversion: {target_unit: <object's unit>}`; if non-unit, emit a warning in the coverage report.
  6. Schema coverage check: every SSSOM subject/object resolves; every *required* master slot has a mapping (warn if not).
  7. Exit 1 on unresolvable CURIE or empty filtered set (unless `--force` / `--allow-empty`).
- New CLI: refactored `rosetta-rml-gen` with mode `--emit transform-spec` (default in 16-01).
- **No YARRRML output yet.**
- Dependencies to add: `linkml-map >= <fork-sha>` (pinned to fork git URL), `curies >= 0.7`.

### Plan 16-02 — YarrrmlCompiler (forked linkml-map)

- Fork `linkml/linkml-map` to a personal GitHub repo (user creates).
- Add `src/linkml_map/compiler/yarrrml_compiler.py` extending `J2BasedCompiler`.
- Add `src/linkml_map/compiler/templates/yarrrml.j2`.
- Compile rules:
  - `class_derivations[X]` → `mappings: {X: {sources: […], subjects: …, predicateobjects: [...]}}`
  - `slot_derivations[s] with populated_from` → one `po:` entry with reference from source schema's slot-level JSONPath/XPath/column annotation
  - `unit_conversion` → FnML GREL inline (linear) or UDF stub (nonlinear, deferred)
  - Data-format selection via TransformSpec annotation `rosetta:source_format` OR compiler flag
- Register CLI: `linkml-tr compile yarrrml`
- Tests in fork's `tests/` using linkml-map's existing pytest harness.
- rosetta-cli `pyproject.toml` pins fork git SHA.
- **PR to upstream deferred** — user will manually test + file PR after 16-03.

### Plan 16-03 — morph-kgc runner + JSON-LD framing + E2E (rosetta-cli)

- New module: `rosetta/core/rml_runner.py` — wraps `morph_kgc.materialize()` → `rdflib.Graph`.
- JSON-LD: generate context from master LinkML via `linkml gen-jsonld-context`; frame the graph via `graph.serialize(format="json-ld", context=ctx)`.
- `rosetta-rml-gen --run` — one-shot mode: SSSOM → TransformSpec → YARRRML (via compiler API) → N-Triples → JSON-LD.
- README section rewrite.
- E2E test: NOR CSV fixture → SSSOM → JSON-LD validated against master ontology via pySHACL.

## Master ontology ingestion

Master ontology stays as `.ttl`/`.owl` (hand-authored or imported from NATO sources). `rosetta-ingest` converts it to LinkML YAML via the existing `rdfs` format dispatch (verified in `rosetta/core/normalize.py:58`). This LinkML form is then passed to `rosetta-rml-gen --master-schema`.

## Locked decisions

| # | Decision |
|---|----------|
| 1 | Class-level SSSOM rows auto-populate `populated_from` on ClassDerivation. No class-level row for a source class referenced by slot-level rows → exit 1 with "add class mapping for src:X". |
| 2 | Multi-class schemas: one TransformSpec file with multiple `class_derivations`. |
| 3 | Unit conversion: marked on TransformSpec in 16-01 (`unit_conversion: {target_unit: …}`); compiled to FnML GREL in 16-02 (linear) with UDF escape hatch (deferred). |
| 4 | JSON-LD output: context-only (compaction) in 16-03. `--frame` flag is post-Phase-16 work. |
| 5 | Master ontology starts as `.ttl`/`.owl`; LinkML form produced by `rosetta-ingest`. No new ingest code. |
| 6 | `--sssom` points at the full audit log; builder filters for `semapv:HumanCuration`. `--include-manual` bypasses filter. |
| 7 | Empty filtered SSSOM → exit 1 unless `--allow-empty`. |
| 8 | Fork hosted as a separate GitHub repo under user's account; installed via git URL in `pyproject.toml`. |
| 9 | No upstream PR during Phase 16. User tests manually, files PR later. |
| 10 | Fork base: latest `main`; pin SHA in `pyproject.toml`; rebase on demand. |
| 11 | YarrrmlCompiler PR scope: compiler + tests only. SSSOM importer contribution deferred. |
| GA1 | Mapping-generating predicates: `skos:exactMatch` + `skos:closeMatch`. Other predicates → coverage-report entry, excluded from TransformSpec. `owl:differentFrom` always excluded (derank marker). |
| GA2 | 1:N and N:1 mappings handled via SSSOM composite-entity pattern: new columns `mapping_group_id`, `composition_expr`, `subject_type`, `object_type`. Rows sharing a `mapping_group_id` compose one logical mapping. `composition_expr` flows directly into linkml-map `SlotDerivation.expr`. Added in Plan 16-00 as prerequisite. |
| GA3 | Datatype mismatch without unit context → emit `slot_derivation.range: <target xsd type>` (linkml native coercion). Lossy coercion → coverage-report warning, not hard error. `rosetta-lint` remains authoritative on suspicious cases. |
| GA4 | Source-format resolution: hybrid. Infer from source schema's `source_format` annotation (added by ingest); CLI `--source-format` overrides; exit 1 if neither. |
| GA5 | Fork CI: linkml-map's own GitHub Actions run YarrrmlCompiler tests. rosetta-cli CI installs the fork via git URL and runs integration tests only. Fork README links back to this design doc. |

## CLI surface (target — may evolve)

```
rosetta-rml-gen \
  --sssom store/audit-log.sssom.tsv \
  --source-schema schemas/nor.linkml.yaml \
  --master-schema schemas/master.linkml.yaml \
  --emit [transform-spec|yarrrml|both]        # 16-01: transform-spec only; 16-02 adds yarrrml; 16-03 adds --run
  --source-format [json|csv|xml]              # required for --emit yarrrml; infers from ingest metadata if possible
  --coverage-report store/coverage.json       # optional
  --output store/mapping.transform.yaml       # or stdout
  --run                                       # 16-03 only: execute via morph-kgc → JSON-LD
  --source-data data/nor_tracks.csv           # 16-03 only: required with --run
  --include-manual                            # include semapv:ManualMappingCuration rows
  --allow-empty                               # don't fail on empty filtered set
  --force                                     # don't fail on unresolvable CURIEs
```

Exit codes: 0 = success; 1 = errors or unresolvable mappings (unless `--force`).

## Out of scope (deferred)

- JSON-LD framing with `@frame` (post-phase flag).
- Nonlinear unit conversions via Python UDF (GREL only in 16-02).
- Class-inheritance mappings (leaf class only for 16-01; SchemaView materialises inherited slots automatically, so simple cases work).
- Decomposition (1 source slot → N master slots) and aggregation (N → 1): manual `expr` override by user editing TransformSpec. Auto-generation deferred.
- Upstream PR to `linkml/linkml-map` — user-managed post-phase.

## Risk register

| Risk | Mitigation |
|------|-----------|
| linkml-map data model changes mid-phase | Pin to specific SHA; rebase only when deliberate; integration tests catch drift |
| Fork diverges from upstream | Keep YarrrmlCompiler small (one file + one template); document rebase steps in fork README |
| morph-kgc YARRRML support has edge-case gaps | Parallel run through `yatter` for diffing in 16-03 test suite |
| linkml `gen-jsonld-context` bugs | Test with master ontology early; hand-author context as fallback |
| Fork CI/test separation friction | Fork has its own GitHub Actions; rosetta-cli CI runs integration tests only |
