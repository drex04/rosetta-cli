# Plan 14-01: Audit-Log Accreditation Pipeline

## Goal

Replace the `ledger.json`-based accreditation state machine with a SSSOM append-only audit log. All accreditation decisions (analyst proposals and accreditor approvals/rejections) are persisted in `audit-log.sssom.tsv`, which becomes the single source of truth for boost/derank in `rosetta-suggest` and conflict-checking in `rosetta-lint`.

## Tasks

### Task 1 — `SSSOMRow` model extension + `core/accredit.py` rewrite

**Update `rosetta/core/models.py`:**

Add two fields to `SSSOMRow`:

```python
class SSSOMRow(BaseModel):
    subject_id: str
    predicate_id: str
    object_id: str
    mapping_justification: str
    confidence: float
    subject_label: str = ""
    object_label: str = ""
    mapping_date: datetime | None = None   # stamped by accredit ingest
    record_id: str | None = None           # UUID4 stamped by accredit ingest
```

Import `datetime` from `datetime` (already imported in the models file via pydantic; add explicit `from datetime import datetime` if needed).

Remove `LedgerEntry` and `Ledger` classes entirely. `MappingDecision` is retained (used by `rosetta-rml-gen`).

**Rewrite `rosetta/core/accredit.py`:**

Replace all ledger logic with these functions:

```python
def load_log(path: Path) -> list[SSSOMRow]:
    """Read the audit log. Returns [] if file absent."""

def append_log(rows: list[SSSOMRow], path: Path) -> None:
    """
    Append rows to the log. Creates file + SSSOM header if absent.
    Stamps mapping_date (utcnow ISO 8601) and record_id (uuid4) on each row.
    """

def current_state_for_pair(
    log: list[SSSOMRow], subject_id: str, object_id: str
) -> SSSOMRow | None:
    """
    Return the most recent row for (subject_id, object_id), ordered by
    mapping_date (None dates sort first). Returns None if pair absent.
    """

def query_pending(log: list[SSSOMRow]) -> list[SSSOMRow]:
    """
    Return ManualMappingCuration rows that have no subsequent HumanCuration
    for the same (subject_id, object_id) pair.
    """

def check_ingest_row(row: SSSOMRow, log: list[SSSOMRow]) -> None:
    """
    Raise ValueError with a descriptive message if:
    - row.mapping_justification ends with 'ManualMappingCuration' AND
      pair has any HumanCuration row in log
    - row.mapping_justification ends with 'HumanCuration' AND
      pair has no ManualMappingCuration row in log
    """
```

SSSOM justification constants to match against (use `endswith` to be namespace-agnostic):
- ManualMappingCuration: `"semapv:ManualMappingCuration"` or ends with `"ManualMappingCuration"`
- HumanCuration: `"semapv:HumanCuration"` or ends with `"HumanCuration"`

**Log file format:** Standard SSSOM TSV with comment-block header, same format as `_parse_sssom_tsv` already handles. Append new rows after existing content. The header block (lines starting with `#`) is written once on file creation. When appending, seek to end of file and write rows only.

**Files modified:** `rosetta/core/models.py`, `rosetta/core/accredit.py`

**Done when:**
- `SSSOMRow` round-trips with `mapping_date` and `record_id` fields; existing SSSOM TSV files without these columns still parse without error (fields default to `None`)
- `append_log` creates a valid SSSOM TSV file with header on first call; subsequent calls append rows
- `check_ingest_row` raises `ValueError` for both blocked transitions
- `query_pending` returns only rows with unmatched ManualMappingCuration
- `basedpyright` passes; no imports of `Ledger` or `LedgerEntry` remain

### Task 2 — `rosetta-accredit` CLI rewrite

**Rewrite `rosetta/cli/accredit.py`:**

Global group options: `--log PATH` (default from `[accredit].log` in rosetta.toml, fallback `store/audit-log.sssom.tsv`), `--config PATH`.

**`ingest` subcommand:**

```
rosetta-accredit ingest FILE
```

- Parse FILE as SSSOM TSV using the existing `_parse_sssom_tsv` logic (move or import from `cli/suggest.py` into `core/accredit.py` as `parse_sssom_tsv` — shared helper)
- Load the current log
- For each row in FILE with `mapping_justification` containing `ManualMappingCuration` or `HumanCuration`: call `check_ingest_row`; collect errors
- If any errors: print all errors to stderr, exit 1 (no partial writes)
- If clean: call `append_log` with passing rows; print count to stderr; exit 0
- Rows with other justifications (e.g., `CompositeMatching`): skip silently

**`review` subcommand:**

```
rosetta-accredit review [-o OUTPUT]
```

- Load log; call `query_pending`
- Write pending rows as SSSOM TSV to OUTPUT (default stdout)
- If no pending rows: write empty SSSOM TSV (header only); exit 0

**`status` subcommand:**

```
rosetta-accredit status [--source TEXT] [--target TEXT]
```

- Load log
- For each unique (subject_id, object_id) pair, call `current_state_for_pair`
- Optional `--source` / `--target` substring filters
- Output JSON array to stdout:
  ```json
  [{"subject_id": "...", "object_id": "...", "state": "ManualMappingCuration|HumanCuration", "predicate_id": "...", "mapping_date": "..."}]
  ```
- Exit 0

**`dump` subcommand:**

```
rosetta-accredit dump [-o OUTPUT]
```

- Load log
- For each (subject_id, object_id) pair, take the latest HumanCuration row (via `current_state_for_pair`, then filter for HumanCuration justification)
- Write as SSSOM TSV to OUTPUT (default stdout)
- These rows are suitable for use as boost/derank input (same format as the old `--approved-mappings`)

**Move `_parse_sssom_tsv`:**

Extract `_parse_sssom_tsv` from `rosetta/cli/suggest.py` into `rosetta/core/accredit.py` as `parse_sssom_tsv` (public). Update `suggest.py` to import from there. This avoids duplicating parsing logic.

**Files modified:** `rosetta/cli/accredit.py`, `rosetta/core/accredit.py`, `rosetta/cli/suggest.py` (import change only)

**Done when:**
- `uv run rosetta-accredit ingest` writes ManualMappingCuration and HumanCuration rows to log
- `uv run rosetta-accredit ingest` exits 1 with error messages for all blocked transitions; no partial writes
- `uv run rosetta-accredit review` outputs only pending ManualMappingCuration rows
- `uv run rosetta-accredit dump` outputs latest HumanCuration per pair
- `uv run rosetta-accredit status` prints JSON array
- `basedpyright` passes

```python
    result = apply_sssom_feedback(result, hc_rows)
```

**Existing-pair merge:**

After `rank_suggestions` produces `result` (list of candidate dicts per source URI), and *before* writing TSV output, merge log rows for already-tracked pairs:

```python
# Build index: (subject_id, object_id) -> latest non-CompositeMatching log row
log_index: dict[tuple[str, str], SSSOMRow] = {}
for row in log:
    if not (row.mapping_justification or "").endswith("CompositeMatching"):
        key = (row.subject_id, row.object_id)
        existing = log_index.get(key)
        if existing is None or (row.mapping_date or datetime.min) >= (existing.mapping_date or datetime.min):
            log_index[key] = row

# For each candidate in result: if pair is in log_index, replace justification/predicate/record_id
# but keep the freshly computed confidence score
for src_uri, candidates in result.items():
    for cand in candidates:
        key = (src_uri, cand["object_id"])
        if key in log_index:
            log_row = log_index[key]
            cand["mapping_justification"] = log_row.mapping_justification
            cand["predicate_id"] = log_row.predicate_id
```

### Task 4 — `rosetta-lint --sssom` mode

**Update `rosetta/cli/lint.py`:**

Add a `--sssom` option to the existing `lint` CLI. When `--sssom FILE` is provided, run the SSSOM validation mode instead of (or in addition to) the existing schema lint mode. The existing `--source`/`--master`/`--suggestions` path remains unchanged — `--sssom` is an independent mode.

```
rosetta-lint --sssom FILE [--config PATH]
```

Lint checks (in order):

1. **MaxOneMmcPerPair** — scan all rows with `ManualMappingCuration` justification; if any (subject_id, object_id) pair appears more than once, report an error per duplicate:
   ```
   ERROR [MaxOneMmcPerPair] nor:speed → mst:velocity: 2 ManualMappingCuration rows (max 1)
   ```

2. **NoHumanCurationReproposal** — load log from config (if configured and present); for each `ManualMappingCuration` row in the file, check if pair has any `HumanCuration` in log; report error per violation:
   ```
   ERROR [NoHumanCurationReproposal] nor:speed → mst:velocity: pair already has a HumanCuration decision in the audit log
   ```

3. **ValidPredicate** — for each `ManualMappingCuration` row, validate `predicate_id` is one of:
   `skos:exactMatch`, `skos:closeMatch`, `skos:narrowMatch`, `skos:broadMatch`, `skos:relatedMatch`, `owl:differentFrom`
   Report error per invalid predicate.

All errors printed to stderr, one per line. Exit 1 if any errors found, 0 if clean.

**Add to `core/lint.py`** (or inline in `cli/lint.py` if lint has no separate core module — check first):

```python
def check_sssom_proposals(
    rows: list[SSSOMRow],
    log: list[SSSOMRow],
) -> list[str]:
    """Return list of error strings. Empty list = no errors."""
```

**Files modified:** `rosetta/cli/lint.py`, `rosetta/core/lint.py` (if exists) or inline

**Done when:**
- `rosetta-lint --sssom candidates.sssom.tsv` exits 0 on clean file, 1 with errors on stderr for each violation
- MaxOneMmcPerPair catches duplicate proposals
- NoHumanCurationReproposal catches re-proposals when log is configured
- ValidPredicate catches invalid predicate values
- Existing lint behaviour (`--source/--master/--suggestions`) is unaffected
- `basedpyright` passes
