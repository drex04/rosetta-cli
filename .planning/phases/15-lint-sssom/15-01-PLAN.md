# Plan 15-01: rosetta-lint SSSOM enrichment

## Goal

Remove the legacy RDF lint path (--source/--master/--suggestions). Enrich the SSSOM lint
path with unit and datatype compatibility checks, and make it emit a JSON LintReport.

Datatype flows: LinkML slot.range → embed.py → EmbeddingVectors.datatype → suggest.py →
SSSOMRow.subject_datatype / object_datatype → SSSOM TSV → lint.py.

## Context

- All 14 v2 phases are complete. This is Phase 15 post-milestone.
- `detect_unit(name, description)` in `unit_detect.py` already parses unit strings from
  field names. `UNIT_STRING_TO_IRI` and `units_compatible()` in `units.py` handle QUDT
  comparison. The SSSOM path just needs to call these directly.
- `parse_sssom_tsv` uses `csv.DictReader` + `.get()` defaults — adding optional columns
  is backward-compatible with existing files that lack them.
- The `check_sssom_proposals()` function currently returns `list[str]`; it will be
  refactored to return `list[LintFinding]` so everything merges into one LintReport.

#### 4b — Refactor `check_sssom_proposals()` → returns `list[LintFinding]`

Change signature:
```python
def check_sssom_proposals(
    rows: list[SSSOMRow],
    log: list[SSSOMRow],
) -> list[LintFinding]:
```

For each error condition, append a `LintFinding` instead of a string:
- `max_one_mmc_per_pair` → severity `"BLOCK"`, source_uri=subject_id, target_uri=object_id
- `reproposal_of_approved` → severity `"BLOCK"`
- `reproposal_of_rejected` → severity `"BLOCK"`
- `invalid_predicate` → severity `"BLOCK"`

#### 4c — Add `_unit_label()` helper
```python
def _unit_label(row_id: str, label: str) -> str:
    """Return the best available field name for unit detection."""
    if label:
        return label
    # strip schema/prefix → local name
    local = row_id.rsplit("/", 1)[-1].rsplit(":", 1)[-1]
    return local
```

#### 4d — Add `_check_units()` helper
```python
def _check_units(
    findings: list[LintFinding],
    row: SSSOMRow,
    qudt_graph: rdflib.Graph,
) -> None:
```

Logic (mirrors old RDF path):
1. `detect_unit(_unit_label(row.subject_id, row.subject_label), "")` → `src_unit_str`
2. `detect_unit(_unit_label(row.object_id, row.object_label), "")` → `tgt_unit_str`
3. If either is `None` → append `unit_not_detected` INFO finding; return.
4. Look up both in `UNIT_STRING_TO_IRI`. If either maps to `None` → append
   `unit_not_detected` INFO ("no QUDT IRI mapping"); return.
5. Call `units_compatible(src_iri, tgt_iri, qudt_graph)`:
   - `False` → `unit_dimension_mismatch` BLOCK
   - `True` + different expanded IRIs → `unit_conversion_required` WARNING + `fnml_suggestion`
   - `True` + same IRI → no finding
   - `None` → `unit_vector_missing` INFO

#### 4e — Add `_check_datatype()` helper
```python
_NUMERIC_LINKML = {
    "integer", "int", "float", "double", "decimal", "long", "short",
}

def _check_datatype(findings: list[LintFinding], row: SSSOMRow) -> None:
```

Logic:
- If `row.subject_datatype` is None or `row.object_datatype` is None → return (skip).
- `src_numeric = row.subject_datatype.lower() in _NUMERIC_LINKML`
- `tgt_numeric = row.object_datatype.lower() in _NUMERIC_LINKML`
- If `src_numeric != tgt_numeric` → append `datatype_mismatch` WARNING.

#### 4f — Rewrite `cli()` SSSOM path

New `cli()` body (SSSOM is the only mode):

```python
@click.command()
@click.option("--sssom", ...)
@click.option("--output", ...)
@click.option("--strict", ...)
@click.option("--config", ...)
def cli(sssom, output, strict, config):
    if sssom is None:
        click.echo("Error: --sssom is required.", err=True)
        sys.exit(1)

    rows = parse_sssom_tsv(Path(sssom))
    cfg = load_config(Path(config)) if config else load_config()
    log_path_str = get_config_value(cfg, "accredit", "log")
    log = load_log(Path(log_path_str)) if log_path_str and Path(log_path_str).exists() else []

    findings: list[LintFinding] = []

    # 1. Proposal checks (MaxOneMmcPerPair, NoReproposal, ValidPredicate)
    findings.extend(check_sssom_proposals(rows, log))

    # 2. Per-row unit + datatype checks
    qudt_graph = load_qudt_graph()
    for row in rows:
        _check_units(findings, row, qudt_graph)
        _check_datatype(findings, row)

    # 3. --strict: upgrade WARNINGs → BLOCKs
    if strict:
        findings = [
            f.model_copy(update={"severity": "BLOCK"}) if f.severity == "WARNING" else f
            for f in findings
        ]

    # 4. Build report and write
    summary = LintSummary(
        block=sum(1 for f in findings if f.severity == "BLOCK"),
        warning=sum(1 for f in findings if f.severity == "WARNING"),
        info=sum(1 for f in findings if f.severity == "INFO"),
    )
    report = LintReport(findings=findings, summary=summary)
    with open_output(output) as fh:
        fh.write(report.model_dump_json(indent=2))

    sys.exit(1 if any(f.severity == "BLOCK" for f in findings) else 0)
```

Also add `import rdflib` back only if needed by `_check_units` (it needs `rdflib.Graph`
for the `qudt_graph` type). Actually `load_qudt_graph` returns `rdflib.Graph` and
`units_compatible` accepts `rdflib.Graph` — keep `import rdflib` for the type annotation
on `_check_units`. Remove `XSD`, `Node` imports.

**Done:** `rosetta-lint --sssom` produces JSON LintReport. `--source/--master/--suggestions`
are gone. Unit and datatype findings appear in the report.
