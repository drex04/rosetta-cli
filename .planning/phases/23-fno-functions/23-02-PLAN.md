    5. Add `_bridge_unit_conversion` private method on the class:
       ```python
       @staticmethod
       def _bridge_unit_conversion(
           uc: UnitConversionConfiguration,
       ) -> FunctionCallConfiguration | None:
           """Convert legacy UnitConversionConfiguration to FunctionCallConfiguration.

           Temporary bridge — remove after rosetta-cli Plan 23-04 migrates
           build_slot_derivation() to emit function_call directly.
           """
           src = uc.source_unit or ""
           tgt = uc.target_unit or ""
           fun_id = _UNIT_PAIR_TO_FNO.get((src, tgt))
           if fun_id is None:
               return None
           return FunctionCallConfiguration(
               function_id=fun_id,
               parameter_predicate=_GREL_VALUE_PARAM,
           )
       ```

    6. Replace the unit_conversion branch in `_build_mapping_context` (lines 323-354)
       with a generic function_call branch:

       ```python
       # Resolve function_call: direct or via deprecation bridge
       fc = slot_deriv.function_call
       if fc is None and slot_deriv.unit_conversion is not None:
           fc = self._bridge_unit_conversion(slot_deriv.unit_conversion)

       if fc is not None:
           po = {
               "predicate": predicate,
               "reference": reference,
               "function": {
                   "name": fc.function_id,
                   "parameters": [
                       {
                           "name": fc.parameter_predicate,
                           "value": reference,
                       }
                   ],
               },
           }
           dt = fc.output_datatype or xsd_type
           if dt is not None:
               po["function"]["datatype"] = dt
       else:
           # Simple slot — no function
           po = {"predicate": predicate, "reference": reference}
           if xsd_type is not None:
               po["datatype"] = xsd_type
       ```

       Delete the old `if slot_deriv.unit_conversion is not None:` branch and the
       `try/except ValueError` block entirely. The new code is simpler — no exception
       handling needed since `_bridge_unit_conversion` returns None on unknown pairs.
