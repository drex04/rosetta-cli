# Plan 23-04: Transform Builder Migration + E2E

## Goal

Replace all hardcoded unit-conversion logic in `transform_builder.py` with the generic
FnO function mechanism. The transform builder reads `conversion_function` from the SSSOM
row (populated by suggest in 23-03), resolves function metadata via `FunctionLibrary`
(created in 23-01), and emits `FunctionCallConfiguration` (added to the fork model in
23-02). Delete all legacy dicts and inline UDF generation.

## Tasks

**Files:** `rosetta/core/transform_builder.py`

**Delete:**
- `_QUDT_TO_FORK_UNIT` dict (L35-38)

**Add:**
- `FunctionCallConfiguration` to the linkml_map import block
- `from rosetta.core.function_library import FunctionLibrary` import

**Rewrite `build_slot_derivation()`:**
```python
def build_slot_derivation(
    m: _SlotMapping, *, library: FunctionLibrary | None = None
) -> SlotDerivation:
    fn_call: FunctionCallConfiguration | None = None
    if m.row.conversion_function and library is not None:
        try:
            fn_iri = library.resolve_curie(m.row.conversion_function)
            param_curie = library.get_parameter_predicate(m.row.conversion_function)
            raw_out = library.get_output_type(m.row.conversion_function)
            fn_call = FunctionCallConfiguration(
                function_id=fn_iri,
                parameter_predicate=library.resolve_curie(param_curie),
                output_datatype=library.resolve_curie(raw_out) if raw_out else None,
            )
        except KeyError:
            raise ValueError(
                f"conversion_function {m.row.conversion_function!r} on row "
                f"{m.row.record_id!r} is not declared in the FunctionLibrary. "
                f"Check rosetta.toml [conversions] or the FnO Turtle declarations."
            )
    return SlotDerivation(
        name=m.target_slot_name,
        populated_from=m.source_slot_name,
        function_call=fn_call,
    )
```

**Thread `function_library` through the call chain:**
- `build_spec()` — add `function_library: FunctionLibrary | None = None` kwarg
- `_assemble_class_derivations()` — add `function_library` param, pass to `_collect_mappings()`
- `_collect_mappings()` — add `function_library` param, pass to `build_slot_derivation(sm, library=function_library)`

**Simplify `_build_slot_mapping()`:**
Remove the `detect_unit` calls and drop `source_unit=` / `target_unit=` kwargs from the
`_SlotMapping` constructor. The two description-reading lines (`src_desc`, `mst_desc`)
also become dead — remove them.

**[review] Update `_remap_to_mapped_classes()` (L466-473):**
This function reconstructs `_SlotMapping` instances with `source_unit=sm.source_unit,
target_unit=sm.target_unit`. After the field deletion, these kwargs must also be removed
or the reconstruction raises `TypeError`. Remove those two kwargs from the constructor call.

**Done:** `grep -c '_LINEAR_CONVERSION_PAIRS\|_QUDT_TO_FORK_UNIT\|UnitConversionConfiguration\|detect_unit' rosetta/core/transform_builder.py` returns 0.

**Rewrite `_write_udf_file()`:**
```python
def _write_udf_file(work_dir: Path) -> Path:
    udf_path = work_dir / "rosetta_udfs.py"
    try:
        source = files("rosetta.functions").joinpath("unit_conversion_udfs.py").read_text(encoding="utf-8")
        udf_path.write_text(source, encoding="utf-8")
    except OSError as exc:
        raise RuntimeError(f"Failed to write UDF file to {udf_path}: {exc}") from exc
    return udf_path
```

## Risks

1. **morph-kgc UDF namespace mismatch:** The old `_UDF_SOURCE` used `udf/` namespace; the
   new static file uses `rfns:` namespace. The fork compiler (since 23-02) emits `rfns:`
   IRIs, so this should match. The E2E test verifies end-to-end.

2. **FunctionLibrary.resolve_curie for unknown CURIEs:** If a CURIE prefix isn't in the
   library's graph, `resolve_curie` may raise. In practice, all builtins are registered
   via `load_builtins()`, and custom functions (23-05) will have their own Turtle loaded.
   The lint gate (23-03) already validates that conversion_function references exist.

3. **15-column SSSOM files without conversion_function:** These continue to work because
   `conversion_function` defaults to `None` on `SSSOMRow` and `build_slot_derivation`
   checks for it before creating `FunctionCallConfiguration`.
