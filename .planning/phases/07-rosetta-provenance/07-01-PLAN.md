## Context

Phase 7 implements `rosetta-provenance`, which stamps PROV-O metadata onto mapping artifacts produced by earlier phases. The stub `rosetta/cli/provenance.py` already exists with a `cli` entrypoint that exits with code 1. This plan replaces it with a working Click group with `stamp` and `query` subcommands.

**Key constraints from prior phases:**
- `_PROV` namespace (`http://www.w3.org/ns/prov#`) is already declared in `rdf_utils.py`
- `ROSE_NS` (`http://rosetta.interop/ns/`) is the namespace for `rose:version` and activity/agent URIs
- All user-facing JSON outputs use Pydantic v2 models (Phase 5 convention)
- `rosetta-provenance` entrypoint is already registered in `pyproject.toml`
- No new pip dependencies needed — rdflib 6.3+ covers all PROV-O needs

**Autonomous decisions** (see `.decisions-pending.md`):
- D7-01: In-place augmentation (no sidecar)
- D7-02: UUID activity URI + default agent `rose:agent/rosetta-cli`
- D7-03: `rose:version` integer literal, incremented per stamp
- D7-04: Subcommand group (`stamp` / `query`); `--format json` for machine output
- D7-05: Minimal PROV-O triple set (Activity + Entity + Agent)

### Task 2 — Implement rosetta/core/provenance.py

**File:** `rosetta/core/provenance.py` (new file)

Implement two public functions with full type annotations:

#### `stamp_artifact`

```python
def stamp_artifact(
    g: Graph,
    artifact_uri: str,
    agent_uri: str = "http://rosetta.interop/ns/agent/rosetta-cli",
    label: str | None = None,
    now: datetime | None = None,
) -> int:
```

Logic:
1. Determine `now` (use `datetime.now(timezone.utc)` if not provided — injecting `now` enables deterministic tests).
2. Use `query_graph(g, SPARQL, bindings={"artifact": URIRef(artifact_uri)})` — never f-string interpolation (SPARQL injection risk). Query: `SELECT (MAX(?v) AS ?max_v) WHERE { ?artifact rose:version ?v }`. The MAX aggregate returns one row; the result variable may be `None` on an empty graph — None-guard before `int()`: `max_v = int(rows[0]["max_v"]) if rows and rows[0]["max_v"] is not None else 0`. new_version = max_v + 1.
3. (new_version is set per step 2 above).
4. Remove any existing `<artifact_uri> rose:version ?any` triple via `g.remove((URIRef(artifact_uri), ROSE_NS.version, None))`.
5. Add new version triple: `g.add((URIRef(artifact_uri), ROSE_NS.version, Literal(new_version, datatype=XSD.integer)))`.
6. Generate activity URI: `ROSE_NS[f"activity/{uuid4()}"]`.
7. Add PROV-O triples (see D7-05 in `.decisions-pending.md`):
   - `activity a prov:Activity`
   - `activity prov:startedAtTime Literal(now.isoformat(), datatype=XSD.dateTime)`
   - `activity prov:endedAtTime Literal(now.isoformat(), datatype=XSD.dateTime)`
   - `activity prov:wasAssociatedWith URIRef(agent_uri)`
   - If label: `activity rdfs:label Literal(label)`
   - `URIRef(artifact_uri) a prov:Entity`
   - `URIRef(artifact_uri) prov:wasGeneratedBy activity`
   - `URIRef(agent_uri) a prov:Agent`
8. Bind namespaces: call `bind_namespaces(g)` and also `g.bind("xsd", XSD)` and `g.bind("rdfs", RDFS)`.
9. Return `new_version`.

#### `query_provenance`

```python
def query_provenance(
    g: Graph,
    artifact_uri: str,
) -> list[ProvenanceRecord]:
```

Logic:
1. SPARQL SELECT over the graph to fetch activity, agent, label, startedAtTime, endedAtTime.
2. For version: join on `<artifact_uri> rose:version ?v` (there is only one version triple — the current one). For historical per-activity versions, embed version into the activity URI is not feasible; instead, report the current `rose:version` on every record (acceptable for MVP).
3. Coerce None-guarded OPTIONAL vars (e.g., `?label`) to Python None using the rdflib SPARQL gotcha pattern: `str(row["label"]) if row["label"] is not None else None`.
4. Return list of `ProvenanceRecord` instances sorted by `started_at` ascending.

**SPARQL query skeleton (use `query_graph` with initBindings — never f-string):**
```sparql
PREFIX prov: <http://www.w3.org/ns/prov#>
PREFIX rose: <http://rosetta.interop/ns/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>
SELECT ?activity ?agent ?label ?started ?ended ?version WHERE {
    ?artifact prov:wasGeneratedBy ?activity ;
              rose:version ?version .
    ?activity prov:startedAtTime ?started ;
              prov:endedAtTime   ?ended ;
              prov:wasAssociatedWith ?agent .
    OPTIONAL { ?activity rdfs:label ?label }
}
ORDER BY ASC(?started)
```

Call as: `query_graph(g, SPARQL, bindings={"artifact": URIRef(artifact_uri)})`.

**Version semantics:** `rose:version` is a single-valued triple (only the current version is stored). When an artifact has been stamped N times, the SELECT returns N rows (one per Activity), but all rows carry the same `?version` value (the current version). This is intentional MVP behavior. Map each row to a `ProvenanceRecord` — the `version` field documents "artifact version at query time."

**Imports needed:**
```python
from __future__ import annotations
import uuid
from datetime import datetime, timezone
from rdflib import Graph, Literal, URIRef
from rdflib.namespace import RDFS, XSD
from rosetta.core.rdf_utils import bind_namespaces, query_graph, ROSE_NS, _PROV as PROV
from rosetta.core.models import ProvenanceRecord
```

Note: Import `_PROV` from `rdf_utils` (aliased as `PROV`) — do NOT re-import `rdflib.namespace.PROV` separately. `bind_namespaces` binds `_PROV` under the "prov" prefix; a second `Namespace` object at the same URI creates redundant bindings and confuses readers.

**Done when:** Unit tests in Task 4 pass; `basedpyright` reports zero errors on this file.

### Task 3 — Implement rosetta/cli/provenance.py (full CLI)

**File:** `rosetta/cli/provenance.py` (replace stub)

Structure: Click group `cli` with two subcommands `stamp` and `query`.

```python
@click.group()
@click.option("--config", "-c", default=None, help="Path to rosetta.toml.")
@click.pass_context
def cli(ctx: click.Context, config: str | None) -> None:
    """Record and query provenance metadata for mapping decisions."""
    ctx.ensure_object(dict)
    ctx.obj["config"] = config
```

#### `stamp` subcommand

```
rosetta-provenance stamp <INPUT> [--output PATH] [--agent URI] [--label TEXT] [--format text|json]
```

Options:
- `INPUT` — positional argument: path to input Turtle artifact (required)
- `--output / -o` — output path; if omitted, **overwrites INPUT in-place** (D7-01); pass `-` to write to stdout
- `--agent` — agent URI string; default `"http://rosetta.interop/ns/agent/rosetta-cli"`
- `--label / -l` — human-readable label for this stamp event
- `--format` — `type=click.Choice(["text", "json"])`, default `"text"`; controls stamp summary on stderr (Turtle always goes to the output path)

Logic:
1. Wrap everything in `try/except Exception as exc: click.echo(f"Error: {exc}", err=True); sys.exit(1)` — consistent with lint.py pattern.
2. Load graph from INPUT path via `load_graph(Path(input_path))`. (Raises `ValueError` on bad TTL or missing file — caught by outer try/except.)
3. Determine artifact URI: `str(ROSE_NS[Path(input_path).stem])`. **Document in `--help`:** "Artifact URI is derived as `rose:<stem>` from the input filename. Files with the same stem in different directories share an artifact URI."
4. Call `stamp_artifact(g, artifact_uri, agent_uri, label)` → returns `new_version`.
5. Determine output path: if `--output` is `-` use `sys.stdout`; if `--output` is given use `Path(output)`; otherwise overwrite INPUT in-place with `Path(input_path)`. Call `save_graph(g, output_dest)`. (Raises `OSError` on read-only path or missing directory — caught by outer try/except.)
6. Build stamp summary: construct a `ProvenanceRecord` instance (use `datetime.now(timezone.utc).isoformat()` for timestamps), call `.model_dump(mode="json")`. Print to stderr: if `--format json` print JSON; else print `Stamped <artifact_uri> version <N>`.
7. Exit 0 on success.

#### `query` subcommand

```
rosetta-provenance query <INPUT> [--format text|json]
```

Options:
- `INPUT` — positional argument: path to stamped Turtle artifact
- `--format` — `type=click.Choice(["text", "json"])`, default `"text"`

Logic:
1. Load graph from INPUT.
2. Determine artifact URI (same `ROSE_NS[stem]` heuristic).
3. Call `query_provenance(g, artifact_uri)` → list of `ProvenanceRecord`.
4. If `--format text`: print one line per record: `v{version}  {started_at}  {agent_uri}  {label or "(no label)"}`.
5. If `--format json`: `print(json.dumps([r.model_dump(mode="json") for r in records], indent=2))`.
6. Exit 0; if no records found, print `"No provenance records found."` to stderr and exit 0.

**Done when:** `uv run rosetta-provenance stamp tests/fixtures/nor_schema.ttl --label "test"` produces valid Turtle on stdout with PROV-O triples; `uv run rosetta-provenance query` prints stamp events.
