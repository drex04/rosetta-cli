# Plan 17-01 — QUDT-native multi-library unit detection

## Context

`detect_unit(name, description)` currently returns internal canonical strings like `"meter"` which
`lint.py` looks up in `UNIT_STRING_TO_IRI` to get QUDT IRIs. This two-step indirection requires
keeping two artefacts in sync and caused the key-contract gotcha in prior sessions.

**QUDT-native design:** `detect_unit()` returns QUDT IRI strings directly (e.g. `"unit:M"`).
`UNIT_STRING_TO_IRI` is deleted. Lint uses the IRI straight from detection. The pint→IRI mapping
table lives only inside `unit_detect.py` — one place of truth.

Units with no QUDT IRI (dBm, etc.) continue to return `None`.

## Cascade architecture

```
detect_unit(name, description) -> str | None:   # str = QUDT IRI e.g. "unit:M"
  1. Name regex (_NAME_PATTERNS)       → return QUDT IRI on first match
  2. Description regex (_DESC_PATTERNS) → return QUDT IRI on first match
  3. quantulum3.parser.parse(description) → for each extracted unit:
       a. pint UnitRegistry.parse_expression(unit.name)
       b. _PINT_TO_QUDT_IRI.get(str(parsed.units))
       c. Return first non-None mapping
  4. return None
```

Both quantulum3 and pint are **lazily imported** to avoid startup overhead for callers that only
use the regex path.

### Updated `detect_unit()` signature and body

```python
def detect_unit(name: str, description: str) -> str | None:
    """Return the QUDT unit IRI for a field, or None if not detected.

    Three-layer cascade:
      1. Name regex — high-precision token patterns on field name
      2. Description regex — keyword patterns on description text
      3. quantulum3 + pint — NLP extraction with validation on description
    Returns None both when no unit is detected and when the detected unit
    has no QUDT IRI (e.g. dBm).
    """
    for pattern, iri in _NAME_PATTERNS:
        if pattern.search(name):
            return iri

    for pattern, iri in _DESC_PATTERNS:
        if pattern.search(description):
            return iri

    return _detect_from_nlp(description)


def _detect_from_nlp(description: str) -> str | None:
    """quantulum3 + pint layer — lazy imports, called only when regex misses."""
    from pint import UnitRegistry          # noqa: PLC0415
    from quantulum3 import parser as q3   # noqa: PLC0415

    ureg = UnitRegistry()
    for qty in q3.parse(description):
        try:
            parsed = ureg.parse_expression(qty.unit.name)
            iri = _PINT_TO_QUDT_IRI.get(str(parsed.units))
            if iri is not None or str(parsed.units) in _PINT_TO_QUDT_IRI:
                return iri   # return even if IRI is None (e.g. dBm explicitly mapped)
        except Exception:  # noqa: BLE001
            continue
    return None
```

**Note on the None-vs-absent distinction:** `_PINT_TO_QUDT_IRI.get(key)` returns `None` for both
"key absent" and "key maps to None". The `str(parsed.units) in _PINT_TO_QUDT_IRI` guard ensures
an explicit `None` entry (e.g. dBm) is returned as `None` (no IRI) rather than falling through to
the next candidate.

**Done:** `detect_unit("freq_mhz", "")` returns `"unit:MegaHZ"`; `detect_unit("signal_dbm", "")` returns `None`; `detect_unit("", "carrier at 100 megahertz")` returns `"unit:MegaHZ"`.
