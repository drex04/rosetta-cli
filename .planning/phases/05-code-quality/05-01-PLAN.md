## Goal

Enforce static type checking and linting as a core part of the development workflow. When done, `uv run basedpyright` and `uv run ruff check .` pass cleanly on all source and test files, and CI gates every push/PR on both.

## Must-Haves

### Truths

1. `uv run basedpyright` exits 0 with strict mode on `rosetta/cli/` and `rosetta/core/`; basic mode on `rosetta/tests/`
2. `uv run ruff check .` exits 0 on the whole repo; `uv run ruff format --check .` exits 0
3. All functions in `rosetta/core/` and `rosetta/cli/` have explicit parameter and return type annotations
4. GitHub Actions CI runs ruff, basedpyright, and pytest on every push/PR to master; all three jobs must pass
5. `uv run pytest` still exits 0 — no regressions from annotation changes
6. `CLAUDE.md` contains a `## Code Quality` section with mandatory check commands, annotation rules, and Pydantic conventions

## Tasks

### Task 1: Add basedpyright + configure pyproject.toml ruff section

**What:** Add `basedpyright` to dev dependencies. Add `[tool.ruff]` and `[tool.ruff.lint]` sections to `pyproject.toml`.

**Files modified:**
- `pyproject.toml`

**Steps:**
1. `uv add --dev basedpyright`
2. Add to `pyproject.toml`:
   ```toml
   [tool.ruff]
   line-length = 100
   target-version = "py311"

   [tool.ruff.lint]
   select = ["E", "W", "F", "I", "UP"]
   ```
3. Run `uv run ruff check .` — fix any violations surfaced (expected: import ordering, minor style issues)
4. Run `uv run ruff format .` — apply formatting

**Done when:** `uv run ruff check .` and `uv run ruff format --check .` both exit 0.

**What:** Add full type annotations to all 9 core modules + 3 parser modules.

**Files modified:**

- `rosetta/core/io.py`

- `rosetta/core/parsers/json_schema_parser.py`
- `rosetta/core/parsers/openapi_parser.py`

**Annotation rules:**
- Use broad rdflib types: `rdflib.term.Node | None`, `rdflib.Graph`
- Use `list[dict[str, Any]]` for structured dicts not yet covered by Pydantic (Plan 02)
- Use `from __future__ import annotations` only if needed for forward refs (prefer inline for 3.11+)
- No `Any` without explicit justification comment

**Done when:** `uv run basedpyright rosetta/core` exits 0.

### Task 5: Add GitHub Actions CI

**What:** Create `.github/workflows/ci.yml` with three jobs: ruff, basedpyright, pytest.

**Files created:**
- `.github/workflows/ci.yml`

**Content:**
```yaml
name: CI

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

jobs:
  ruff:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v4
      - run: uv sync --all-extras
      - run: uv run ruff format --check .
      - run: uv run ruff check .

  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v4
      - run: uv sync --all-extras
      - run: uv run basedpyright

  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v4
      - run: uv sync --all-extras
      - run: uv run pytest -m "not slow"
```

**Done when:** File exists; `uv run pytest -m "not slow"` still exits 0 locally.

### Task 6: Update CLAUDE.md with code quality guidelines

**What:** Add a `## Code Quality` section to `CLAUDE.md` documenting the mandatory checks, annotation conventions, and Pydantic usage rules so every future session enforces them without re-deriving them.

**Files modified:**
- `CLAUDE.md`

**Section to add:**

```markdown
## Code Quality

### Mandatory checks — run before every commit

- `uv run ruff format .` — auto-format
- `uv run ruff check .` — lint (rules: E, W, F, I, UP)
- `uv run basedpyright` — static type check (strict: source, basic: tests)
- `uv run pytest -m "not slow"` — regression guard

CI enforces all four on every push/PR (`.github/workflows/ci.yml`).

### Type annotation rules

- All functions in `rosetta/core/` and `rosetta/cli/` must have explicit parameter and return annotations.
- Use **broad rdflib types**: `rdflib.term.Node | None`, `rdflib.Graph` — do not narrow to `URIRef`/`Literal` at function boundaries.
- Use `list[dict[str, Any]]` only for internal transient structures. User-facing JSON outputs must use Pydantic models (see below).
- Tests are annotated (basic mode) — annotate fixture return types and non-obvious variables.

### Pydantic models (rosetta/core/models.py)

User-facing JSON outputs are typed via Pydantic v2:

| Output | Model |
|--------|-------|
| `rosetta-lint` | `LintReport` (contains `LintFinding`, `LintSummary`, `FnmlSuggestion`) |
| `rosetta-suggest` | `SuggestionReport` (RootModel) |
| `rosetta-embed` | `EmbeddingReport` (RootModel) |

- Construct model instances in the CLI, not bare dicts.
- Serialise with `model.model_dump(mode="json")` before `json.dumps()`.
- New structured outputs must define a model in `rosetta/core/models.py` first.
```

**Done when:** `CLAUDE.md` contains the `## Code Quality` section with all three subsections.
