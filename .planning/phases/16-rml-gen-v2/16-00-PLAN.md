# Plan 16-00: SSSOM audit-log schema extension (composite-entity support)

## Goal

Extend the SSSOM audit log and suggest TSV with four columns that support the [SSSOM composite-entity pattern](https://github.com/mapping-commons/sssom/blob/master/examples/schema/composite-entities.sssom.tsv) for 1:N decomposition and N:1 aggregation mappings. This is a prerequisite for Plan 16-01 (`transform_builder`), which will read `mapping_group_id` and `composition_expr` to emit linkml-map `SlotDerivation.expr` for composite mappings.

## Context

- Current `SSSOMRow` has 11 fields (9 core + `subject_datatype` + `object_datatype`).
- Current `_SSSOM_COLUMNS` (suggest.py) lists 11 columns.
- Current `AUDIT_LOG_COLUMNS` (accredit.py) lists 9 columns — datatype columns were intentionally excluded in Phase 15 because they are schema-derived, not reviewer-asserted.
- `_parse_sssom_row` already uses `.get()` defaults, so adding new optional columns is backward-compatible with existing files.
- `append_log` builds rows positionally (list of values in `AUDIT_LOG_COLUMNS` order) — must be kept in sync with the column list.
- New columns added in this plan (per GA2 resolution, design doc §Locked decisions):
  - `subject_type` — e.g., `"composed entity expression"` on composite rows; typically empty for standard mappings.
  - `object_type` — ditto.
  - `mapping_group_id` — shared identifier for rows that compose one logical mapping. `None` for standard 1:1 rows.
  - `composition_expr` — Python/GREL expression consumed by linkml-map `SlotDerivation.expr` in 16-01.

### Task 3b — [review] Atomic migration of pre-16-00 audit log on append

**File:** `rosetta/core/accredit.py`

**Why:** `store/audit-log.sssom.tsv` exists today with a 9-column header and 9-column data rows (Phase 14 era). `append_log` only writes a header when the file is absent or empty (`accredit.py:110`). After Task 3 ships, the next `append_log` call writes 13-column data rows under the stale 9-column header — `csv.DictReader` then silently drops the four new composite fields on re-read. The audit log is the production SSSOM artifact, so this data-loss path must be closed before any 16-00 code runs against a live log.

**What to implement:**

1. At the top of `append_log`, before the file is opened for append, call a new `_migrate_audit_log_if_needed(path)` helper.
2. The helper checks the existing file's header column count against `len(AUDIT_LOG_COLUMNS)` and, if short, rewrites the entire file atomically:

```python
def _migrate_audit_log_if_needed(path: Path) -> None:
    """If the audit log at `path` has fewer columns than AUDIT_LOG_COLUMNS,
    rewrite it atomically with the current column list, padding legacy rows
    with empty strings. No-op if file is absent, empty, or already current shape.

    Atomicity: write to <path>.tmp; os.replace(tmp, path). Crash-safe on POSIX.
    """
    if not path.exists() or path.stat().st_size == 0:
        return

    text = path.read_text(encoding="utf-8")
    lines = text.splitlines()
    comment_lines = [ln for ln in lines if ln.startswith("#")]
    data_lines = [ln for ln in lines if not ln.startswith("#")]
    if not data_lines:
        return

    existing_header = data_lines[0].split("\t")
    if len(existing_header) >= len(AUDIT_LOG_COLUMNS):
        return  # already current or wider (wider is out of scope — fail loudly? see Done)

    # Migrate: rewrite header + pad rows.
    tmp = path.with_suffix(path.suffix + ".tmp")
    with tmp.open("w", encoding="utf-8", newline="") as fh:
        for ln in comment_lines:
            fh.write(ln + "\n")
        writer = csv.writer(fh, delimiter="\t", lineterminator="\n")
        writer.writerow(AUDIT_LOG_COLUMNS)
        padding = [""] * (len(AUDIT_LOG_COLUMNS) - len(existing_header))
        # Re-parse as DictReader against the OLD header so we emit values
        # under the correct new column positions. Any 11-col legacy rows
        # (subject_datatype/object_datatype from suggest output fed in) land
        # in their matching new columns; unknown legacy columns are dropped.
        old_reader = csv.DictReader(io.StringIO("\n".join(data_lines)), delimiter="\t")
        for raw_row in old_reader:
            writer.writerow(
                [raw_row.get(col, "") or "" for col in AUDIT_LOG_COLUMNS]
            )
    os.replace(tmp, path)
```

3. Call `_migrate_audit_log_if_needed(path)` as the first statement inside `append_log`, before `path.parent.mkdir(...)`.

4. Import `os` at the top of `accredit.py`.

**Edge cases:**
- If existing header is **wider** than `AUDIT_LOG_COLUMNS` (future downgrade), the helper no-ops — this plan does not support downgrade. A WARNING log to stderr is acceptable but not required.
- If the file has only a header and no data rows, rewriting is still safe and produces a clean 13-col header file.
- Comment lines (`#`-prefixed) are preserved verbatim at the top.

**Done:** After Task 3b ships, running `append_log([new_row], existing_9col_path)` produces a 13-column file with all legacy rows preserved (now padded with four empty trailing fields) + the new row appended. The migration is crash-safe (atomic `os.replace`).

```python
def check_prefix_collision(output_path: Path, schema_def: SchemaDefinition) -> None:
    """Error if any sibling *.linkml.yaml in output_path.parent has the same
    default_prefix or id as schema_def.

    Raises ValueError (caught at CLI layer, exits 1).
    """
    parent = output_path.parent
    if not parent.is_dir():  # [review] covers missing AND is_file() edge cases
        return
    new_prefix = schema_def.default_prefix
    new_id = str(schema_def.id)
    for sibling in parent.glob("*.linkml.yaml"):
        if sibling.resolve() == output_path.resolve():
            continue
        try:
            data = _yaml.safe_load(sibling.read_text(encoding="utf-8"))
        except (OSError, _yaml.YAMLError) as exc:
            # [review] Surface skipped siblings instead of failing silently.
            print(
                f"WARNING: prefix-collision check could not read {sibling}: {exc}",
                file=sys.stderr,
            )
            continue
        if not isinstance(data, dict):
            continue
        if new_prefix and data.get("default_prefix") == new_prefix:
            raise ValueError(
                f"default_prefix {new_prefix!r} already used by {sibling}. "
                "Choose a unique --schema-name."
            )
        if new_id and str(data.get("id", "")) == new_id:
            raise ValueError(
                f"id {new_id!r} already used by {sibling}. "
                "Choose a unique --schema-name."
            )
```

Use `yaml.safe_load` exclusively (bandit gate). Add `import sys` to `normalize.py` if absent.

### Task 7 — [review from 16-01] Stamp source-format + per-slot path annotations on ingest output

**Why:** Plan 16-01 review surfaced that `rosetta-yarrrml-gen` needs `source_format` as a hybrid CLI/schema-annotation lookup (GA4), and Plan 16-02's `YarrrmlCompiler` needs per-slot path hints (`rosetta_jsonpath` for JSON, `rosetta_xpath` for XML, `rosetta_csv_column` for CSV) to construct RML references. **Files:** `rosetta/core/normalize.py`, `rosetta/cli/ingest.py`, `rosetta/tests/test_ingest.py`

#### 7a — Stamp schema-level `rosetta_source_format`

In `rosetta/core/normalize.py::normalize_schema` (or right after, in the CLI), attach the detected format as a top-level schema annotation before writing:

```python
def _stamp_source_format(schema_def: SchemaDefinition, fmt: str) -> None:
    """Set `annotations.rosetta_source_format` on the LinkML schema.
    `fmt` is the normalised CLI `--format` value (one of json|csv|xml|json-sample|...
    mapped to the downstream domain: json|csv|xml). Unknown formats are not stamped."""
    domain_fmt = {
        "json": "json", "json-sample": "json", "json-schema": "json", "openapi": "json",
        "csv": "csv", "tsv": "csv",
        "xml": "xml", "xsd": "xml", "rdfs": None,  # RDFS has no source-data-format equivalent
    }.get(fmt)
    if domain_fmt is None:
        return
    existing = getattr(schema_def, "annotations", None) or {}
    # LinkML annotations are dict[str, Annotation]; tolerate plain-dict form too.
    existing["rosetta_source_format"] = domain_fmt  # LinkML accepts str on write; reader treats str OR Annotation
    schema_def.annotations = existing  # pyright: ignore[reportAttributeAccessIssue]
```

Call `_stamp_source_format(schema_def, fmt)` after `normalize_schema(...)` in `rosetta/cli/ingest.py`, before `check_prefix_collision` (order doesn't matter; collision check is orthogonal).

#### 7b — Stamp per-slot source-path annotations

For each slot on every generated class, attach the identifier needed downstream:

- **csv/tsv:** `annotations.rosetta_csv_column = <original header name>` (schema-automator's CSV importer already preserves header names as slot names; stamp that value verbatim).
- **json/json-sample/json-schema/openapi:** `annotations.rosetta_jsonpath = f"$.{slot.name}"` as a safe default (arrays / nested objects handled opportunistically — a post-phase refinement if users need deeper paths).
- **xml/xsd:** `annotations.rosetta_xpath = f"./{slot.name}"` (relative XPath under the class element).
- **rdfs:** no stamp; RDFS master schemas are not data sources.

Single helper:

```python
def _stamp_slot_paths(schema_def: SchemaDefinition, fmt: str) -> None:
    """For every slot, attach the format-specific path hint annotation consumed by 16-02."""
    annotation_key = {
        "json": "rosetta_jsonpath", "json-sample": "rosetta_jsonpath",
        "json-schema": "rosetta_jsonpath", "openapi": "rosetta_jsonpath",
        "csv": "rosetta_csv_column", "tsv": "rosetta_csv_column",
        "xml": "rosetta_xpath", "xsd": "rosetta_xpath",
    }.get(fmt)
    if annotation_key is None:
        return
    slots = getattr(schema_def, "slots", None) or {}
    for slot_name, slot in slots.items():
        annotations = getattr(slot, "annotations", None) or {}
        if annotation_key == "rosetta_csv_column":
            annotations[annotation_key] = slot_name  # original column name
        elif annotation_key == "rosetta_jsonpath":
            annotations[annotation_key] = f"$.{slot_name}"
        elif annotation_key == "rosetta_xpath":
            annotations[annotation_key] = f"./{slot_name}"
        slot.annotations = annotations  # pyright: ignore[reportAttributeAccessIssue]
```

Call order in `ingest.py` after normalize:

```python
schema_def = normalize_schema(...)
_stamp_source_format(schema_def, fmt)
_stamp_slot_paths(schema_def, fmt)
check_prefix_collision(output_path, schema_def)
# ...write yaml_dumper.dumps(schema_def)...
```
