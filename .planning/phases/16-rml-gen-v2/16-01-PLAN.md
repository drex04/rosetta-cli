# Plan 16-01: SSSOM → linkml-map TransformSpec builder

## Goal

Build `rosetta-yarrrml-gen` — a CLI that reads an approved SSSOM audit log plus source and master LinkML schemas, emits a linkml-map `TransformationSpecification` YAML, and optionally emits a `CoverageReport` JSON. This is the data-format-agnostic first half of the SSSOM → YARRRML pipeline. Plan 16-02's `YarrrmlCompiler` (in the linkml-map fork) consumes the TransformSpec to produce YARRRML.

See `16-01-SPEC.md` for architecture, data flow, and failure modes. See `CONTEXT.md` for locked decisions (including GA1–GA5 and the schema-as-registry model).

## Context

- linkml-map's `TransformationSpecification` is a Pydantic model importable from `linkml_map.datamodel.transformer_model`. Its nested types are `ClassDerivation(name, populated_from, slot_derivations)` and `SlotDerivation(name, populated_from, expr, unit_conversion, range)`. Verified by inspection of linkml-map main.
- **Container asymmetry (verified from upstream source):** `TransformationSpecification.class_derivations` is `Optional[list[ClassDerivation]]` (list). `ClassDerivation.slot_derivations` is `Optional[dict[str, SlotDerivation]]` (dict keyed by target slot name). Passing a list at the TransformationSpecification level and a dict at the ClassDerivation level matches upstream and passes `model_validate` round-trip. A `@field_validator(mode='before')` coerces dict→list on TransformationSpec, but we construct as list explicitly.
- `linkml_runtime.dumpers.YAMLDumper().dumps(obj)` serialises LinkML Pydantic models to the dict-keyed YAML shape that `linkml-tr` expects (`class_derivations:` is a dict keyed by target class name, not a list).
- `SchemaView.get_class(uri)` and `SchemaView.get_slot(uri)` resolve by `class_uri` / `slot_uri` as well as by name — useful for CURIE lookups.
- `curies.Converter` merges prefix maps with last-wins semantics; we control ordering.
- Existing `rosetta-ingest` produces LinkML outputs with unique `default_prefix` per schema (verified: `demo_out/master_cop.linkml.yaml` → `mc`, `demo_out/nor_radar.linkml.yaml` → `nor_radar`). 16-00 Task 6 adds the collision lint on top.
- Legacy `rosetta/core/rml_builder.py` (73 lines) and the `MappingDecision` model (in `models.py:143-149`) are removed — fully replaced by this plan.

#### 4b — Row filtering

```python
_EXCLUDED_PREDICATE_OWL_DIFFERENT_FROM = "owl:differentFrom"
_ACCEPTED_PREDICATES = frozenset({"skos:exactMatch", "skos:closeMatch"})
_HC = "semapv:HumanCuration"
_MMC = "semapv:ManualMappingCuration"


def filter_rows(
    rows: list[SSSOMRow],
    source_prefix: str,
    include_manual: bool,
) -> tuple[list[SSSOMRow], dict[str, list[SSSOMRow]]]:
    """Return (accepted, excluded_by_stage) where excluded_by_stage has keys:
    'prefix', 'predicate', 'justification'. owl:differentFrom rows are silently dropped.
    """
    accepted_justifications: set[str] = {_HC}
    if include_manual:
        accepted_justifications.add(_MMC)

    excluded: dict[str, list[SSSOMRow]] = {"prefix": [], "predicate": [], "justification": []}
    remaining: list[SSSOMRow] = []
    prefix_marker = f"{source_prefix}:"

    for row in rows:
        if row.predicate_id == _EXCLUDED_PREDICATE_OWL_DIFFERENT_FROM:
            continue  # silent drop (derank marker)
        if not row.subject_id.startswith(prefix_marker):
            excluded["prefix"].append(row)
            continue
        if row.predicate_id not in _ACCEPTED_PREDICATES:
            excluded["predicate"].append(row)
            continue
        if row.mapping_justification not in accepted_justifications:
            excluded["justification"].append(row)
            continue
        remaining.append(row)
    return remaining, excluded
```

#### 4c — Row classification

```python
@dataclass
class _ClassMapping:
    source_class_name: str
    target_class_name: str
    row: SSSOMRow


@dataclass
class _SlotMapping:
    source_slot_name: str
    source_owning_class: str
    target_slot_name: str
    target_owning_class: str
    row: SSSOMRow


@dataclass
class _Unresolved:
    row: SSSOMRow
    side: str  # "subject" or "object"
    reason: str


_Classification = _ClassMapping | _SlotMapping | _Unresolved
```

```python
def classify_row(
    row: SSSOMRow,
    src_view: SchemaView,
    master_view: SchemaView,
) -> _Classification:
    """Resolve subject in source schema, object in master schema.
    Mixed-kind (slot↔class) is an error: caller raises.
    """
    src_class = src_view.get_class(row.subject_id, strict=False)
    src_slot = src_view.get_slot(row.subject_id, strict=False)
    mst_class = master_view.get_class(row.object_id, strict=False)
    mst_slot = master_view.get_slot(row.object_id, strict=False)

    if src_class and mst_class:
        return _ClassMapping(
            source_class_name=str(src_class.name),  # pyright: ignore[reportUnknownMemberType]
            target_class_name=str(mst_class.name),  # pyright: ignore[reportUnknownMemberType]
            row=row,
        )
    if src_slot and mst_slot:
        owner_src = _owning_class(src_view, src_slot.name)  # pyright: ignore[reportUnknownMemberType]
        owner_mst = _owning_class(master_view, mst_slot.name)  # pyright: ignore[reportUnknownMemberType]
        return _SlotMapping(
            source_slot_name=str(src_slot.name),  # pyright: ignore[reportUnknownMemberType]
            source_owning_class=owner_src,
            target_slot_name=str(mst_slot.name),  # pyright: ignore[reportUnknownMemberType]
            target_owning_class=owner_mst,
            row=row,
        )
    if not src_class and not src_slot:
        return _Unresolved(row=row, side="subject", reason="no matching class_uri or slot_uri")
    if not mst_class and not mst_slot:
        return _Unresolved(row=row, side="object", reason="no matching class_uri or slot_uri")
    # mixed kinds — signal to caller
    return _Unresolved(row=row, side="mixed", reason="subject and object are different kinds (class vs slot)")


def _build_slot_owner_index(view: SchemaView) -> dict[str, str]:
    """**[review]** Build a slot_name → owning_class index once per SchemaView.

    Replaces the O(classes × slots) per-call scan in the legacy `_owning_class`.
    Iteration order mirrors `view.all_classes()` so "first owner wins" semantics
    match the legacy helper. Called exactly twice in `build_spec` (source + master).
    """
    index: dict[str, str] = {}
    for class_name in view.all_classes():  # pyright: ignore[reportUnknownMemberType]
        for slot in view.class_induced_slots(class_name):  # pyright: ignore[reportUnknownMemberType,reportUnknownVariableType]
            slot_key = str(slot.name)  # pyright: ignore[reportUnknownMemberType]
            index.setdefault(slot_key, str(class_name))
    return index


def _owning_class(index: dict[str, str], slot_name: str, schema_name: str) -> str:
    """Lookup in the pre-built index; raise if missing."""
    owner = index.get(slot_name)
    if owner is None:
        raise ValueError(f"slot {slot_name!r} has no owning class in schema {schema_name!r}")
    return owner
```

```python
def build_class_derivation(
    target_class: str,
    source_class: str,
    slot_derivations: list[SlotDerivation],
) -> ClassDerivation:
    return ClassDerivation(
        name=target_class,
        populated_from=source_class,
        slot_derivations={sd.name: sd for sd in slot_derivations},
    )


def build_slot_derivation(m: _SlotMapping) -> SlotDerivation:
    unit_conv: UnitConversionConfiguration | None = None
    dtype_range: str | None = None
    if m.row.subject_datatype and m.row.object_datatype:
        # GA3 — datatype or unit flow
        if m.row.subject_datatype != m.row.object_datatype:
            dtype_range = m.row.object_datatype
    # Quantity-kind heuristic (deferred to 16-02): for now, leave unit_conv None in 16-01;
    # 16-02's YarrrmlCompiler inspects SlotDerivation.range + schema units to decide.
    return SlotDerivation(
        name=m.target_slot_name,
        populated_from=m.source_slot_name,
        range=dtype_range,
        unit_conversion=unit_conv,
    )


def build_composite_slot_derivation(group_id: str, members: list[SSSOMRow]) -> SlotDerivation:
    exprs = {r.composition_expr for r in members if r.composition_expr}
    if len(exprs) != 1:
        raise ValueError(
            f"mapping_group_id={group_id!r} has inconsistent composition_expr "
            f"across rows: {sorted(exprs)!r}. Row IDs: {[r.record_id for r in members]}"
        )
    target_slots = {r.object_id for r in members}
    if len(target_slots) != 1:
        raise ValueError(
            f"mapping_group_id={group_id!r} spans multiple target slots: {sorted(target_slots)!r}"
        )
    expr = members[0].composition_expr
    target = next(iter(target_slots))
    # The target CURIE's local name is the slot name; CURIE → name resolution is caller's job.
    return SlotDerivation(name=target, expr=expr)
```

(Note the `target` above is a CURIE; the top-level `build_spec` resolves it to a schema slot name before setting `SlotDerivation.name`. Adjust in 4f.)

#### 4f — Top-level `build_spec` (orchestrator) + helpers

Per F5, split into three private helpers so `build_spec` stays small and each helper is individually testable and radon-grade-B. `--force` semantics (per F4): bypasses unresolvable-CURIE errors only. Mixed-kind, missing-class-mapping, and inconsistent composite expressions are always fatal — these are data-integrity errors, not coverage issues.

```python
def _classify_singletons(
    singletons: list[SSSOMRow],
    src_view: SchemaView,
    master_view: SchemaView,
    coverage: CoverageReport,
    force: bool,
) -> tuple[list[_ClassMapping], list[_SlotMapping]]:
    """Classify non-composite rows. Populates coverage.unresolved_*; raises on mixed-kind always; raises on unresolved unless force."""
    class_mappings: list[_ClassMapping] = []
    slot_mappings: list[_SlotMapping] = []
    for row in singletons:
        c = classify_row(row, src_view, master_view)
        if isinstance(c, _ClassMapping):
            class_mappings.append(c)
        elif isinstance(c, _SlotMapping):
            slot_mappings.append(c)
        else:
            if c.side == "mixed":
                raise ValueError(
                    f"mixed-kind mapping in row record_id={row.record_id!r}: "
                    f"subject={row.subject_id}, object={row.object_id} "
                    "(data-integrity error; --force does NOT bypass this)"
                )
            target_list = (
                coverage.unresolved_subjects if c.side == "subject" else coverage.unresolved_objects
            )
            target_list.append(
                {"curie": row.subject_id if c.side == "subject" else row.object_id, "reason": c.reason}
            )
    if (coverage.unresolved_subjects or coverage.unresolved_objects) and not force:
        raise ValueError(
            f"Unresolvable CURIEs: "
            f"{len(coverage.unresolved_subjects)} subject(s), "
            f"{len(coverage.unresolved_objects)} object(s). "
            "Pass --force to proceed with a best-effort spec."
        )
    return class_mappings, slot_mappings


def _resolve_composite_groups(
    groups: dict[str, list[SSSOMRow]],
    master_view: SchemaView,
    coverage: CoverageReport,
) -> list[tuple[str, SlotDerivation]]:
    """Resolve composite groups to (owner_class_name, SlotDerivation). Always fatal on internal inconsistency."""
    out: list[tuple[str, SlotDerivation]] = []
    for gid, members in groups.items():
        sd_tentative = build_composite_slot_derivation(gid, members)  # raises on inconsistency
        target_slot_obj = master_view.get_slot(sd_tentative.name, strict=False)
        if not target_slot_obj:
            coverage.composite_groups.append({
                "group_id": gid,
                "member_row_ids": [m.record_id or "" for m in members],
                "target_slot": sd_tentative.name,
                "resolved": False,
            })
            continue
        sd = SlotDerivation(name=str(target_slot_obj.name), expr=sd_tentative.expr)  # pyright: ignore[reportUnknownMemberType]
        owner = _owning_class(master_view, sd.name)
        out.append((owner, sd))
        coverage.composite_groups.append({
            "group_id": gid,
            "member_row_ids": [m.record_id or "" for m in members],
            "target_slot": sd.name,
            "resolved": True,
        })
    return out


def _assemble_class_derivations(
    class_mappings: list[_ClassMapping],
    slot_mappings: list[_SlotMapping],
    composite_derivations: list[tuple[str, SlotDerivation]],
    coverage: CoverageReport,
    master_view: SchemaView,  # [review] required for unmapped-required-slots computation
) -> list[ClassDerivation]:
    """Group slot derivations under their owning ClassDerivation. Fatal if any referenced class lacks a class-level mapping."""
    slots_by_target_class: dict[str, list[SlotDerivation]] = {}
    source_for_target: dict[str, str] = {}

    for cm in class_mappings:
        source_for_target[cm.target_class_name] = cm.source_class_name
        slots_by_target_class.setdefault(cm.target_class_name, [])
        coverage.resolved_class_mappings.append(f"{cm.row.subject_id} → {cm.row.object_id}")

    for sm in slot_mappings:
        slots_by_target_class.setdefault(sm.target_owning_class, []).append(build_slot_derivation(sm))
        source_for_target.setdefault(sm.target_owning_class, sm.source_owning_class)
        coverage.resolved_slot_mappings.append(f"{sm.row.subject_id} → {sm.row.object_id}")
        if sm.row.subject_datatype and sm.row.object_datatype and sm.row.subject_datatype != sm.row.object_datatype:
            coverage.datatype_warnings.append({
                "subject_id": sm.row.subject_id,
                "subject_datatype": sm.row.subject_datatype,
                "object_id": sm.row.object_id,
                "object_datatype": sm.row.object_datatype,
            })

    for owner_class, sd in composite_derivations:
        slots_by_target_class.setdefault(owner_class, []).append(sd)

    # F7: every target class referenced by any slot (singleton OR composite) must have a class-level mapping.
    for target_class in slots_by_target_class:
        if target_class not in source_for_target:
            raise ValueError(
                f"slots mapped to target class {target_class!r} but no class-level mapping "
                f"for it. Add a SSSOM row mapping the source class to {target_class}. "
                "(--force does NOT bypass this — class resolution is required for a valid TransformSpec.)"
            )

    # [review] Populate coverage.unmapped_required_master_slots for every resolved target class.
    # For each class-level mapping, inspect the master class's induced slots; any slot with
    # required=True that has NO corresponding SlotDerivation (singleton or composite) is reported.
    for target_class in source_for_target:
        master_cls = master_view.get_class(target_class, strict=False)  # pyright: ignore[reportUnknownMemberType]
        if master_cls is None:
            continue
        induced = master_view.class_induced_slots(target_class)  # pyright: ignore[reportUnknownMemberType]
        required_slot_names = {
            str(s.name)  # pyright: ignore[reportUnknownMemberType]
            for s in induced
            if bool(getattr(s, "required", False))
        }
        resolved_slot_names = {sd.name for sd in slots_by_target_class.get(target_class, [])}
        for missing in sorted(required_slot_names - resolved_slot_names):
            coverage.unmapped_required_master_slots.append(f"{target_class}.{missing}")

    return [
        build_class_derivation(
            target_class=tc,
            source_class=source_for_target[tc],
            slot_derivations=slots,
        )
        for tc, slots in slots_by_target_class.items()
    ]


def build_spec(
    sssom_rows: list[SSSOMRow],
    source: SchemaDefinition,
    master: SchemaDefinition,
    *,
    include_manual: bool = False,
    force: bool = False,
) -> tuple[TransformationSpecification, CoverageReport]:
    """Top-level orchestrator. Pure-function style; all I/O is caller's job.

    `--force` (passed by CLI) bypasses unresolvable-CURIE errors only.
    Mixed-kind mappings, missing class-level mappings, and inconsistent composite
    expressions are ALWAYS fatal — they are data-integrity errors, not coverage issues.
    """
    src_prefix = str(source.default_prefix or "")
    if not src_prefix:
        raise ValueError("source schema lacks default_prefix")
    mst_prefix = str(master.default_prefix or "")

    remaining, excluded = filter_rows(sssom_rows, src_prefix, include_manual)

    src_view = SchemaView(source)
    master_view = SchemaView(master)

    coverage = CoverageReport(
        source_schema_prefix=src_prefix,
        master_schema_prefix=mst_prefix,
        rows_total=len(sssom_rows),
        rows_after_prefix_filter=len(sssom_rows) - len(excluded["prefix"]),
        rows_after_predicate_filter=(
            len(sssom_rows) - len(excluded["prefix"]) - len(excluded["predicate"])
        ),
        rows_after_justification_filter=len(remaining),
    )
    coverage.skipped_non_exact_predicates = [
        {"row_id": r.record_id or "", "predicate_id": r.predicate_id} for r in excluded["predicate"]
    ]

    groups, singletons = group_composites(remaining)
    class_mappings, slot_mappings = _classify_singletons(
        singletons, src_view, master_view, coverage, force
    )
    composite_derivations = _resolve_composite_groups(groups, master_view, coverage)
    class_derivations = _assemble_class_derivations(
        class_mappings, slot_mappings, composite_derivations, coverage
    )

    spec = TransformationSpecification(
        id=f"https://rosetta.interop/transform/{src_prefix}-to-{mst_prefix}",
        title=f"Transform {src_prefix} → {mst_prefix}",
        class_derivations=list(class_derivations),  # upstream Pydantic type is list[ClassDerivation]
    )

    return spec, coverage
```

**Done:** `build_spec` constructs and returns both artifacts. Internal helpers unit-tested in Task 6.
