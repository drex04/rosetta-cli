### Task 2 — Create `rosetta/core/rml_runner.py`

**Goal:** Public functions `run_materialize` and `graph_to_jsonld`, plus private helpers per SPEC §4.1.

**Steps:**
1. Create module with module-level docstring and imports.
2. Implement private helpers in order:
   - `_substitute_data_path(yarrrml: str, data_path: Path) -> str` — raises `ValueError("YARRRML missing $(DATA_FILE) placeholder")` if `"$(DATA_FILE)"` not present before substitution.
   - `_build_ini(mapping_path: Path) -> str` — returns INI string with `[CONFIGURATION]` (output_format=N-TRIPLES) and `[DataSource1]` (mappings=<absolute path>).
   - `_generate_jsonld_context(master_schema_path: Path) -> dict[str, Any]` — `ContextGenerator(str(master_schema_path)).serialize()` → `json.loads(...)` → return the `@context` sub-dict (top-level if `@context` key present, else whole dict).
3. Implement `run_materialize(yarrrml_text, data_path, work_dir) -> rdflib.Graph`:
   - Substitute placeholder.
   - Write `work_dir/mapping.yml`.
   - Build INI string.
   - `return morph_kgc.materialize(ini_string)`.
4. Implement `graph_to_jsonld(graph, master_schema_path, context_output=None) -> bytes`:
   - Generate context dict.
   - Optionally write context to `context_output` (indent=2, utf-8).
   - `return graph.serialize(format="json-ld", context=context_dict, indent=2).encode("utf-8")`.
5. Type-annotate every signature and every module-level variable. Use `rdflib.Graph` at boundaries (not `URIRef`/`Literal`).
6. Commit: `feat(16-03): rml_runner — morph-kgc wrapper and JSON-LD framing`

**Done when:**
- `uv run basedpyright rosetta/core/rml_runner.py` clean.
- `uv run radon cc rosetta/core/rml_runner.py -s` — no grade C+ functions.
- `uv run vulture rosetta/core/rml_runner.py --min-confidence 80` clean (public API only).

### Task 4 — Extend `yarrrml_gen.py` CLI with `--run` flags

**Goal:** Wire runner into the CLI with the exact stdout matrix from SPEC §4.2.

**Steps:**
1. Add Click options: `--run` (flag), `--data` (Path, exists, not-required at Click level — validated in handler), `--jsonld-output` (Path), `--workdir` (Path), `--context-output` (Path).
2. After the existing step 9 (write TransformSpec YAML):
   - Guard: `if not run: sys.exit(0)`.
   - Validate: `if not data: click.echo("Error: --run requires --data", err=True); sys.exit(1)`.
3. Import `YarrrmlCompiler`, `run_materialize`, `graph_to_jsonld` at module top.
4. Compile spec to YARRRML in-process:
   ```python
   compiler = YarrrmlCompiler(spec, source_schema=source_schema, target_schema=master_schema)
   yarrrml_text = compiler.serialize()   # or compile() — confirm the fork's public method name during Task 4 impl
   ```
   Wrap in try/except → exit 1 on error.
5. Resolve `work_dir`: if `--workdir`, use it (mkdir if absent); else `tempfile.mkdtemp(prefix="rosetta-yarrrml-")`. If tempdir, register `atexit.register(shutil.rmtree, wd, ignore_errors=True)`.
6. Call `graph = run_materialize(yarrrml_text, Path(data), Path(work_dir))` in try/except → exit 1.
7. Call `jsonld_bytes = graph_to_jsonld(graph, Path(master_schema), context_output=Path(context_output) if context_output else None)` in try/except → exit 1.
8. Write JSON-LD per stdout matrix:
   - If `--jsonld-output`: write bytes to that path.
   - Else: `sys.stdout.buffer.write(jsonld_bytes)`.
9. Verify CLI option placement and names match SPEC §4.2 exactly.
10. Commit: `feat(16-03): --run flag for rosetta-yarrrml-gen (YARRRML → JSON-LD)`

**Done when:**
- `uv run rosetta-yarrrml-gen --help` shows all new flags.
- Existing non-`--run` invocations still pass through unchanged (regression-check in Task 5).
- basedpyright clean on `cli/yarrrml_gen.py`.

**Open during implementation:** Confirm `YarrrmlCompiler`'s public entry method name (`compile()` vs `serialize()`) — look at `test_yarrrml_compile_integration.py` usages and match them exactly. Document the chosen name in the commit body.
