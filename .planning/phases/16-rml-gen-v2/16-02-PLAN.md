# Plan 16-02: YarrrmlCompiler in forked linkml-map

## Goal

Add a `YarrrmlCompiler` to the forked `linkml-map` repo that compiles a `TransformationSpecification` into valid YARRRML consumable by morph-kgc. Also extend rosetta-cli's `build_spec()` to populate `source_schema`/`target_schema` fields (prerequisite), pin the fork in `pyproject.toml`, and add an integration test.

## Prerequisites

- **Plan 16-01 complete** (TransformSpec builder + CLI live).
- **Fork exists** at `https://github.com/drex04/linkml-map` (user-created).
- **Cross-phase contracts** from 16-01: per-slot path annotations on source schema; `spec.comments = ["rosetta:source_format=<fmt>"]`.

## Context

See `CONTEXT.md` for locked decisions. Key ones for this plan:
- **A2:** Subclass `Compiler` directly, own Jinja env with `autoescape=False`
- **B1:** `elif target == "yarrrml"` in CLI
- **C2+C1:** Populate `spec.source_schema`/`spec.target_schema`; CLI `--target-schema` override
- **GA-02-1:** Auto-detect identifier slot with heuristic fallback
- **GA-02-2b:** Composites as separate TriplesMaps with parentTriplesMap reference
- **GA-02-3:** Always emit `datatype:` when `SlotDerivation.range` is set
- **GA-02-4:** `$(DATA_FILE)` placeholder in sources block
- **E2:** GREL pre-computed in Python

### Task 2 — Implement `YarrrmlCompiler` class (linkml-map fork)

**File:** `src/linkml_map/compiler/yarrrml_compiler.py`

```python
from dataclasses import dataclass, field
from pathlib import Path
from typing import Any

from jinja2 import Environment, FileSystemLoader
from linkml_runtime import SchemaView

from linkml_map.compiler.compiler import CompiledSpecification, Compiler
from linkml_map.datamodel.transformer_model import TransformationSpecification


YARRRML_TEMPLATE_DIR = str(Path(__file__).parent / "templates")


@dataclass
class YarrrmlCompiler(Compiler):
    """Compiles a TransformationSpecification to YARRRML."""

    target_schemaview: SchemaView | None = None

    def compile(self, specification: TransformationSpecification) -> CompiledSpecification:
        ...
```

**Key implementation details:**

**2a. Schema resolution (`_resolve_schemas`):**
- Read `specification.source_schema` → `SchemaView(source_schema)` (use `self.source_schemaview` if already set by parent)
- Read `specification.target_schema` → `SchemaView(target_schema)` (use `self.target_schemaview` if set, else resolve from spec field)
- Raise `ValueError("source_schema not set on TransformSpec and not provided via constructor")` if neither available
- Same for target

**2b. Source format extraction (`_resolve_source_format`):**
- Scan `specification.comments` for entry matching `rosetta:source_format=<fmt>`
- Return the format string (`json`, `csv`, `xml`)
- Raise `ValueError` if not found

**2c. Subject template resolution (`_resolve_subject_template`):**
- Given a class name and source `SchemaView`:
  1. Check `source_view.get_identifier_slot(class_name)` — if found, use that slot name
  2. Fall back to slots named `id`, `identifier`, `{class_name}_id` (case-insensitive)
  3. If none found, raise `ValueError(f"No identifier slot found for class {class_name}")`
- Return template string: `{prefix}:{class_name}/$({{identifier_slot}})`

**2d. Reference resolution (`_resolve_reference`):**
- Given a slot name, source `SchemaView`, and format:
  - Look up slot's annotations for the format-appropriate key:
    - `csv` → `rosetta_csv_column`
    - `json` → `rosetta_jsonpath`
    - `xml` → `rosetta_xpath`
  - If annotation found, return its value (e.g., `$(latitude)` for CSV, `$.latitude` for JSON)
  - If not found, fall back to `$(slot_name)` with stderr warning
  - For JSON: wrap in YARRRML JSONPath syntax `$.path`
  - For XML: wrap in XPath syntax
  - For CSV: wrap in `$(column_name)`

**2e. GREL for linear unit conversion (`_grel_for_linear`):**
- Takes `UnitConversionConfiguration` with `source_unit` and `target_unit`
- Use known linear conversion factors (metric table or pint library if available)
- Return GREL string: `"value.toNumber() * {m} + {b}"` where `m` is scale, `b` is offset
- If conversion is nonlinear or unknown, raise `ValueError` (caller emits warning, falls back to raw value)

**2f. Composite extraction (`_extract_composites`):**
- Walk `class_derivation.slot_derivations`
- Identify slots with `expr` set (these are composite derivations from 16-01)
- Group by naming convention: slots from the same `mapping_group_id` (encoded in `SlotDerivation.comments` by 16-01) form one composite TriplesMap
- Return list of composite context dicts for the template

**2g. Build mapping context (`_build_mapping_context`):**
- For each `ClassDerivation`:
  - Resolve subject template (2c)
  - Build `sources:` entry with `$(DATA_FILE)` placeholder + format iterator
  - For each `SlotDerivation`:
    - Resolve reference (2d)
    - Look up target slot URI from target SchemaView
    - If `unit_conversion` set: compute GREL (2e), emit function-mapped po entry
    - If `range` set: emit `datatype:` annotation (GA-02-3)
    - Else: emit simple `po:` entry
  - Extract composites (2f): emit separate mapping blocks + reference entries

**2h. `compile()` orchestrator:**
- Call `_resolve_schemas(spec)`
- Call `_resolve_source_format(spec)`
- Build mapping contexts for all class derivations
- Build prefix map from source + target schema prefixes
- Set up Jinja `Environment(loader=FileSystemLoader(YARRRML_TEMPLATE_DIR), autoescape=False)`
- Render `yarrrml.j2` with context: `prefixes`, `mappings`
- Return `CompiledSpecification(serialization=rendered)`

**Done:** `YarrrmlCompiler` class implemented with all helper methods; imports resolve; no tests yet (Task 4).

### Task 3 — Write `yarrrml.j2` template (linkml-map fork)

**File:** `src/linkml_map/compiler/templates/yarrrml.j2`

Single flat Jinja2 template. Receives context from `YarrrmlCompiler.compile()`:

```
prefixes:
{% for prefix, uri in prefixes.items() %}
  {{ prefix }}: {{ uri }}
{% endfor %}

mappings:
{% for mapping in mappings %}
  {{ mapping.name }}:
    sources:
{% for source in mapping.sources %}
      - [{{ source }}]
{% endfor %}
    s: {{ mapping.subject }}
    po:
{% for po in mapping.predicateobjects %}
{% if po.function %}
      - p: {{ po.predicate }}
        o:
          function: {{ po.function.name }}
          parameters:
{% for param in po.function.parameters %}
            - [{{ param.name }}, {{ param.value }}]
{% endfor %}
{% elif po.mapping %}
      - p: {{ po.predicate }}
        o:
          mapping: {{ po.mapping }}
{% elif po.datatype %}
      - p: {{ po.predicate }}
        o:
          value: {{ po.reference }}
          datatype: {{ po.datatype }}
{% else %}
      - [{{ po.predicate }}, {{ po.reference }}]
{% endif %}
{% endfor %}
{% endfor %}
```

Exact template structure will be refined during implementation based on YARRRML spec requirements. Key invariants:
- `prefixes:` block always present
- Each mapping has `sources:`, `s:`, `po:` sections
- Function-mapped entries use FnO vocabulary
- Composite references use `mapping:` syntax
- No HTML escaping anywhere

**Done:** Template renders valid YARRRML when fed well-formed context dicts; `yaml.safe_load()` succeeds on output.
