# Spec: Plan 16-01

## Problem

Produce a valid linkml-map `TransformationSpecification` YAML from an approved SSSOM audit log + a pair of LinkML schemas (source, master), so that Plan 16-02's `YarrrmlCompiler` (living in the linkml-map fork) can consume the TransformSpec as its sole input and emit YARRRML.

The builder is data-format agnostic — it knows nothing about JSON/CSV/XML or JSONPath/XPath. Those concerns live in the compiler (16-02). The builder only does schema-level reasoning: which SSSOM rows describe class mappings, which describe slot mappings, which belong to composite groups, which are datatype-lossy, and whether every CURIE resolves.

## Inputs / outputs

**Inputs:**

| Input | Form | Notes |
|-------|------|-------|
| Approved SSSOM audit log | TSV (post-16-00 format, 13 columns) | Typically `store/audit-log.sssom.tsv` |
| Source schema | LinkML YAML | Produced by `rosetta-ingest` |
| Master schema | LinkML YAML | Produced by `rosetta-ingest` from `.ttl`/`.owl` |
| `rosetta.toml` globals | `[prefixes]` table | `skos`, `semapv`, `xsd`, `qudt` |
| CLI flags | `--source-format [json\|csv\|xml]`, `--include-manual`, `--allow-empty`, `--force`, `--coverage-report <path>`, `--output <path>` | See CLI surface |

**Outputs:**

| Output | Form | Sink |
|--------|------|------|
| `TransformationSpecification` | YAML (dict-keyed class_derivations, serialized via `linkml_runtime.dumpers.YAMLDumper`) | stdout or `--output <path>` |
| `CoverageReport` | JSON (new Pydantic model) | `--coverage-report <path>` if provided; else not emitted |

## Data flow

```
audit-log.sssom.tsv ──┐
                      │
source.linkml.yaml ───┤
                      │  (curies.Converter: merge source + master + rosetta.toml)
master.linkml.yaml ───┤
                      │
rosetta.toml  ────────┘
                      │
                      ▼
              filter by prefix
         (subject_id.startswith(src_prefix + ":"))
                      │
                      ▼
              filter by predicate
        (skos:exactMatch or skos:closeMatch; GA1)
                      │
                      ▼
              filter by justification
        (default: semapv:HumanCuration; --include-manual adds MMC)
                      │
                      ▼
              group by mapping_group_id (non-null)
                      │
                      ├─► composite groups  ──► SlotDerivation(name=target, expr=composition_expr)
                      │
                      └─► singletons        ──► resolve via SchemaView:
                                                  is subject a class?     → ClassDerivation
                                                  is subject a slot?      → SlotDerivation under owning class
                                                  datatype mismatch?      → slot_derivation.range = target_datatype
                                                  quantity-kind mismatch? → slot_derivation.unit_conversion.target_unit
                      │
                      ▼
          assemble TransformationSpecification
                      │
                      ▼
             validate via linkml-map Pydantic
                      │
                      ▼
              YAMLDumper.dumps(spec) → stdout / file
             CoverageReport.model_dump(mode='json') → --coverage-report file
```

## Key components

### `rosetta/core/transform_builder.py` (new module)

Pure-function style; no I/O except schema loading via `SchemaView`.

| Function | Purpose |
|----------|---------|
| `build_converter(source: SchemaDefinition, master: SchemaDefinition, toml_prefixes: dict) -> curies.Converter` | Merge prefix maps into a single Converter. |
| `filter_rows(rows: list[SSSOMRow], source_prefix: str, include_manual: bool) -> list[SSSOMRow]` | Apply prefix, predicate (GA1), and justification filters. |
| `classify_row(row: SSSOMRow, src_view: SchemaView, master_view: SchemaView, converter: Converter) -> RowClassification` | Internal tagged-union dataclass: ClassMapping \| SlotMapping \| Unresolved. |
| `group_composites(rows: list[SSSOMRow]) -> tuple[dict[str, list[SSSOMRow]], list[SSSOMRow]]` | Split into `(groups_by_id, singletons)`. |
| `build_class_derivation(...)` / `build_slot_derivation(...)` / `build_composite_slot_derivation(...)` | Emit linkml-map Pydantic objects. |
| `build_spec(...) -> tuple[TransformationSpecification, CoverageReport]` | Top-level — runs the pipeline, returns both outputs. |

### `rosetta/cli/yarrrml_gen.py` (renamed from `rml_gen.py`)

| Responsibility | Detail |
|---------------|--------|
| Parse CLI flags | All options live at the command level |
| Load inputs | `parse_sssom_tsv`, `yaml_loader` for LinkML, `toml` for `rosetta.toml` |
| Invoke builder | `build_spec(...)` |
| Serialize | `YAMLDumper().dumps(spec)` to `open_output(output)` |
| Coverage report | Only if `--coverage-report` provided; `model_dump_json(indent=2)` |
| Exit codes | `0` success; `1` unresolvable CURIEs or empty filtered set (unless `--force` / `--allow-empty`) |

### `rosetta/core/models.py` additions

```python
class CoverageReport(BaseModel):
    source_schema_prefix: str
    master_schema_prefix: str
    rows_total: int
    rows_after_prefix_filter: int
    rows_after_predicate_filter: int
    rows_after_justification_filter: int
    resolved_class_mappings: list[str]     # "src:Track → master:Track"
    resolved_slot_mappings: list[str]      # "src:speed_kts → master:airSpeed"
    unresolved_subjects: list[str]         # CURIE + reason
    unresolved_objects: list[str]
    skipped_non_exact_predicates: list[dict]  # {"row_id", "predicate_id"}
    composite_groups: list[dict]           # {"group_id", "member_rows", "target_slot"}
    datatype_warnings: list[dict]          # {"subject_id", "subject_datatype", "object_id", "object_datatype"}
    unmapped_required_master_slots: list[str]
```

## Failure modes

| Mode | Trigger | Response |
|------|---------|----------|
| Unresolvable `subject_id` | CURIE doesn't resolve in source schema | Log to `coverage.unresolved_subjects`; **exit 1** unless `--force` |
| Unresolvable `object_id` | CURIE doesn't resolve in master schema | Same |
| Class-level mapping missing | Source slots have mappings but owning class isn't mapped | Exit 1 with `add class mapping for src:X` |
| Mixed-kind mapping | `subject_id` resolves as slot, `object_id` resolves as class (or vice versa) | Exit 1 with clear error; coverage report lists row |
| Empty filtered set | No rows survive filters | Exit 1 unless `--allow-empty` |
| Predicate not in {exact, close} (GA1) | `skos:narrowMatch` etc. | Skip row; log to `coverage.skipped_non_exact_predicates` |
| `mapping_justification` not Human/Manual curated | Auto-suggested, not approved | Filter out silently |
| Composite group has inconsistent `composition_expr` | Multiple rows in same `mapping_group_id` disagree | Exit 1 with row IDs |
| Composite group spans multiple target slots | Ambiguous grouping | Exit 1; document expectation |
| `owl:differentFrom` row | Phase 14 derank marker | Always excluded silently |
| linkml-map Pydantic validation fails | Model mismatch (unlikely with verified Pydantic classes) | Exit 1, raise |
| Source schema lacks `default_prefix` | Malformed schema | Exit 1 with message |
| Builder output YAML can't round-trip through `linkml-map.TransformationSpecification.model_validate` | Regression | Test guard: dump → validate → compare |

## Out of scope (deferred)

- JSONPath / XPath / CSV column reference generation (Plan 16-02 compiler)
- Unit conversion FnML GREL generation (Plan 16-02)
- morph-kgc execution (Plan 16-03)
- JSON-LD framing (Plan 16-03)
- User review UX for composing `mapping_group_id` + `composition_expr` (Phase 17+)
- Inference of 1:N/N:1 decomposition/aggregation expressions (user-authored)
- Running against the linkml-map fork in CI — 16-01 depends only on upstream linkml-map's public Pydantic model (imported path may shift if fork renames, mitigation: pin SHA)

## External dependencies added to `pyproject.toml`

- `linkml-map >= 0.3.0` (or latest upstream; pinned to a SHA once the fork exists in 16-02 — for now, pin to a release so 16-01 builds cleanly)
- `curies >= 0.7.0`

## Validation & acceptance

The plan is complete when:
1. `rosetta-yarrrml-gen --source-schema nor.linkml.yaml --master-schema master.linkml.yaml --sssom store/audit-log.sssom.tsv --source-format csv --output mapping.transform.yaml` emits a valid TransformSpec that round-trips through linkml-map's `TransformationSpecification.model_validate`.
2. `rosetta-yarrrml-gen ... --coverage-report coverage.json` emits a valid `CoverageReport` JSON matching the model.
3. All failure modes produce exit 1 with the documented error messages; `--force` / `--allow-empty` escape hatches work.
4. README documents the new CLI, flags, and output formats.
5. All eight quality gates pass clean.
