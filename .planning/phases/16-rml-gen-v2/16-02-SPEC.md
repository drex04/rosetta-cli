# Spec: Plan 16-02

## Problem

Compile a linkml-map `TransformationSpecification` into valid YARRRML that morph-kgc can execute. The TransformSpec (produced by 16-01's `rosetta-yarrrml-gen`) is data-format-agnostic; the compiler bridges the gap by reading source-schema path annotations (`rosetta_csv_column`, `rosetta_jsonpath`, `rosetta_xpath`) and the effective source format to emit format-specific RML references.

This work lives in a **fork** of `linkml/linkml-map` (`https://github.com/drex04/linkml-map`), not in rosetta-cli. rosetta-cli pins the fork's SHA in `pyproject.toml`.

## Inputs / outputs

| Direction | Artifact | Format |
|-----------|----------|--------|
| **In** | `TransformationSpecification` YAML | linkml-map datamodel (from `rosetta-yarrrml-gen`) |
| **In** | Source LinkML schema | YAML (carries per-slot path annotations) |
| **In** | Target/master LinkML schema | YAML (carries slot URIs, class URIs, datatypes) |
| **Out** | YARRRML document | YAML (consumable by morph-kgc or yatter) |

## Data flow

```
TransformSpec YAML
       │
       ▼
┌──────────────────────────────┐
│  YarrrmlCompiler.compile()   │
│  ┌─────────────────────────┐ │
│  │ 1. Parse source_schema  │ │  ← SchemaView(spec.source_schema)
│  │    + target_schema from  │ │  ← SchemaView(spec.target_schema)
│  │    spec fields or CLI    │ │
│  ├─────────────────────────┤ │
│  │ 2. Extract source_format│ │  ← spec.comments["rosetta:source_format=…"]
│  ├─────────────────────────┤ │
│  │ 3. For each class_deriv:│ │
│  │  a. Resolve subject ID  │ │  ← identifier slot heuristic (GA-02-1)
│  │  b. Build sources: block│ │  ← format → iterator (GA-02-4)
│  │  c. For each slot_deriv:│ │
│  │    - Resolve reference   │ │  ← annotation lookup by format
│  │    - Emit po: entry      │ │
│  │    - If unit_conversion: │ │
│  │      pre-compute GREL    │ │  ← _grel_for_linear() (E2)
│  │  d. Composite groups →   │ │  ← separate TriplesMap (GA-02-2b)
│  │     parentTriplesMap ref  │ │
│  ├─────────────────────────┤ │
│  │ 4. Render yarrrml.j2     │ │
│  └─────────────────────────┘ │
└──────────────────────────────┘
       │
       ▼
   YARRRML YAML string
   (CompiledSpecification.serialization)
```

## Key components

### `YarrrmlCompiler` (compiler class)

**Location:** `src/linkml_map/compiler/yarrrml_compiler.py`

Subclasses `Compiler` directly (not `J2BasedCompiler`) because the parent's `autoescape=True` corrupts YAML output. Manages its own `jinja2.Environment(autoescape=False)`.

**Responsibilities:**
- Load source + target `SchemaView` from `spec.source_schema` / `spec.target_schema` (or constructor overrides)
- Extract `rosetta:source_format` from `spec.comments`
- For each `ClassDerivation`: resolve subject identifier, build YARRRML mapping block
- For each `SlotDerivation`: look up source-schema annotation for the reference path; emit `po:` entry with appropriate predicate URI, reference, and optional datatype
- For composite groups: emit separate mapping blocks; owning class references via `o: mapping: <name>`
- For `unit_conversion` slots: pre-compute GREL expression string and attach as FnML function reference in the `po:` entry
- Build `sources:` block with `$(DATA_FILE)` placeholder and format-appropriate iterator
- Render via `yarrrml.j2` template

**Key methods:**
- `compile(spec) → CompiledSpecification` — main entry, overrides `Compiler.compile()`
- `_resolve_schemas(spec)` — load SchemaViews from spec fields or constructor overrides
- `_resolve_source_format(spec) → str` — parse `rosetta:source_format=X` from `spec.comments`
- `_build_mapping_context(class_deriv, source_view, target_view, fmt) → dict` — per-class template context
- `_resolve_reference(slot_name, source_view, fmt) → str` — annotation-based path lookup
- `_resolve_subject_template(class_name, source_view) → str` — identifier slot heuristic
- `_grel_for_linear(source_unit, target_unit) → str` — linear unit conversion GREL string
- `_extract_composites(class_deriv) → list[dict]` — identify composite groups needing separate TriplesMaps

### `yarrrml.j2` (Jinja2 template)

**Location:** `src/linkml_map/compiler/templates/yarrrml.j2`

Receives a context dict with:
- `prefixes: dict[str, str]` — CURIE prefix map
- `mappings: list[MappingContext]` — one per class + composites
- Each `MappingContext` has: `name`, `sources`, `subject_template`, `predicateobjects: list[POEntry]`, optional `functions: list[FnEntry]`

YARRRML structure emitted:
```yaml
prefixes:
  ex: http://example.org/
  # ... from source + target schema prefixes

mappings:
  ClassName:
    sources:
      - [$(DATA_FILE)~csv]    # or ~json with iterator, ~xpath with iterator
    s: ex:classname/$(identifier_slot)
    po:
      - [ex:hasProperty, $(source_column)]
      - p: ex:hasUnit
        o:
          value: "grel:value * 0.3048"
          datatype: xsd:float
      - p: ex:hasComposite
        o:
          mapping: CompositeName

  CompositeName:
    sources:
      - [$(DATA_FILE)~csv]
    s: ex:composite/$(id)
    po:
      - [ex:lat, $(latitude)]
      - [ex:lon, $(longitude)]
```

### CLI patch (fork)

**Location:** `src/linkml_map/cli/cli.py`

Add `elif target == "yarrrml"` in the `compile` command. Add `--target-schema` option to supply master schema path (falls back to `spec.target_schema` field).

## Architecture decisions

| # | Decision | Rationale |
|---|----------|-----------|
| A2 | Subclass `Compiler` directly | `J2BasedCompiler.autoescape=True` corrupts YAML |
| B1 | if/elif CLI patch | Smallest diff; no plugin registry needed; PR deferred |
| C2+C1 | `spec.source_schema` + `spec.target_schema` + CLI override | Self-describing spec; override for flexibility |
| D1 | Single flat template | Start simple; refactor to macros if >100 lines |
| E2 | GREL pre-computed in Python | Template stays declarative; math is testable |
| GA-02-2b | Composites as separate TriplesMaps | RML-idiomatic; parentTriplesMap references |

## Failure modes

| Failure | Detection | Recovery |
|---------|-----------|----------|
| `spec.source_schema` missing and no `--source-schema` CLI | Compiler raises `ValueError` before render | User re-runs `rosetta-yarrrml-gen` with updated builder (Task 0) |
| Source schema slot lacks path annotation for the format | Compiler emits warning + skips slot | Coverage-report-style stderr listing skipped slots |
| No identifier slot found for a class | Compiler raises `ValueError` | User adds `identifier: true` on the source schema slot |
| Unit conversion references unknown unit | `_grel_for_linear` raises `ValueError` | Compiler emits warning, skips FnML, emits raw `populated_from` |
| `autoescape` contaminates YAML output | Integration test validates YARRRML round-trip via `yaml.safe_load` | Using own Environment with `autoescape=False` prevents this |
| Fork diverges from upstream | Pin SHA; rebase deliberately | Integration tests catch regressions |
