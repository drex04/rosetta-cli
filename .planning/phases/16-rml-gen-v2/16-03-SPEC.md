# Plan 16-03 SPEC — morph-kgc Runner + JSON-LD Framing + E2E

## 1. Goal

Execute the YARRRML produced by the forked `YarrrmlCompiler` against real data via `morph_kgc.materialize()` and frame the resulting graph as JSON-LD using a context derived from the master LinkML schema. Expose via `rosetta-yarrrml-gen --run`. Prove with a slow E2E test.

## 2. Non-goals

- JSON-LD `@frame` output (deferred).
- Multi-source data binding (single-source only).
- Non-linear unit conversion via Python UDF.
- Upstream PRs to `morph-kgc` or `linkml`.

## 3. Architecture

### 3.1 Data flow

```
┌──────────────────────────────────────────────────────────────┐
│  rosetta-yarrrml-gen --run --data DATA.csv                   │
│                                                              │
│  1. build_spec(rows, source_def, master_def, …)              │
│      → TransformSpec (in memory)                             │
│                                                              │
│  2. YarrrmlCompiler(spec, source_schema=…, target_schema=…)  │
│      .compile()                                              │
│      → YARRRML YAML text (with $(DATA_FILE) placeholder)     │
│                                                              │
│  3. rml_runner.run_materialize(yarrrml_text, data_path, wd)  │
│      a. substitute $(DATA_FILE) → data_path                  │
│      b. write wd/mapping.yml                                 │
│      c. build INI string → morph_kgc.materialize(ini)        │
│      → rdflib.Graph                                          │
│                                                              │
│  4. rml_runner.graph_to_jsonld(graph, master_schema_path)    │
│      a. ContextGenerator(master_schema).serialize()          │
│      b. graph.serialize(format="json-ld", context=ctx)       │
│      → bytes                                                 │
│                                                              │
│  5. write bytes to stdout or --jsonld-output                 │
└──────────────────────────────────────────────────────────────┘
```

## 4. Interfaces

### 4.1 `rml_runner.py` public API

```python
def run_materialize(
    yarrrml_text: str,
    data_path: Path,
    work_dir: Path,
) -> rdflib.Graph: ...

def graph_to_jsonld(
    graph: rdflib.Graph,
    master_schema_path: Path,
    context_output: Path | None = None,
) -> bytes: ...
```

Private helpers:
- `_substitute_data_path(yarrrml: str, data_path: Path) -> str` — replaces `$(DATA_FILE)`; raises `ValueError` if placeholder absent.
- `_build_ini(mapping_path: Path) -> str` — emits INI with `[CONFIGURATION]` + `[DataSource1]` sections.
- `_generate_jsonld_context(master_schema_path: Path) -> dict` — calls `ContextGenerator.serialize()`, parses to dict (so rdflib gets a dict, not a string).

### 4.2 CLI additions (`rosetta-yarrrml-gen`)

| Flag | Type | Default | Notes |
|------|------|---------|-------|
| `--run` | flag | off | Enables materialization stage |
| `--data` | Path (exists) | — | Required when `--run`; source data file |
| `--jsonld-output` | Path | — | Optional JSON-LD output file; stdout if omitted |
| `--workdir` | Path | tempdir | If supplied, artifacts retained for debugging |
| `--context-output` | Path | — | Optional JSON-LD `@context` dump |

**Exit codes:**
- `0` — success
- `1` — `--run` without `--data`, morph-kgc error, context generation error, placeholder missing
- `2` — Click validation error (data file missing, bad path, etc.)

**stdout matrix:**

| `--run` | `--output` | `--jsonld-output` | stdout |
|---------|------------|-------------------|--------|
| off | — | n/a | TransformSpec YAML |
| off | set | n/a | (empty; YAML to file) |
| on | — | — | JSON-LD |
| on | — | set | (empty; JSON-LD to file) |
| on | set | — | JSON-LD (YAML also written to `--output`) |

### 4.3 Python imports

- `morph_kgc` — must import at module level of `rml_runner.py`
- `rdflib` — already a dep
- `linkml.generators.jsonldcontextgen.ContextGenerator` — already a dep via `linkml`
- `YarrrmlCompiler` — `from linkml_map.compiler.yarrrml_compiler import YarrrmlCompiler`

## 5. Error handling

All runner calls from the CLI are wrapped in `try/except` that converts library exceptions to `click.echo(..., err=True); sys.exit(1)`. No raw tracebacks reach the user. Exception types caught:

- `morph_kgc` → generic `Exception` (morph-kgc does not declare a typed error hierarchy)
- `linkml` context generation → generic `Exception`
- `ValueError` from `_substitute_data_path` (placeholder missing)
- `OSError` for file writes

The runner itself raises cleanly; the CLI is responsible for exit semantics.

## 6. Test strategy

### 6.1 Unit (`test_rml_runner.py`, fast)

- `test_substitute_data_path_replaces_placeholder`
- `test_substitute_data_path_raises_when_placeholder_missing`
- `test_build_ini_has_configuration_and_datasource_sections`
- `test_generate_jsonld_context_returns_dict` — small inline master schema fixture
- `test_run_materialize_with_inline_yarrrml_and_csv` — minimal YARRRML + 2-row CSV; assert graph has expected triples
- `test_graph_to_jsonld_preserves_typed_instance` — build small graph in-test, serialize, parse JSON-LD back, assert `@type`

### 6.2 CLI (`test_yarrrml_gen.py`, fast — new tests added)

- `test_run_without_data_exits_1`
- `test_run_with_morph_kgc_error_exits_1` (monkeypatch `run_materialize` to raise)
- `test_run_happy_path_writes_jsonld_to_stdout` (monkeypatch runner to return fixed graph)
- `test_run_with_jsonld_output_writes_to_file_and_stdout_empty`

### 6.3 E2E (`test_yarrrml_run_e2e.py`, **slow**)

Single test: `test_e2e_nor_radar_csv_to_jsonld`
- Seeds fresh audit log with a couple of approved mappings using existing accredit helpers.
- Invokes `rosetta-yarrrml-gen --run --data fixtures/nor_radar_sample.csv ...` via `CliRunner`.
- Asserts exit 0, stdout parses as JSON-LD, `@context` contains master `default_prefix`, `@graph` has ≥1 `@type` entry referencing a master class, one unit-converted numeric field has the expected converted value.
