5. `rosetta-lint --strict` treats WARNING findings as BLOCKs, causing exit 1 for unit-mismatch warnings.

7. [review] When `units_compatible()` returns `None` because both unit IRIs are present but their dimension vectors are missing from qudt_units.ttl, the finding is INFO with `rule: unit_vector_missing`.
8. [review] The `summary` block in the output reflects final (post-`--strict`) severities — if `--strict` is set, `summary.warning` is 0 and all promoted findings count toward `summary.block`.

## Tasks

### Task 1 — Create rosetta/policies/qudt_units.ttl

Curated QUDT unit subset for defense/military schemas. Include the ~20 units that appear in the master ontology and national schemas: M, FT, KiloM, KiloM-PER-HR, M-PER-SEC, KN (knots), DEG, RAD, KiloGM, GM, SEC, MIN, HR, PA (pascal), K (kelvin), degC, dimensionless, and a few more.

Each unit entry must have:
- `rdf:type qudt:Unit`
- `rdfs:label`
- `qudt:symbol` (the short symbol string)
- `qudt:hasDimensionVector` as a plain string literal (e.g. `"A0E0L1I0M0H0T0D0"` for length)

Dimension vector format: `"A{a}E{e}L{l}I{i}M{m}H{h}T{t}D{d}"` where letters are SI base dimensions:
- A = amount of substance, E = electric current, L = length, I = luminous intensity
- M = mass, H = thermodynamic temperature, T = time, D = dimensionless

Defense-relevant dimension vectors:
- Length: `A0E0L1I0M0H0T0D0`
- Speed (L/T): `A0E0L1I0M0H0T-1D0`
- Angle (dimensionless): `A0E0L0I0M0H0T0D0`
- Mass: `A0E0L0I0M1H0T0D0`
- Time: `A0E0L0I0M0H0T1D0`
- Pressure (M/LT²): `A0E0L-1I0M1H0T-2D0`
- Temperature: `A0E0L0I0M0H1T0D0`

**done:** File exists at `rosetta/policies/qudt_units.ttl`, parses with rdflib, contains `qudt:hasDimensionVector` for each unit.

### Task 2 — Create rosetta/policies/fnml_registry.ttl

Turtle RDF registry of unit conversion functions. Use proper QUDT unit IRIs and a `rose:conversion` predicate pattern.

Prefixes needed:
```turtle
@prefix rose:  <http://rosetta.interop/ns/> .
@prefix unit:  <http://qudt.org/vocab/unit/> .
@prefix fnml:  <http://rosetta.interop/fnml/> .
@prefix rdfs:  <http://www.w3.org/2000/01/rdf-schema#> .
@prefix qudt:  <http://qudt.org/schema/qudt/> .
```

Triple pattern per conversion:
```turtle
fnml:ConvertFtToM
    a rose:ConversionFunction ;
    rose:fromUnit  unit:FT ;
    rose:toUnit    unit:M ;
    rdfs:label     "Convert feet to metres"@en ;
    qudt:conversionMultiplier 0.3048 .
```

Include at minimum:
- `fnml:ConvertFtToM` — unit:FT → unit:M (× 0.3048)
- `fnml:ConvertMToFt` — unit:M → unit:FT (× 3.28084)
- `fnml:ConvertKiloMToM` — unit:KiloM → unit:M (× 1000)
- `fnml:ConvertMToKiloM` — unit:M → unit:KiloM (× 0.001)
- `fnml:ConvertKphToMs` — unit:KiloM-PER-HR → unit:M-PER-SEC (× 0.277778)
- `fnml:ConvertMsToKph` — unit:M-PER-SEC → unit:KiloM-PER-HR (× 3.6)
- `fnml:ConvertKnToMs` — unit:KN → unit:M-PER-SEC (× 0.514444)
- `fnml:ConvertDegToRad` — unit:DEG → unit:RAD (× π/180)
- `fnml:ConvertRadToDeg` — unit:RAD → unit:DEG (× 180/π)
- `fnml:ConvertKgToG` — unit:KiloGM → unit:GM (× 1000)
- `fnml:ConvertCelsiusToKelvin` — unit:degC → unit:K (+ 273.15, use `qudt:conversionOffset`)

**done:** File exists at `rosetta/policies/fnml_registry.ttl`, parses with rdflib, contains `fnml:ConvertFtToM` triple with `rose:fromUnit unit:FT` and `rose:toUnit unit:M`.

### Task 3 — Create rosetta/core/units.py

Five public functions:

```python
UNIT_STRING_TO_IRI: dict[str, str | None]
# Maps detect_unit() output strings to QUDT unit: IRIs (or None for unmappable units)
# Keys MUST match detect_unit() return values exactly (from unit_detect.py patterns):
#   "meter"       -> "unit:M"
#   "kilometer"   -> "unit:KiloM"
#   "km_per_hour" -> "unit:KiloM-PER-HR"
#   "foot"        -> "unit:FT"
#   "knot"        -> "unit:KN"
#   "degree"      -> "unit:DEG"
#   "dBm"         -> None  (no standard QUDT IRI; emits unit_not_detected INFO, skip compat check)
# NOTE: "metre"/"kilometre" are WRONG — detect_unit() never returns those strings.

def load_qudt_graph() -> Graph:
    """Load curated QUDT units TTL + FnML registry TTL from rosetta/policies/.
    Uses importlib.resources to locate files within the package.
    Both files parsed into the same Graph for unified SPARQL access.
    Returns merged rdflib Graph."""

def dimension_vector(unit_iri: str, qudt_graph: Graph) -> str | None:
    """Look up qudt:hasDimensionVector for a unit IRI string.
    unit_iri may be either:
      - short form prefix:localname e.g. 'unit:M' (from UNIT_STRING_TO_IRI)
      - full IRI e.g. 'http://qudt.org/vocab/unit/M' (from master graph qudt:unit query)
    Both must resolve to the same full IRI before SPARQL query.
    Prefix map: {"unit": "http://qudt.org/vocab/unit/"}.
    Returns the vector string or None if not found."""

def units_compatible(src_iri: str, tgt_iri: str, qudt_graph: Graph) -> bool | None:
    """Compare dimension vectors of two unit IRIs.
    Returns True (compatible), False (incompatible), None (cannot determine — missing data)."""

def suggest_fnml(src_iri: str, tgt_iri: str, qudt_graph: Graph) -> dict | None:
    """SPARQL query the merged policy graph for a conversion function between two unit IRIs.
    qudt_graph is the same merged graph returned by load_qudt_graph() — it contains both
    qudt_units.ttl and fnml_registry.ttl data; there is no separate fnml_graph object.
    Returns {"fnml_function": str, "label": str, "multiplier": float | None, "offset": float | None}
    or None if no conversion registered.
    NOTE: Always None-guard OPTIONAL SPARQL vars before use (qudt:conversionOffset is OPTIONAL)."""
```

Use `importlib.resources` (Python 3.11 `files()` API) to load both policy files — avoids brittle `__file__`-relative paths.

The `UNIT_STRING_TO_IRI` dict must cover all strings that `detect_unit()` can return. Check `unit_detect.py` patterns for the full list.

**done:** `rosetta/core/units.py` exists, all 5 symbols present, `from rosetta.core.units import units_compatible` succeeds.

### Task 4 — Implement rosetta/cli/lint.py

Replace the stub. Full Click CLI:

```
rosetta-lint
  --source     PATH  required  National schema RDF (Turtle)
  --master     PATH  required  Master ontology RDF (Turtle)
  --suggestions PATH required  Suggestions JSON from rosetta-suggest
  --output     PATH  optional  Output file (default: stdout)
  --strict     flag  optional  Treat WARNINGs as BLOCKs
  --config     PATH  optional  rosetta.toml path
```

**Logic:**
1. Load source TTL → rdflib Graph
2. Load master TTL → rdflib Graph
3. Load suggestions JSON
4. Load QUDT graph via `units.load_qudt_graph()`
5. FnML lookup is a SPARQL query called per-pair (lazy per-pair, not lazy file load — both TTL files are loaded eagerly in step 4)
6. For each source field URI in suggestions:
   a. Get top suggestion: `data[src_uri]["suggestions"][0]`; target URI at key `"uri"`. If `data[src_uri]["suggestions"]` is empty, skip (emit no finding).
   b. Query source graph: `rose:detectedUnit` literal → map to QUDT IRI via `UNIT_STRING_TO_IRI`. If IRI is None (e.g. "dBm") or key missing → emit `unit_not_detected` INFO and continue.
   c. Query master graph: `qudt:unit` on master URI → full URIRef. If none → emit `master_unit_missing` INFO and continue.
   d. Call `units_compatible(src_iri, tgt_iri, qudt_graph)`:
      - `False` → emit BLOCK (`unit_dimension_mismatch`), `fnml_suggestion: null`
      - `True`, same IRI → no unit finding (units identical)
      - `True`, different IRI → call `suggest_fnml()` → emit WARNING (`unit_conversion_required`) with fnml_suggestion (may be null if no registry entry)
      - `None` → emit INFO (`unit_vector_missing`)
   e. Query source `rose:dataType`, master `rdfs:range` → emit WARNING (`datatype_mismatch`) on numeric vs string mismatch. Always None-guard OPTIONAL SPARQL vars before string use; coerce URIRef→str before json.dumps().
7. Build output: `{"findings": [...], "summary": {"block": N, "warning": N, "info": N}}`
8. If `--strict`: re-classify all WARNING → BLOCK in the report
9. Exit 1 if any BLOCK in final report, else 0

Finding schema:
```json
{
  "severity": "BLOCK",
  "rule": "unit_dimension_mismatch",
  "source_field": "http://...",
  "target_field": "http://...",
  "source_unit": "unit:FT",
  "target_unit": "unit:KiloGM",
  "message": "Incompatible dimensions: length vs mass",
  "fnml_suggestion": null
}
```

Rules to implement:
- `unit_dimension_mismatch` → BLOCK (incompatible dimension vectors)
- `unit_conversion_required` → WARNING (same dims, different units; includes `fnml_suggestion`, may be null if no registry entry)
- `unit_not_detected` → INFO (source field has no `rose:detectedUnit`, or maps to None in UNIT_STRING_TO_IRI e.g. "dBm")
- `master_unit_missing` → INFO (master attr has no `qudt:unit`)
- `unit_vector_missing` → INFO (both IRIs present but dimension vectors missing from qudt_units.ttl; units_compatible() returned None)
- `datatype_mismatch` → WARNING (e.g. source xsd:integer/float vs master xsd:string or vice versa)

Follow existing CLI patterns from `embed.py` and `suggest.py`: `load_config`, `open_output`, `sys.exit(1)` on error.

**done:** `uv run rosetta-lint --help` shows all options; stub text "Not yet implemented" is gone.
