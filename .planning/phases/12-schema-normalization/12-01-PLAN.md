## Design notes

### Format → importer map
| fmt flag | Extension auto-detect | Importer | Import path |
|----------|-----------------------|----------|-------------|
| `json-schema` | `.json` | `JsonSchemaImportEngine` | `schema_automator.importers.jsonschema_import_engine` |
| `openapi` | `.yaml`/`.yml` (if file contains `openapi:`) | `json_schema_from_open_api()` → tmp → `JsonSchemaImportEngine` | same module |
| `xsd` | `.xsd` | `XsdImportEngine` | `schema_automator.importers.xsd_import_engine` |
| `csv` | `.csv` | `CsvDataGeneralizer(column_separator=",")` | `schema_automator.generalizers.csv_data_generalizer` |
| `tsv` | `.tsv` | `CsvDataGeneralizer(column_separator="\t")` | same |
| `json-sample` | never auto-detected | `genson.SchemaBuilder` → tmp JSON Schema → `JsonSchemaImportEngine` | genson (existing dep) |
| `rdfs` | `.ttl` / `.owl` / `.rdf` | `RdfsImportEngine(format="turtle")` | `schema_automator.importers.rdfs_import_engine` |

Auto-detect for `.yaml`/`.yml`: peek at first 512 bytes; if `openapi:` is present, use `openapi` importer; otherwise treat as unknown format and raise `ValueError`.

### `rosetta-translate` + LinkML
In v1, translation targeted `rdfs:label` triples in an RDF graph. In v2, the equivalent is:
- `ClassDefinition.title` — human-readable class label (translate this; derive from `name.replace("_", " ").title()` if absent)
- `SlotDefinition.title` — human-readable slot label (same treatment)
- `ClassDefinition.description` / `SlotDefinition.description` — free-text (translate if present)
- `ClassDefinition.aliases` / `SlotDefinition.aliases` — preserve original non-English title here (prepend to list)

The `name` field (the schema identifier, e.g. `geschwindigkeit_kmh`) is **never** modified — it is the structural key used by schema references.

### Known broken state after Phase 12
After this phase `rosetta-suggest`, `rosetta-accredit`, and other downstream tools that consume the embed JSON output will be non-functional pending Phase 13 and 14 updates. `rosetta-embed` itself is updated in Phase 12 (Task 7) to accept LinkML YAML. This is accepted as part of the v2 migration.

### `rosetta-embed` context flags
When embedding LinkML YAML schemas, text is built per-node (class or slot) from:

```
base text = title (or name.replace("_"," ").title() if absent)

--include-definitions  → append description if present
--include-parents      → append direct is_a parent title
--include-ancestors    → append all transitive is_a ancestor titles (supersedes --include-parents)
--include-children     → append names of direct is_a children (subclasses one level below)
```

`--include-ancestors` and `--include-children` can be combined (include context both up and down the hierarchy).

Text is joined with `". "`. Node ID format: `"{schema.name}/{node_name}"`.

`SchemaView` from `linkml_runtime.utils.schemaview` is used for ancestor and children traversal. Children are computed as classes where `is_a == node_name`.

### Task 2 — Implement `rosetta/core/normalize.py`

**File:** `rosetta/core/normalize.py`

```python
def normalize_schema(
    input_path: Path,
    fmt: str | None = None,
    schema_name: str | None = None,
) -> SchemaDefinition:
    """Normalise any supported schema format to a LinkML SchemaDefinition.

    fmt: explicit format override. If None, inferred from file extension.
    schema_name: schema identifier. Defaults to input_path.stem.
    """
```

Format detection (when `fmt` is None):
```python
ext = input_path.suffix.lower()
match ext:
    case ".json":   fmt = "json-schema"
    case ".xsd":    fmt = "xsd"
    case ".csv":    fmt = "csv"
    case ".tsv":    fmt = "tsv"
    case ".ttl" | ".owl" | ".rdf":
                    fmt = "rdfs"
    case ".yaml" | ".yml":
        # peek to distinguish OpenAPI from plain YAML
        content = input_path.read_text(encoding="utf-8", errors="replace")[:512]
        fmt = "openapi" if "openapi:" in content else None
    case _:
        fmt = None
if fmt is None:
    raise ValueError(
        f"Cannot infer format from extension {ext!r}. "
        "Use --format {json-schema,openapi,xsd,csv,tsv,json-sample,rdfs}."
    )
```

Importer dispatch (import each engine lazily inside the branch). After each branch, post-assign `name` uniformly — do NOT pass `name=` to individual importer constructors/convert methods:

### Task 4 — Update `rosetta-translate` for LinkML YAML

**Files:** `rosetta/core/translation.py`, `rosetta/cli/translate.py`

`rosetta/core/translation.py` — replace the `rdflib.Graph` translation path with a `SchemaDefinition` path:

```python
def translate_schema(
    schema: SchemaDefinition,
    source_lang: str,
    target_lang: str = "EN",
    deepl_key: str | None = None,
) -> SchemaDefinition:
```

Passthrough: if `source_lang.upper() == target_lang.upper()`, return `schema` unchanged.

For each `ClassDefinition` in `schema.classes.values()`:
1. Derive display label: `label = cls.title or cls.name.replace("_", " ").title()`
2. If `source_lang != "EN"`: call DeepL to translate `label` → `en_label`
3. Set `cls.title = en_label`
4. Prepend original `label` to `cls.aliases` (so `cls.aliases[0]` is the original)
5. If `cls.description`: translate it; set `cls.description = translated`

For each `SlotDefinition` in `schema.slots.values()`:
- Same treatment as classes (title → translate → aliases; description → translate)

DeepL error handling — wrap the `translator.translate_text()` call:
```python
import deepl.exceptions

try:
    results = translator.translate_text(unique_texts, target_lang="EN-US", source_lang=sl)
except deepl.exceptions.AuthorizationException as exc:
    raise RuntimeError(
        f"DeepL authentication failed: {exc}. Check --deepl-key or DEEPL_API_KEY env var."
    ) from exc
except deepl.exceptions.QuotaExceededException as exc:
    raise RuntimeError(f"DeepL quota exceeded: {exc}.") from exc
except deepl.exceptions.DeepLException as exc:
    raise RuntimeError(f"DeepL API error: {exc}.") from exc
```
The `RuntimeError` propagates to the CLI's `except Exception` handler, which prints it and exits 1.

#### `rosetta/core/embedding.py` — delete v1, add `extract_text_inputs_linkml()`

Delete: `extract_text_inputs()`, `_MASTER_SPARQL`, `_NATIONAL_SPARQL`, and the three rdflib/rdf_utils imports at the top of the file.

Add:
```python
from linkml_runtime.linkml_model import SchemaDefinition
from linkml_runtime.utils.schemaview import SchemaView

def extract_text_inputs_linkml(
    schema: SchemaDefinition,
    *,
    include_definitions: bool = False,
    include_parents: bool = False,
    include_ancestors: bool = False,
    include_children: bool = False,
) -> list[tuple[str, str]]:
    """Return (node_id, text) pairs for each class and slot in a LinkML SchemaDefinition.

    node_id format: "{schema.name}/{node_name}"
    text: title (base) + optional definition, parents/ancestors, children — joined with ". "
    --include-ancestors supersedes --include-parents (ancestors is a strict superset).
    """
    schema_name = schema.name or "schema"
    need_view = include_parents or include_ancestors or include_children
    view = SchemaView(schema) if need_view else None
    results: list[tuple[str, str]] = []

    for node_name, node in {**schema.classes, **schema.slots}.items():
        label = node.title or node_name.replace("_", " ").title()
        parts: list[str] = [label]

        if include_definitions and node.description:
            parts.append(node.description)

        if include_ancestors and view:
            # class_ancestors returns [self, parent, grandparent, ...] — skip self (index 0)
            try:
                ancs = view.class_ancestors(node_name)[1:]
            except Exception:
                ancs = []
            for anc in ancs:
                anc_node = schema.classes.get(anc) or schema.slots.get(anc)
                anc_label = (anc_node.title if anc_node else None) or anc.replace("_", " ").title()
                parts.append(anc_label)
        elif include_parents:
            parent = getattr(node, "is_a", None)
            if parent:
                parent_node = schema.classes.get(parent) or schema.slots.get(parent)
                parent_label = (
                    (parent_node.title if parent_node else None)
                    or parent.replace("_", " ").title()
                )
                parts.append(parent_label)

        if include_children:
            children = [
                n for n, c in schema.classes.items()
                if getattr(c, "is_a", None) == node_name
            ]
            parts.extend(ch.replace("_", " ").title() for ch in children)

        results.append((f"{schema_name}/{node_name}", ". ".join(parts)))

    return results
```
