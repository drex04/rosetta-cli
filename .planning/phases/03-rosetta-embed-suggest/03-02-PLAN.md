## Must-haves

### Truths

1. `rosetta-suggest --source nor.json --master master.json` exits 0 and writes valid JSON with one entry per source URI, each containing `{"suggestions": [...], "anomaly": bool}`.
2. Suggestions within each entry are sorted by descending cosine score; `rank` is 1-based (rank 1 = highest score).
3. `--top-k N` limits suggestions per source field; `--min-score F` excludes any suggestion below the threshold. Both default from `rosetta.toml [suggest]` with CLI flags overriding (3-tier precedence via `get_config_value`).
4. A source field whose maximum cosine similarity across all master attributes is below `anomaly_threshold` (default 0.3, configurable via `rosetta.toml [suggest]` and `--anomaly-threshold`) gets `"anomaly": true`. **Anomaly is computed from raw scores before `min_score` filtering** — a field with a real match that was filtered by a high `min_score` is not anomalous.
5. All unit tests pass without any model download — similarity functions are tested with synthetic numpy arrays; CLI tests use pre-baked embedding JSON fixtures.
6. Any exception from file loading, matrix computation, or I/O produces a clean stderr message and exits with code 1 (no Python traceback visible to user).
7. `[review]` `cosine_matrix(A, B)` raises `ValueError(f"Embedding dimension mismatch: source={A.shape[1]}, master={B.shape[1]}")` when the two arrays have different vector dimensions — no cryptic numpy traceback reaches the user.
8. `[review]` If any URI in the source or master JSON is missing the `"lexical"` key, the CLI raises `ValueError(f"Missing 'lexical' key for URI: {uri}")` before attempting matrix computation — caught by the top-level try/except and printed to stderr.
9. `[review]` Empty source and master are diagnosed separately: `"No embeddings found in source file: {path}"` vs `"No embeddings found in master file: {path}"`.
10. `[review]` When `top_k > len(master)`, all master entries are returned sorted by score — no crash, no silent truncation.

## Tasks

### Task 1 — `rosetta/core/similarity.py`

Create the similarity library.

**File:** `rosetta/core/similarity.py`

```python
import numpy as np

def cosine_matrix(A: np.ndarray, B: np.ndarray) -> np.ndarray:
    """Return pairwise cosine similarity matrix, shape (len(A), len(B)).

    Uses (A @ B.T) / (||A|| * ||B||.T) — pure numpy, no scipy.
    Zero-norm rows are handled by clipping denominator to avoid division by zero.
    Raises ValueError if A and B have different vector dimensions.
    """

def rank_suggestions(
    src_uris: list[str],
    A: np.ndarray,            # shape (n_src, dim)
    master_uris: list[str],
    B: np.ndarray,            # shape (n_master, dim)
    top_k: int = 5,
    min_score: float = 0.0,
    anomaly_threshold: float = 0.3,
) -> dict:
    """Compute ranked suggestions for each source URI against all master URIs.

    Returns:
        {
            "<source_uri>": {
                "suggestions": [{"uri": str, "score": float, "rank": int}, ...],
                "anomaly": bool
            }
        }
    Scores rounded to 6 decimal places. Rank is 1-based.
    Anomaly computed from raw scores BEFORE min_score filtering.
    If top_k > len(master), returns all master entries sorted by score.
    """
```

**Implementation notes:**
- Dimension guard (first): `if A.shape[1] != B.shape[1]: raise ValueError(f"Embedding dimension mismatch: source={A.shape[1]}, master={B.shape[1]}")`
- Normalise rows: `norms = np.linalg.norm(A, axis=1, keepdims=True); A_norm = A / np.clip(norms, 1e-10, None)`
- Similarity matrix: `sim = A_norm @ B_norm.T` — shape `(n_source, n_master)`
- Per source row: compute `max_sim = float(sim_row.max())` → `anomaly = max_sim < anomaly_threshold`
- Then filter by `min_score`, take `min(top_k, len(master_uris))` results, assign 1-based ranks

**`done`:** `from rosetta.core.similarity import cosine_matrix, rank_suggestions` works; both functions return correct types.

### Task 2 — `rosetta/cli/suggest.py`

Create the CLI entrypoint.

**File:** `rosetta/cli/suggest.py`

```python
@click.command()
@click.option("--source", required=True, type=click.Path(exists=True), help="Source embeddings JSON")
@click.option("--master", required=True, type=click.Path(exists=True), help="Master embeddings JSON")
@click.option("--top-k", default=None, type=int, help="Max suggestions per field")
@click.option("--min-score", default=None, type=float, help="Minimum cosine score")
@click.option("--anomaly-threshold", default=None, type=float, help="Anomaly flag threshold")
@click.option("--output", default=None, type=click.Path(), help="Output file (default: stdout)")
@click.option("--config", default="rosetta.toml", show_default=True)
def cli(source, master, top_k, min_score, anomaly_threshold, output, config):
    ...
```

**Implementation notes:**
- Load config: `cfg = load_config(config)` — tolerate missing file (same as embed.py)
- Resolve defaults via `get_config_value(cfg, "suggest", "top_k")` etc., falling back to 5 / 0.0 / 0.3
- Load source and master JSON: `json.loads(Path(source).read_text())`
- Validate non-empty: `if not src_emb: click.echo(f"No embeddings found in source file: {source}", err=True); sys.exit(1)` (same for master)
- Validate and extract arrays — **the CLI owns this conversion**:
  ```python
  for uri, val in src_emb.items():
      if "lexical" not in val:
          raise ValueError(f"Missing 'lexical' key for URI: {uri}")
  src_uris = list(src_emb)
  A = np.array([src_emb[u]["lexical"] for u in src_uris], dtype=np.float32)
  # same for master_uris / B
  ```
- Call `rank_suggestions(src_uris, A, master_uris, B, top_k, min_score, anomaly_threshold)`
- Write result with `open_output(output)` and `json.dump(..., indent=2)`
- Wrap full body in `try/except Exception as e: click.echo(f"Error: {e}", err=True); sys.exit(1)`

**`done`:** `rosetta-suggest --source a.json --master b.json --output /dev/null` exits 0.
