5. Output parent directories are created automatically if they do not exist.
6. All unit tests pass without downloading the LaBSE model (`SentenceTransformer` is mocked).
7. `[review]` Any exception raised during graph loading, model loading, encoding, or file I/O produces a clean error message on stderr and exits with code 1 (no Python traceback visible to the user).
8. `[review]` OPTIONAL SPARQL fields (`conceptLabel`, `comment`) that are absent from the result produce an empty string `""` in the text template — never the literal string `"None"`.
9. `[review]` JSON output keys are plain strings (not `URIRef` objects) — `json.dumps()` never raises `TypeError`.

### Task 2: Implement `rosetta/core/embedding.py`

Two public exports: `extract_text_inputs(g)` and `EmbeddingModel`.

#### `extract_text_inputs(g: Graph) -> list[tuple[str, str]]`

Returns a list of `(uri_str, text_str)` pairs — one per embeddable attribute in the graph.

**Detection logic:**
- If any triple `(_, RDF.type, ROSE_NS.Attribute)` exists → master ontology path
- Otherwise → national schema path (all `rose:Field` nodes)

**Master path (SPARQL):**
```sparql
PREFIX rose: <http://rosetta.interop/ns/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?attr ?attrLabel ?comment ?conceptLabel WHERE {
  ?attr a rose:Attribute ;
        rdfs:label ?attrLabel .
  OPTIONAL { ?attr rdfs:comment ?comment . }
  OPTIONAL { ?concept rose:hasAttribute ?attr ;
                      rdfs:label ?conceptLabel . }
}
```
Text string: `f"{concept_label} / {attr_label} — {comment}"` (empty string for missing concept or comment).

**IMPORTANT — None guard required:** `query_graph` returns Python `None` for unbound OPTIONAL variables. Coerce before formatting:
```python
concept_label = str(row["conceptLabel"]) if row.get("conceptLabel") is not None else ""
comment       = str(row["comment"])       if row.get("comment")       is not None else ""
attr_label    = str(row["attrLabel"])
```
Without this, missing OPTIONAL fields render as the literal string `"None"` in the text, silently corrupting embedding inputs.

**National schema path (SPARQL):**
```sparql
PREFIX rose: <http://rosetta.interop/ns/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?field ?label WHERE {
  ?field a rose:Field ;
         rdfs:label ?label .
}
```
Parent from URI: split on `/`, take the 3rd path segment after the domain (index -2, e.g. `nor_radar` from `.../field/NOR/nor_radar/hoyde_m` — path segments are `field`/`NOR`/`nor_radar`/`hoyde_m`, so index -2 is correct; do NOT use index 4 which gives the nation code).
Text string: `f"{schema_slug} / {label} — "` (no description on national fields in lexical-only mode).

**Return type:** `list[tuple[str, str]]` — `(str(uri), text_string)`.

#### `EmbeddingModel`

```python
class EmbeddingModel:
    def __init__(self, model_name: str = "sentence-transformers/LaBSE") -> None:
        from sentence_transformers import SentenceTransformer
        self._model = SentenceTransformer(model_name)

    def encode(self, texts: list[str]) -> list[list[float]]:
        """Encode texts; return as list of Python float lists (JSON-serializable)."""
        vectors = self._model.encode(texts)  # numpy array shape (n, dim)
        return [v.tolist() for v in vectors]
```

Lazy import of `sentence_transformers` inside `__init__` to keep the module importable even if the package is not installed (consistent with Phase 2 lazy import pattern).

**Done when:** `from rosetta.core.embedding import extract_text_inputs, EmbeddingModel` imports cleanly.

### Task 3: Implement `rosetta/cli/embed.py`

Replace the stub with a full implementation.

```
rosetta-embed
  --input  / -i   PATH (default: -)    Turtle input file
  --output / -o   PATH (default: -)    JSON output file  
  --mode          TEXT (default: from config or "lexical-only")
  --model         TEXT (default: from config or "sentence-transformers/LaBSE")
  --config / -c   PATH (default: None)
```

**Implementation outline:**
```python
def cli(input_path, output_path, mode, model, config):
    cfg = load_config(config)
    resolved_model = get_config_value(cfg, "embed", "model", cli_value=model) or "sentence-transformers/LaBSE"
    resolved_mode  = get_config_value(cfg, "embed", "mode",  cli_value=mode)  or "lexical-only"

    try:
        with open_input(input_path) as src:
            g = load_graph(src)

        pairs = extract_text_inputs(g)
        if not pairs:
            click.echo("No embeddable attributes found in input.", err=True)
            sys.exit(1)

        em = EmbeddingModel(resolved_model)
        uris, texts = zip(*pairs)
        vectors = em.encode(list(texts))

        # URIRef → str so json.dumps() doesn't raise TypeError
        result = {str(uri): {"lexical": vec} for uri, vec in zip(uris, vectors)}

        # Auto-create output parent directory, then use open_output for both file and stdout
        if output_path != "-":
            Path(output_path).parent.mkdir(parents=True, exist_ok=True)
        with open_output(output_path) as fh:
            fh.write(json.dumps(result, indent=2))
    except Exception as e:
        click.echo(str(e), err=True)
        sys.exit(1)
```

Exit code 1 on any exception — mirrors `ingest.py:31-40` exactly. `open_output` handles `"-"` → stdout uniformly.

**Done when:** `uv run rosetta-embed --help` shows all options; `uv run rosetta-embed --input store/master-ontology/master.ttl --output /tmp/master.json` exits 0 (may be slow; that's OK for manual test).
