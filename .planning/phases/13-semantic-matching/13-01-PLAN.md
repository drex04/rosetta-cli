# Plan 13-01: linkml upgrade + SSSOM output

## Context

Phase 12 complete (166/166 tests). Phase 13 begins.

`linkml` 1.9.3 is installed but `linkml-runtime` 1.10.0 dropped `Format.JSON`, causing
`linkml` and `sssom` to crash on import. Upgrading `linkml` to 1.10.0 (latest, matching
linkml-runtime) unblocks `sssom` usage.

`rosetta-suggest` currently outputs proprietary JSON (`SuggestionReport`). This plan
migrates the output to SSSOM TSV â€” the standard semantic mapping exchange format â€”
to align with the v2 milestone.

## Key decisions

- SSSOM TSV only output (no JSON fallback) â€” clean v2 break
- `predicate_id: skos:relatedMatch` + `mapping_justification: semapv:LexicalMatching` for all auto-generated candidates (Phase 14 handles predicate refinement via human review)
- `anomaly` field dropped â€” not a SSSOM concept; covered by absence of suggestions
- `--ledger` renamed to `--approved-mappings`; reads `.sssom.tsv` instead of `ledger.json`
- Boost logic: `(subject_id, object_id)` pair found in approved SSSOM â†’ apply boost; row missing â†’ no change
- pyparsing save/restore in `normalize_schema()` stays (separate issue from Format.JSON patch)

### Task 3 â€” Add `apply_sssom_feedback`; clean up anomaly + old models; add embed label

**Files:** `rosetta/core/similarity.py`, `rosetta/core/models.py`, `rosetta/cli/embed.py`

#### 3a â€” apply_sssom_feedback (dual boost + penalty)

Replace `apply_ledger_feedback` with `apply_sssom_feedback`:

```python
def apply_sssom_feedback(
    subject_id: str,
    candidates: list[dict[str, Any]],
    approved_rows: list[SSSOMRow],
    boost: float = 0.1,
    penalty: float = 0.2,
) -> list[dict[str, Any]]:
    """Boost or derank candidates based on approved SSSOM rows.

    - Row predicate_id == owl:differentFrom â†’ subtract penalty (floor 0.0), row NOT removed.
    - Any other predicate match on (subject_id, object_id) â†’ add boost (cap 1.0).
    - If ANY differentFrom row exists for subject_id â†’ apply penalty * 0.25 to all
      OTHER candidates for that field (soft subject-breadth deranking).
    Returns a new list (does not mutate input).
    """
```

Remove `apply_ledger_feedback`. It is safe to remove: only `suggest.py` (being rewritten)
and `test_accredit_integration.py` (being rewritten) import it. `units.py` only imports
`FnmlSuggestion` â€” not affected.

#### 3d â€” Add label to rosetta-embed output

```python
def _label_from_uri(uri: str) -> str:
    """Last segment after # or /, empty string if none."""
    for sep in ("#", "/"):
        if sep in uri:
            return uri.rsplit(sep, 1)[-1]
    return ""

report = EmbeddingReport(
    root={
        node_id: EmbeddingVectors(lexical=vec, label=_label_from_uri(node_id))
        for (node_id, _), vec in zip(pairs, vectors, strict=True)
    }
)
```

### Task 4 â€” Rewrite `rosetta-suggest` output to SSSOM TSV

**Files:** `rosetta/cli/suggest.py`

Replace the JSON output block with SSSOM TSV output using the `sssom` library.

CLI interface changes:
- `--ledger` â†’ `--approved-mappings` (path to a `.sssom.tsv` file; optional)
- `--output` default: stdout (same as now)
- Remove `--anomaly-threshold` (anomaly concept dropped with JSON output)

SSSOM TSV output format:
```
# mapping_set_id: https://rosetta-cli/mappings
# mapping_tool: rosetta-suggest
# license: https://creativecommons.org/licenses/by/4.0/
subject_id	predicate_id	object_id	mapping_justification	confidence	subject_label	object_label
```

Each row = one `(source_field, master_candidate)` pair, ordered by descending confidence.
`subject_label` and `object_label` are read from the `"label"` key in the embedding JSON
(written by `rosetta-embed` in Task 3d); fall back to `""` if key absent.

Use `sssom` library for serialisation. Preferred path:
```python
from sssom.util import MappingSetDataFrame
from sssom.writers import write_table  # or sssom.io.write_table
```

If the sssom API does not expose a clean write path, fall back to writing TSV manually.
The hand-rolled header **must** include the curie_map block:
```
# mapping_set_id: https://rosetta-cli/mappings
# mapping_tool: rosetta-suggest
# license: https://creativecommons.org/licenses/by/4.0/
# curie_map:
#   skos: http://www.w3.org/2004/02/skos/core#
#   semapv: https://w3id.org/semapv/vocab/
```
Do NOT block on sssom API issues â€” ship the feature.

If `--approved-mappings` is provided but the file does not exist, exit 1 with the path in
the error message (same pattern as missing source/master files).

**Done:** `uv run rosetta-suggest src.embed.json master.embed.json` prints SSSOM TSV to stdout;
first line starts with `#`; header contains `curie_map` block; column header contains `subject_id`;
`subject_label` and `object_label` columns are populated from embed label field.
