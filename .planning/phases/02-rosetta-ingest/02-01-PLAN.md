**`rosetta/core/unit_detect.py`:**
1. Implement `detect_unit(name: str, description: str) -> str | None` with **end-anchored** regex patterns from CONTEXT.md. Apply to field name first, then description. Key anchoring rule: use `r"(?<![a-z])m(?:_|$)"` for meter to avoid false-positives on `_kmh`, `_max`, etc. Safer: match each suffix pattern as a full token, e.g. `r"(?:^|_)m(?:_|$)"`.

### Task 3 — CSV parser
**File:** `rosetta/core/parsers/csv_parser.py`

```python
def parse_csv(src: TextIO, nation: str, slug: str) -> list[FieldSchema]:
```

**Logic:**
1. Read rows with `csv.DictReader`, limited to `max_sample_rows` rows via `itertools.islice(reader, max_sample_rows)` (default 1000; passed in from CLI).
2. First row determines field names. Remaining rows are sample data.
3. For each field: collect all values → infer `data_type` by attempting float parse (number/integer if all non-empty values convert, integer if all are whole numbers).
4. Compute `sample_values` (raw strings from sampled rows for that column).
5. Call `detect_unit(name, "")` for each field.
6. Call `compute_stats(sample_values)` → populate `numeric_stats`/`categorical_stats`.
7. Slug = `path.stem`. Return `(list[FieldSchema], slug)`.

**Edge cases:** empty cells → exclude from type inference and stats; all-empty column → `data_type = "string"`.

**Done when:** `parse_csv` on `nor_radar.csv` returns `(11 FieldSchema objects, "nor_radar")`; `hoyde_m` has `data_type="number"`, `detected_unit="meter"`, `sample_values` length 7, `numeric_stats` populated.

### Task 4 — JSON Schema parser
**File:** `rosetta/core/parsers/json_schema_parser.py`

```python
def parse_json_schema(src: TextIO, nation: str, slug: str) -> list[FieldSchema]:
```

**Logic:**
1. `json.load(src)` → parse document. If `"properties"` key is missing: `raise ValueError(f"JSON Schema missing 'properties' key in {slug!r}")`.
2. Derive slug: try `schema["title"]`, then last segment of `schema["$id"]`; if both absent, fall back to `path.stem` and emit `click.echo("Warning: no title or $id in JSON Schema; using filename as slug", err=True)`.
3. For each property: extract `type`, `description`; check if in `required` list.
4. Extract sample values from top-level `examples` array only (list of dicts) — collect per-field values across all example objects. **Do not use per-property `example`/`examples` keys.**
5. Call `detect_unit(name, description)`.
6. Call `compute_stats(sample_values)` → populate `numeric_stats`/`categorical_stats`.
7. Return `(list[FieldSchema], slug)`.

**Done when:** `parse_json_schema` on `deu_patriot.json` returns `(9 FieldSchema objects, "deu_patriot")`; `Hoehe_Meter` has `data_type="number"`, `detected_unit="meter"`, 3 sample values, `numeric_stats` populated.

### Task 5 — OpenAPI parser with internal `$ref` resolution
**File:** `rosetta/core/parsers/openapi_parser.py`

```python
def parse_openapi(src: TextIO, nation: str, slug: str) -> list[FieldSchema]:
```

**Logic:**
1. `yaml.safe_load(src)` → parse document.
2. Walk `components.schemas` — collect all schema objects (resolving internal `$ref: "#/components/schemas/X"` by lookup in the same dict).
3. Raise `ValueError("External $ref not supported: {ref}")` for any `$ref` not starting with `#`.
4. Slug: slugify `info.title`.
5. For each resolved schema's `properties`: extract `type`, `description`, `required`.
6. Collect sample values from `components.schemas.<name>.examples` (schema-level list of objects) **only** — do NOT use per-property `example` (singular) keys.
7. Multi-schema scoping: if `components.schemas` has multiple top-level schema objects, all properties are merged into a flat list. If the same field name appears in two schemas, last-schema-wins. Document this as a known limitation (Phase 2 scope).
8. Call `detect_unit(name, description)`.
9. Call `compute_stats(sample_values)` → populate `numeric_stats`/`categorical_stats`.
10. Return `(list[FieldSchema], slug)`.

**Done when:** `parse_openapi` on `usa_c2.yaml` returns `(9 FieldSchema objects, "usa_c2")`; `altitude_ft` has `detected_unit="foot"`, 3 sample values, `numeric_stats` populated; `$ref` to `AirTrack` resolved correctly.

### Task 6 — RDF Turtle emitter
**File:** `rosetta/core/ingest_rdf.py`

```python
def fields_to_graph(fields: list[FieldSchema], nation: str, slug: str) -> Graph:
```

**Imports:** `from rosetta.core.rdf_utils import ROSE_NS as ROSE, bind_namespaces, save_graph` and `from rdflib.namespace import RDF, RDFS, XSD`.

**Logic (pure RDF serializer — no stats computation here):**
1. Create `Graph()`, call `bind_namespaces()`.
2. Bind `xsd:` prefix: `g.bind("xsd", XSD)` (ensures typed literals render with prefix in Turtle output).
3. Bind per-schema prefix: `g.bind("f", Namespace(f"http://rosetta.interop/field/{nation}/{slug}/"))`.
4. For each `FieldSchema`:
   - Add `(field_uri, RDF.type, ROSE.Field)`
   - Add `(field_uri, RDFS.label, Literal(field.name))`
   - Add `(field_uri, ROSE.dataType, Literal(field.data_type))`
   - If `detected_unit`: add `(field_uri, ROSE.detectedUnit, Literal(field.detected_unit))`
   - If `numeric_stats` is not None: add blank node with `rose:count` (xsd:integer), `rose:min` (xsd:double), `rose:max` (xsd:double), `rose:mean` (xsd:double)
   - If `categorical_stats` is not None: add blank node with `rose:count` (xsd:integer), `rose:distinctCount` (xsd:integer)
   - If stats blank node added: `(field_uri, ROSE.stats, stats_bnode)`
5. Return graph.

**Done when:** `fields_to_graph` on NOR CSV fields produces a graph that parses as valid Turtle; `query_graph` returns the correct stats for `hoyde_m`; a `FieldSchema` with no `sample_values` produces no `rose:stats` arc.
