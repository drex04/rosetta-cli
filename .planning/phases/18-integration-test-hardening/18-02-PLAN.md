# Plan 18-02: Positive-Path Pipeline Coverage

## Goal

Author realistic full-chain integration tests that exercise every CLI tool with diverse input formats — deeply nested JSON Schema, complex XSD, CSV edge cases, LinkML with inheritance. Every tool that doesn't already have integration coverage gets at least one chained test. Include four mocked `rosetta-translate` tests and two subprocess smoke tests for installed entry points.

## Must-haves

### Truths

1. A developer can run `uv run pytest rosetta/tests/integration/` and see at least 15 green integration tests covering: ingest, embed, suggest, lint, validate, provenance, translate (DE→EN + FR→EN + batch-verify + mixed-lang), accredit (ingest→approve→status + revoke lifecycle + empty-log status) [review], plus two full-pipeline chains (JSON→embed→suggest→lint and XSD→yarrrml-gen→JSON-LD). (The two subprocess smoke tests live under `rosetta/tests/smoke/` not `integration/`.)
2. A deeply nested JSON Schema (≥4 levels, arrays-of-objects, `oneOf`, `$ref`, `additionalProperties:false`) round-trips through `rosetta-ingest` without loss of field names or types — asserted via the emitted LinkML YAML.
3. A complex XSD (with `<xs:choice>`, attributes, nested namespaces, `<xs:complexType>` inheritance via `extension`) produces a non-empty LinkML schema. If `schema-automator`'s XSD importer preserves `xs:extension` as `is_a`, assert `RadarTrack.is_a == "TrackBase"`; otherwise document the observed flattening in the test docstring. Class count and slot count lower bounds asserted, not exact matches.
4. The CSV edge-case fixture (quoted fields with embedded commas, embedded newlines, UTF-8 BOM, mixed numeric/string columns) is accepted by `rosetta-ingest --format csv` and produces a LinkML schema with correct slot count and `range` types.
5. A LinkML schema using `is_a` + `mixins` + `slot_usage` is accepted by `rosetta-suggest` as the master schema and produces a SSSOM TSV where inherited slots appear in the suggestions.
6. The cross-tool pipeline `ingest → embed → suggest → lint` completes on the deeply nested JSON fixture with exit code 0 at every stage; final `LintReport.summary.block == 0` (per `rosetta/core/models.py::LintSummary` — fields are `block`, `warning`, `info`; there is no `errors` field).
7. Four mocked `rosetta-translate` tests pass: DE→EN happy path, FR→EN happy path, batch efficiency (50 titles → 1 API call), mixed-language (pre-translated English titles preserved). Zero DeepL credits consumed.
8. Two subprocess smoke tests (`rosetta-ingest --help`, `rosetta-yarrrml-gen --help`) exit 0 and print their expected option summaries — marked `@slow @e2e`.
9. [review] `rosetta-accredit` has ≥3 positive-path integration tests covering `ingest → approve → status`, `ingest → approve → revoke` lifecycle, and `status` on an empty log.
10. [review] `test_full_chain_json_to_lint` is marked `@pytest.mark.integration @pytest.mark.slow` because `rosetta-embed` runs in the chain; LaBSE is mocked via the same pattern as `rosetta/tests/test_embed.py`.
11. [review] No test asserts `LintReport.summary.errors` (the field doesn't exist). All lint-severity assertions use `summary.block`, `summary.warning`, or `summary.info`.
12. [review] All lint rule-code assertions reference real codes: `datatype_mismatch` (`cli/lint.py:212`) and `unit_dimension_mismatch` (`cli/lint.py:171`). No test asserts `unit_incompatible` or `missing_required` — those rule codes do not exist.
13. [review] Exception types explicitly referenced by each adversarial-error-path assertion are enumerated in each test's docstring (e.g. "raises `deepl.exceptions.AuthorizationException`").

## Tasks

### Task 1 — Author stress fixtures

**Files to create under `rosetta/tests/fixtures/stress/`:**

1. **`nested_json_schema.json`** — JSON Schema draft-07 with:
   - Root object with required fields
   - Nested object 4 levels deep via object-valued properties
   - Array of objects (`"items": {...}`)
   - `oneOf` discriminated union (e.g. `TrackKind` = `radar | optical | emitter`)
   - `$ref` to `#/definitions/...` for a reused type
   - `additionalProperties: false` at two levels

   Shape target: ~12-15 leaf slots after normalization.

2. **`nested_json_sample.json`** — a single instance document conforming to the above, usable as the `--data` input for `yarrrml-gen --run`.

3. **`complex_types.xsd`** — XSD 1.0 with:
   - A base `<xs:complexType name="TrackBase">` with sequence of elements and two attributes
   - A derived `<xs:complexType name="RadarTrack">` using `<xs:extension base="TrackBase">`
   - An `<xs:choice>` element (e.g. Emitter | Receiver)
   - Two namespaces: target + an imported one (`tns` + `common`)
   - At least 3 `<xs:attribute>` declarations including one `use="required"`

4. **`complex_types_sample.xml`** — instance document using RadarTrack with populated choice branch and attributes.

5. **`csv_edge_cases.csv`** — 5-row CSV with:
   - UTF-8 BOM at file start (`\ufeff`)
   - Column with quoted field containing a comma: `"Oslo, Norway"`
   - Column with quoted field containing a newline: `"line1\nline2"`
   - A numeric column with one blank cell
   - A column header with a space: `"radar type"`

6. **`linkml_inheritance.linkml.yaml`** — LinkML schema with:
   - `TrackBase` class with 3 slots
   - `RadarTrack` class using `is_a: TrackBase` + its own 2 slots + 1 mixin
   - `Identifiable` mixin with an `id` slot
   - `slot_usage` on `RadarTrack` narrowing the range of one inherited slot
   - `@` prefix declaration matching a `default_prefix`

**Done when:** All 6 files exist. Manual smoke: `uv run rosetta-ingest --input rosetta/tests/fixtures/stress/nested_json_schema.json --format json-schema --output /tmp/x.linkml.yaml` exits 0 and produces a non-empty YAML. Same for XSD and CSV inputs.

### Task 5 — Author subprocess smoke tests

**File:** `rosetta/tests/smoke/test_entry_points.py`

```python
"""Subprocess smoke tests — verify installed console scripts resolve."""
from __future__ import annotations

import shutil
import subprocess

import pytest

pytestmark = [pytest.mark.integration, pytest.mark.slow, pytest.mark.e2e]


def _skip_if_not_installed(name: str) -> None:
    if shutil.which(name) is None:
        pytest.skip(f"{name} not installed — run `uv sync` first")


def test_rosetta_ingest_entry_point() -> None:
    _skip_if_not_installed("rosetta-ingest")
    result = subprocess.run(
        ["rosetta-ingest", "--help"],
        capture_output=True,
        text=True,
        check=False,
        timeout=30,
    )
    assert result.returncode == 0
    assert "Normalise a schema file" in result.stdout
    assert "--format" in result.stdout


def test_rosetta_yarrrml_gen_entry_point() -> None:
    _skip_if_not_installed("rosetta-yarrrml-gen")
    result = subprocess.run(
        ["rosetta-yarrrml-gen", "--help"],
        capture_output=True,
        text=True,
        check=False,
        timeout=30,
    )
    assert result.returncode == 0
    assert "--sssom" in result.stdout
    assert "--run" in result.stdout
```

Also create `rosetta/tests/smoke/__init__.py` (empty).

**Done when:** Both tests pass locally. Verify they actually run the installed script (not a mock) by temporarily breaking the entry point in `pyproject.toml` and confirming failure.

## Risks & mitigations

- **Risk:** `schema-automator`'s XSD importer may not support all complex-type features we design in `complex_types.xsd`. **Mitigation:** First manually run ingest on the candidate XSD *before* writing assertions; if a feature is rejected, simplify the fixture to stay within the importer's support envelope and note the limitation in a docstring.
- **Risk:** The CSV BOM handling depends on Python's `csv` + `open(..., encoding="utf-8-sig")`. **Mitigation:** Check `rosetta/core/parsers/csv.py` (or equivalent) before authoring the fixture; if BOM is not stripped, either add a `utf-8-sig` change in production code (out of scope here — file as a 18-03 adversarial test instead) or drop the BOM case from the positive fixture.
- **Risk:** Full-chain tests become brittle if model downloads (LaBSE via sentence-transformers) aren't cached in CI. **Mitigation:** Check the existing `test_embed.py` approach for model mocking; apply the same approach if already in use, otherwise add a conditional skip if the model isn't preloaded.
