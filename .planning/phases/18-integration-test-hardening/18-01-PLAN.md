# Plan 18-01: Test Infrastructure Foundation

## Goal

Land the plumbing that every subsequent integration/e2e test will rely on: pytest markers, shared fixture paths in `conftest.py`, a reusable DeepL mock, subdirectory reorganization under `fixtures/`, retag of the three existing integration test files, CI wiring, and README update. No new tests are authored in this plan — only infrastructure.

## Must-haves

### Truths
1. `uv run pytest -m integration` selects exactly the three existing integration/e2e files (`test_accredit_integration.py`, `test_yarrrml_compile_integration.py`, `test_yarrrml_run_e2e.py`); no warning about unknown markers is emitted.
2. `uv run pytest -m e2e` selects exactly `test_yarrrml_run_e2e.py`.
3. Every existing integration test file imports fixture paths from `conftest.py` fixtures (e.g. `def test_x(nor_csv_path): ...`); no file contains a hardcoded `rosetta/tests/fixtures/...` literal.
4. In the three migrated integration files (`test_accredit_integration.py`, `test_yarrrml_compile_integration.py`, `test_yarrrml_run_e2e.py`), `grep -n '_FIXTURES = Path'` returns zero hits. (Note: `test_yarrrml_gen.py` is a unit test and retains its own `_FIXTURES` constant — out of scope here.)
5. CI (`.github/workflows/ci.yml`) runs the full pytest suite on every push (no `-m "not slow"` selector on the default job); a `fast-gate` job runs `-m "not slow and not e2e"` for PR feedback.
6. `README.md` "Running tests" section explains `integration`/`e2e`/`slow` marker semantics with at least two selection examples.
7. `uv run pytest` (no filter) passes — all existing tests still green after reorganization.
8. [review] Tasks 2, 3, 4, 5 are committed together as one atomic PR. No intermediate commit leaves the repo with fixtures relocated but call sites still referencing flat paths.
9. [review] The `fake_deepl` smoke test (Task 7) declares only `fake_deepl` in its signature — `monkeypatch` is not redundantly declared; the fixture owns the patch.
10. [review] Truth #4 in this plan is scoped to the snapshot at 18-01 completion; after 18-02 and 18-03 land, additional `integration`-marked tests are expected and do not violate this truth.

### Task 4 — Add reusable `fake_deepl` fixture

**File:** `rosetta/tests/conftest.py` (same file as Task 3)

Append:

```python
@pytest.fixture
def fake_deepl(monkeypatch: pytest.MonkeyPatch):
    """
    Install a fake DeepL translator. Returns a configurator callable.

    Usage in tests:
        def test_x(fake_deepl):
            fake_deepl({"Geschwindigkeit": "Speed"})       # happy path
            # or
            import deepl
            fake_deepl(raises=deepl.exceptions.QuotaExceededException("mock quota"))

    Tracks call count and last-received text batch on the fixture object itself:
        fake_deepl.call_count, fake_deepl.last_batch
    """
    from typing import Any

    state: dict[str, Any] = {"mapping": {}, "raises": None, "call_count": 0, "last_batch": None}

    class _FakeResult:
        def __init__(self, text: str) -> None:
            self.text = text

    def _fake_translate_text(self: Any, texts: list[str], **kwargs: Any) -> list[_FakeResult]:
        state["call_count"] += 1
        state["last_batch"] = list(texts)
        if state["raises"] is not None:
            raise state["raises"]
        mapping = state["mapping"]
        return [_FakeResult(mapping.get(t, t)) for t in texts]

    monkeypatch.setattr("deepl.Translator.translate_text", _fake_translate_text)

    def configure(mapping: dict[str, str] | None = None, *, raises: Exception | None = None):
        state["mapping"] = mapping or {}
        state["raises"] = raises
        return state  # returns a handle for introspection

    configure.state = state  # type: ignore[attr-defined]
    return configure
```
