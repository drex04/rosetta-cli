# Plan: Lint Structural Reachability Check

## Context

`rosetta-yarrrml-gen` fails at step 9 with "slots mapped to target class 'Entity' but no class-level mapping" when a user approves a slot mapping whose owning class isn't reachable from the approved class mapping. This error should surface earlier — at `rosetta-lint` (step 5) — so analysts get immediate feedback.

## Tasks

### Task 1: Extract shared utilities into `rosetta/core/schema_utils.py`

Create `rosetta/core/schema_utils.py` with these functions extracted from `transform_builder.py`:

```python
def build_slot_owner_index(view: SchemaView) -> dict[str, str]
def local_name(curie: str) -> str
def ancestors(class_name: str, view: SchemaView) -> set[str]
def nearest_mapped_ancestor(class_name: str, mapped_classes: set[str], view: SchemaView) -> str | None
```

All retain their current logic unchanged. Make them public (drop leading underscore).

**Done:** Functions exist in schema_utils.py with full type annotations. Import statement works.

### Task 2: Refactor `transform_builder.py` to use shared utils

Replace the local `_build_slot_owner_index`, `_local_name`, `_ancestors`, `_nearest_mapped_ancestor` definitions with imports from `schema_utils`:

```python
from rosetta.core.schema_utils import (
    build_slot_owner_index,
    local_name,
    ancestors,
    nearest_mapped_ancestor,
)
```

Update all internal call sites (drop underscore prefix where referenced). Keep private wrappers if needed for backward compat, but likely a straight rename works since these are all module-internal.

**Done:** `transform_builder.py` has no local copies of these functions. All existing yarrrml-gen tests pass.

### Task 3: Add `check_slot_class_reachability` to `schema_utils.py`

Add a high-level validation function:

```python
@dataclass
class SlotClassMismatch:
    row: SSSOMRow
    target_slot_name: str
    target_owning_class: str
    mapped_target_classes: set[str]

def check_slot_class_reachability(
    rows: list[SSSOMRow],
    source_view: SchemaView,
    master_view: SchemaView,
) -> list[SlotClassMismatch]:
```

Logic:
1. Build slot owner indices for both schemas via `build_slot_owner_index`
2. For each confirmed row (MMC or HC justification):
   - Resolve `local_name(subject_id)` in source, `local_name(object_id)` in master
   - If both resolve to classes → add target class to `mapped_classes`
   - If both resolve to slots → record as slot mapping with `target_owning_class`
   - Skip unresolvable / mixed rows (lint doesn't need to be fatal here)
3. For each slot mapping, call `nearest_mapped_ancestor(target_owning_class, mapped_classes, master_view)`
4. If returns None → append to mismatches
5. Return mismatches

**Done:** Function exists, handles edge cases (empty rows, no class mappings), returns typed results.

### Task 4: Wire structural check into `rosetta/cli/lint.py`

Add CLI options:
```python
@click.option("--source-schema", type=click.Path(exists=True), default=None,
              help="Source LinkML schema (enables structural checks).")
@click.option("--master-schema", type=click.Path(exists=True), default=None,
              help="Master LinkML schema (enables structural checks).")
```

Validation: if exactly one is provided, `click.echo("Error: --source-schema and --master-schema must be provided together.", err=True); sys.exit(1)`.

When both provided:
1. Load both as `SchemaView(str(path))`
2. Call `check_slot_class_reachability(confirmed_rows, source_view, master_view)`
3. For each mismatch, append:
   ```python
   LintFinding(
       rule="slot_class_unreachable",
       severity="BLOCK",
       source_uri=mismatch.row.subject_id,
       target_uri=mismatch.row.object_id,
       message=f"Slot '{mismatch.target_slot_name}' belongs to class '{mismatch.target_owning_class}' "
               f"which is not reachable from any mapped class ({', '.join(sorted(mismatch.mapped_target_classes))}). "
               f"Map the source class to '{mismatch.target_owning_class}' or one of its subclasses.",
   )
   ```

Insert structural check as step 2.5 (after proposal checks, before unit/datatype checks) since it's also a BLOCK-level gate.

**Done:** `rosetta-lint --sssom X --source-schema Y --master-schema Z` emits BLOCK findings for unreachable slots.
