## Task 3: Create rosetta/cli/compile.py

Extract steps 1–10 from `yarrrml_gen.py:cli()` into a new `compile.py`. This is the SSSOM → TransformSpec YAML → YARRRML compilation step.

**Interface (already standardized per design):**
```
rosetta compile <SSSOM_FILE> --source-schema <path> --master-schema <path> [-o <output>] [--coverage-report <path>]
```

Key changes from yarrrml-gen:
- `SSSOM_FILE` is a positional Click argument (not `--sssom` flag)
- `--source-schema` and `--master-schema` are required Click options
- `-o`/`--output` defaults to stdout (not required)
- **Removed flags:** `--include-manual`, `--allow-empty`, `--force`, `--source-format`
- HC-only filtering hardcoded (no include_manual parameter)
- Source format read from `source_def.annotations["rosetta_source_format"]` — error if missing
- Empty spec after filtering → always exit 1 (no allow_empty bypass)
- Unresolvable CURIEs → always exit 1 (no force bypass)
- After building TransformSpec, also compile to YARRRML via `YarrrmlCompiler` and write that as output (the compile command produces the YARRRML, not just the TransformSpec)

Wait — design decision needed: should `compile` output TransformSpec YAML or YARRRML? The design says "SSSOM → TransformSpec YAML → YARRRML". The final output should be YARRRML since that's what `run` consumes. But the TransformSpec is an intermediate.

**Resolution:** `compile` outputs YARRRML by default (the executable artifact that `run` consumes). Add `--spec-output <path>` to optionally also write the intermediate TransformSpec YAML.

Also annotate the YARRRML with `rosetta:source_format` comment for `run` to read.

**Files:** `rosetta/cli/compile.py`
**Done:** `rosetta compile <sssom> --source-schema <s> --master-schema <m>` writes YARRRML to stdout.

## Task 4: Create rosetta/cli/run.py

Extract steps 12–15 from `yarrrml_gen.py:cli()` into a new `run.py`. This is the YARRRML + data → JSON-LD execution step.

**Interface (already standardized per design):**
```
rosetta run <MAPPING_FILE> <SOURCE_FILE> [-o <output>] [--validate <shapes-dir>] [--workdir <path>] [--context-output <path>] [--validate-report <path>]
```

Key changes from yarrrml-gen:
- `MAPPING_FILE` (YARRRML) and `SOURCE_FILE` are positional Click arguments
- `-o`/`--output` replaces `--jsonld-output`, defaults to stdout
- `--validate <shapes-dir>` is a single option taking the shapes directory path (replaces `--validate` boolean + `--shapes-dir` pair)
- Reads YARRRML from the mapping file (no in-memory compilation — that's compile's job)
- Stdout collision guard simplified: only `--output` and `--validate-report` can target stdout

**Files:** `rosetta/cli/run.py`
**Done:** `rosetta run <mapping.yml> <source.csv>` writes JSON-LD to stdout.
