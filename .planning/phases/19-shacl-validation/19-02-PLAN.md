## Goal

Establish the on-disk convention for hand-edited SHACL overrides, ship a worked example, retire the v1 `mapping.shacl.ttl`, and switch `rosetta-validate --shapes-dir` to recursive directory walking so a single `--shapes-dir` flag pulls both regen output and user overrides.

- **D-19-04:** Two-subdir convention (`generated/` + `overrides/`).

## Out of scope

- Generator behavior (Plan 19-01).
- `rosetta-yarrrml-gen --validate` flag (Plan 19-03).
- Per-class override granularity (deferred â€” see G-19-02).

### Task 3 â€” Worked-example override

**File:** `rosetta/policies/shacl/overrides/track_bearing_range.ttl` (new)

Write a small SHACL shape demonstrating override semantics. Suggested constraint: every `mc:AirTrack` that declares `mc:hasBearing` must have a value in [0, 360):

```turtle
@prefix sh: <http://www.w3.org/ns/shacl#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
@prefix mc: <https://ontology.nato.int/core/MasterCOP#> .

mc:AirTrackBearingRangeShape
    a sh:NodeShape ;
    sh:targetClass mc:AirTrack ;
    sh:property [
        sh:path mc:hasBearing ;
        sh:datatype xsd:double ;
        sh:minInclusive 0 ;
        sh:maxExclusive 360 ;
        sh:message "mc:hasBearing must be a double in [0, 360)." ;
    ] .
```

### Task 4 â€” Switch `--shapes-dir` to recursive (symlink-safe + non-shape-warned)

**Files:** `rosetta/cli/validate.py` (modify), `rosetta/core/shapes_loader.py` (new)

Extract the shapes-dir discovery + load into a small reusable helper (Plan 19-03 will also call it from `yarrrml_gen.py`). Single source of truth for the walk semantics.

**`rosetta/core/shapes_loader.py`:**

```python
def load_shapes_from_dir(shapes_dir: Path) -> rdflib.Graph:
    """Recursively load *.ttl shapes from a directory into a single graph.

    - Walks with os.walk(followlinks=False) â€” symlink loops cannot hang.
    - Sorts file list for deterministic merge order.
    - For each file: parses as Turtle, counts sh:NodeShape + sh:PropertyShape
      triples; if zero, emits a click.echo stderr warning but still merges
      (open-world principle â€” user explicitly put it in --shapes-dir).
    - Raises click.UsageError if no .ttl files found at all.
    """
```

Use `os.walk(shapes_dir, followlinks=False)` and filter `.ttl`; sort with `sorted(...)` over the resulting list. For each file: parse, then run a quick `len(list(g.triples((None, RDF.type, SH.NodeShape)))) + len(list(g.triples((None, RDF.type, SH.PropertyShape))))`; emit warning when zero.

**`rosetta/cli/validate.py`:**

- Replace inline `Path(shapes_dir).glob("*.ttl")` block with `from rosetta.core.shapes_loader import load_shapes_from_dir; shapes_g = load_shapes_from_dir(Path(shapes_dir))`.

**Done when:**
- `validate.py` walks subdirs via the shared helper.
- Symlink loops cannot hang the walk (proven by Task 5 test 5).
- Non-shape Turtle files trigger a stderr warning but still merge (proven by Task 5 test 6).
