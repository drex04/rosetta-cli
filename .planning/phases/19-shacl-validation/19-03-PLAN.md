# Plan 19-03 — `--validate` pipeline wiring + JSON-LD input

## Goal

Close the loop. Extract the SHACL invocation into a shared helper so the CLI path and the in-pipeline path can't drift, teach `rosetta-validate` to consume the v2 pipeline's JSON-LD output, and wire `--validate` into `rosetta-yarrrml-gen --run` against the in-memory `rdflib.Graph` (blocking JSON-LD emission on violation per locked decision D-19-06).

## Locked decisions feeding this plan

- **D-19-05:** `--validate` uses the in-memory `rdflib.Graph` from `run_materialize` — no re-parse cost.
- **D-19-06:** `--validate` blocks JSON-LD emission on any violation (exit 1, no partial file).
- **D-19-07:** `rosetta-validate --data` accepts JSON-LD; suffix-based autodetect + `--data-format` override.
- **D-19-08:** Single shared helper `rosetta/core/shacl_validate.py` for both entrypoints.

## Out of scope

- Generator (Plan 19-01).
- Override workflow + recursive `--shapes-dir` (Plan 19-02 — relied on but not modified here).
- Streaming validation, partial reports, or multi-shape-source merging beyond what `--shapes-dir` already does.

### Task 2 — Refactor `rosetta-validate` + add JSON-LD input

**File:** `rosetta/cli/validate.py` (modify)

Two changes:

1. Replace the inline pyshacl call with `validate_graph(data_g, shapes_g)`. Drop the moved helpers + SPARQL constant.
2. Add JSON-LD support:
   - New flag: `--data-format {turtle,json-ld,auto}` (default `auto`).
   - In `auto` mode: dispatch by suffix — `.ttl` → turtle; `.jsonld` / `.json` / `.json-ld` → json-ld; anything else → fall back to turtle (current behavior, no regression).
   - rdflib's bundled JSON-LD parser handles both (`format='json-ld'`).
   - Unknown explicit format → Click `UsageError`.

**Done when:**
- All existing `rosetta-validate` tests still pass.
- New tests (Task 4) for JSON-LD input + `--data-format` flag pass.

### Task 3 — Wire `--validate` into `rosetta-yarrrml-gen`

**File:** `rosetta/cli/yarrrml_gen.py` (modify)

Add three flags:
- `--validate` (boolean, default false). Requires `--run` (already implies `--data`); requires `--shapes-dir`. Click validation: raise `UsageError` if `--validate` is set without `--shapes-dir`, OR without `--run`.
- `--shapes-dir PATH` (optional unless `--validate` is set). Calls `rosetta.core.shapes_loader.load_shapes_from_dir` (introduced in Plan 19-02 Task 4) — single shared walker, symlink-safe via `os.walk(followlinks=False)`, emits stderr warning for non-shape Turtle files.
- `--validate-report PATH` (optional). When set, writes the `ValidationReport` JSON to that path; otherwise writes to stderr.

**Step 0 stdout-collision guard (added per `[review-harden]` truth):** Extend the existing `_check_stdout_collisions` block (`yarrrml_gen.py:134`) to also check `--validate-report`. Reject all three pairwise collisions (`--output - + --jsonld-output -`, `--output - + --validate-report -`, `--jsonld-output - + --validate-report -`) with a Click `UsageError` *before* any I/O or materialization. The "no partial output" invariant must hold for the new flag too.

Inside the existing `with run_materialize(...) as graph:` block at `rosetta/cli/yarrrml_gen.py:275`:

```python
with run_materialize(yarrrml_text, data_path, resolved_workdir) as graph:
    if validate:
        report = validate_graph(graph, shapes_g)
        if not report.summary.conforms:
            _write_validate_report(report, validate_report_path)  # stderr or file
            click.echo(
                f"SHACL validation failed: "
                f"{report.summary.violation} violation(s), "
                f"{report.summary.warning} warning(s). "
                f"JSON-LD emission blocked.",
                err=True,
            )
            sys.exit(1)
    # existing graph_to_jsonld + write block runs only on conformance
    jsonld_bytes = graph_to_jsonld(...)
    ...
```

**Critical:** the validation + early-exit must happen **before** any JSON-LD bytes are written to stdout or `--jsonld-output`. The "no partial output" invariant from Phase 18 hardening applies.

**Done when:**
- Three new flags accept user input; `--help` documents them.
- Validation runs against the in-memory graph; happy path reaches `graph_to_jsonld`; violation path writes report + exits 1 with no JSON-LD bytes emitted.

## Risks / unknowns

- **rdflib JSON-LD parser quirks.** rdflib supports JSON-LD natively but requires the JSON-LD plugin (`pyld`-backed); `linkml` already pulls it transitively, so no new dep — verify in Task 2 by inspecting `uv.lock`.
- **Stdout collision** — addressed in Task 3 step-0 guard + Task 4 adversarial tests for all three pairs. No longer a deferred risk.
- **Order of operations on violation.** The early-exit must run *before* the existing stdout write block. Easy to get wrong — Task 4's "no partial JSON-LD" adversarial test pins the contract.
