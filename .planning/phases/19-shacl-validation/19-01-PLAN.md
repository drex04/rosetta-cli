# Plan 19-01 — `rosetta-shacl-gen`

## Goal

Ship a new CLI tool that auto-generates SHACL shapes from the master LinkML schema. Output is a single Turtle file ready for either standalone `rosetta-validate --shapes` or directory-merged `--shapes-dir` use (Plan 19-02). Closed-world default with `sh:ignoredProperties` for PROV-O / dcterms / rdf:type to survive downstream stamping. Unit-aware shapes leverage existing Phase 17 unit detection so master slots like `hasAltitudeM` and `hasSpeedKnots` get a `qudt:hasUnit` constraint without any new schema annotations.

## Locked decisions feeding this plan

- **D-19-02:** New CLI, not a flag on `rosetta-validate`.
- **D-19-03 / D-19-11:** Subclass `linkml.generators.shaclgen.ShaclGenerator` — single extension point for closed-default + ignored-properties + unit-aware shapes.
- **D-19-10:** Closed-world default; `--open` flag emits open-world shapes.
- **G-19-02:** One-file output (`--output PATH` → single `.ttl`). Per-class output deferred.

## Out of scope

- Override workflow (Plan 19-02).
- `--validate` flag on `rosetta-yarrrml-gen` (Plan 19-03).
- JSON-LD input for `rosetta-validate` (Plan 19-03).
- Per-class output mode.

## Tasks

### Task 0 — Extend `detect_unit` for master COP slot patterns *(added in plan-review)*

**File:** `rosetta/core/unit_detect.py` (modify) + `rosetta/tests/test_unit_detect.py` (modify)

System audit found that of 6 sampled master slots, only `hasAltitudeM` was recognized by Phase 17's `detect_unit`. Per plan-review decision **2C**, extend the regex cascade so the unit-aware shape feature in Task 1 has meaningful coverage on the actual master schema.

Add regex patterns + `_PINT_TO_QUDT_IRI` entries:

| Slot pattern | QUDT IRI | Unit |
|---|---|---|
| `(?:^\|_\|(?<=[a-z]))Knots?$` (case-insensitive after `_snake_case`) | `unit:KN` | knot |
| `(?:^\|_\|(?<=[a-z]))Bearing$` OR `(?:^\|_\|(?<=[a-z]))Degrees?$` (heading/bearing context) | `unit:DEG` | degree |
| `(?:^\|_\|(?<=[a-z]))VerticalRate$` AND description contains `feet per minute` / `ft/min` / `fpm` | `unit:FT-PER-MIN` | foot per minute |

Reuse the existing `_snake_case` preprocessor (added by the 2026-04-18 unit-conversion fix) so `hasSpeedKnots` snake-cases to `has_speed_knots` and matches `_knots$`.

Add 4 new tests in `test_unit_detect.py`:
1. `test_detect_knots_suffix` — `detect_unit("hasSpeedKnots", "")` → `unit:KN`.
2. `test_detect_bearing_pattern` — `detect_unit("hasBearing", "Bearing in degrees true")` → `unit:DEG`.
3. `test_detect_vertical_rate_with_fpm_description` — `detect_unit("hasVerticalRate", "Rate of climb or descent in feet per minute")` → `unit:FT-PER-MIN`.
4. `test_detect_vertical_rate_without_fpm_description_returns_none` — same slot name, empty description → `None` (description-disambiguated; avoids matching every "rate" slot).

**Done when:**
- 4 new tests pass.
- `detect_unit` returns the expected QUDT IRIs for the 4 master COP slot patterns above.
- Existing 3 unit-detect regression tests from the 2026-04-18 fix still pass.

### Task 1 — `RosettaShaclGenerator` subclass + helper

**File:** `rosetta/core/shacl_generator.py` (new)

**Sub-step 1.0 — spike (≤30 min, added per plan-review decision 3A):** Read `linkml.generators.shaclgen.ShaclGenerator` source. Determine which of {`closed`, `add_prefix`, `default_prefix`, `aliased_slot_name`, `serialize`, `as_graph`} are settable via constructor / kwargs vs. require subclassing. The system audit found `closed` is already a public attribute — if `sh:closed true` and `sh:ignoredProperties` injection are achievable via constructor kwargs + a single `as_graph` post-walk, **collapse this task to a thin wrapper** instead of a full subclass. Document the chosen surface in the module docstring so the next maintainer doesn't re-spike.

After the spike, implement (subclass OR wrapper, whichever the spike chose):

- Pass `closed=closed_flag` to the underlying `ShaclGenerator` (covers `sh:closed`).
- Post-walk the produced `rdflib.Graph` to:
  - Add `sh:ignoredProperties ( prov:wasGeneratedBy prov:wasAttributedTo dcterms:created dcterms:source rdf:type )` to every `sh:NodeShape` (skip if `--open`).
  - For each `SlotDefinition` in the schema: call `rosetta.core.unit_detect.detect_unit(slot.name, slot.description or "")`; if a QUDT IRI is returned, emit a `sh:property` block on the parent class shape with `sh:path qudt:hasUnit ; sh:hasValue <iri>`. Task 0 ensures the four master COP unit-bearing slots are recognized.
- Public helper: `generate_shacl(linkml_path: Path, *, closed: bool = True) -> str` returns Turtle.
- Annotate types per repo strict-pyright rules. Use broad `rdflib.Graph` / `rdflib.term.Node` at boundaries.

**Done when:**
- File exists and exports `generate_shacl` (and `RosettaShaclGenerator` if the spike chose subclassing).
- Module docstring records the spike outcome (what's settable via constructor vs. post-walk).
- Unit test (Task 4) covers: closed-default emission, `--open` skips closed/ignored, QUDT IRI injection on `hasAltitudeM` and `hasSpeedKnots` (the latter validates Task 0's `detect_unit` extension flowed through).

### Task 2 — `rosetta-shacl-gen` CLI

**File:** `rosetta/cli/shacl_gen.py` (new)

- Calls `generate_shacl(input_path, closed=not open_flag)`; writes Turtle output.

### Task 4 — Unit + behavioral tests

**File:** `rosetta/tests/test_shacl_gen.py` (new)

Six tests minimum:
1. `test_generates_valid_turtle` — output parses with `rdflib.Graph().parse(format="turtle")` and contains at least one `sh:NodeShape`.
2. `test_closed_default_adds_sh_closed_true` — every `sh:NodeShape` in output has `sh:closed true`.
3. `test_closed_default_adds_ignored_properties` — every `sh:NodeShape` has an `sh:ignoredProperties` list containing the five expected IRIs.
4. `test_open_flag_omits_closed_and_ignored` — with `--open`, no `sh:closed` and no `sh:ignoredProperties` triples.
5. `test_unit_aware_shape_emitted_for_qudt_slot` — feed a fixture schema with a slot named `hasAltitudeM`; assert the parent class shape has `sh:path qudt:hasUnit ; sh:hasValue unit:M`.
6. `test_validates_conformant_graph_and_rejects_typo` — round-trip: generate shapes → run `pyshacl.validate()` against (a) a conformant data graph (passes, `conforms=True`) and (b) a graph with a typo'd predicate `mc:hasAltidude` on a `mc:AirTrack` (fails, `conforms=False`, ≥1 violation).

**Done when:**
- All 6 tests pass under `uv run pytest -m "not slow" rosetta/tests/test_shacl_gen.py`.

- **Default-prefix collision.** Master schema's `default_prefix: mc` must not clash with rosetta global prefixes (`qudt`, `prov`, `dcterms`); double-check in Task 1 that the generator emits explicit prefix declarations before any constrained URI.
- **Test fixture for unit-aware shape.** May need a tiny LinkML fixture with `hasAltitudeM` — keep alongside `master_cop.linkml.yaml` or inline as a string in the test.
