# Plan 08-01: rosetta-validate

## Context

Phase 8 implements `rosetta-validate`, a pySHACL-based CLI that validates RDF mapping
artifacts against SHACL constraint shapes. pySHACL (`>=0.20`) is already declared as a
dependency. A stub `rosetta/cli/validate.py` exists and needs full implementation.

The tool follows the same Unix-composable pattern as other Rosetta tools: reads files,
writes JSON to stdout or `--output`, exits 0/1.

## Tasks

### Task 1 — Add ValidationReport Pydantic models to rosetta/core/models.py

**File:** `rosetta/core/models.py`

Append after the existing `# --- Embeddings ---` section:

```python
# --- Validate ---

class ValidationFinding(BaseModel):
    focus_node: str
    severity: Literal["Violation", "Warning", "Info"]
    constraint: str
    source_shape: str | None = None
    message: str

class ValidationSummary(BaseModel):
    violation: int
    warning: int
    info: int
    conforms: bool

class ValidationReport(BaseModel):
    findings: list[ValidationFinding]
    summary: ValidationSummary
```

`severity` uses SHACL vocabulary terms (`sh:Violation`, `sh:Warning`, `sh:Info`) with the
`sh:` prefix stripped (e.g. `"Violation"`).

**Done when:** `from rosetta.core.models import ValidationReport, ValidationFinding, ValidationSummary` succeeds and `uv run basedpyright` passes clean.

### Task 2 — Implement rosetta/cli/validate.py

Replace the stub with a full Click entrypoint.

**CLI signature:**
```
rosetta-validate
  --data     PATH   (required) RDF Turtle to validate
  --shapes   PATH   (optional) Single SHACL shapes Turtle file
  --shapes-dir PATH (optional) Directory; loads all *.ttl inside
  --output   PATH   (optional, default stdout) JSON report destination
  --config   PATH   (default rosetta.toml)
```

At least one of `--shapes` or `--shapes-dir` must be provided; raise `click.UsageError`
if neither is given.

**Implementation steps inside `cli()`:**

1. Load data graph: `rdflib.Graph().parse(data, format="turtle")`
2. Build shapes graph:
   - If `--shapes`: `rdflib.Graph().parse(shapes, format="turtle")`
   - If `--shapes-dir`: glob `Path(shapes_dir).glob("*.ttl")`, parse each, merge via
     `shapes_graph += g` (rdflib in-place union)
   - If both provided: parse shapes file first, then merge dir on top
3. Run pySHACL: `pyshacl.validate(data_g, shacl_graph=shapes_g, do_owl_imports=False, meta_shacl=False)`
   - Returns `(conforms: bool, results_graph: rdflib.Graph, results_text: str)`
4. Parse results from `results_graph`:
   - SPARQL on results_graph:
     ```sparql
     PREFIX sh: <http://www.w3.org/ns/shacl#>
     SELECT ?focusNode ?severity ?message ?constraint ?shape WHERE {
         ?result a sh:ValidationResult ;
                 sh:focusNode ?focusNode ;
                 sh:resultSeverity ?severity ;
                 sh:resultMessage ?message ;
                 sh:sourceConstraintComponent ?constraint .
         OPTIONAL { ?result sh:sourceShape ?shape . }
     }
     ```
   - Coerce all rdflib nodes to `str` before constructing `ValidationFinding`
   - Strip `http://www.w3.org/ns/shacl#` prefix from severity IRI to get `"Violation"` etc.
5. Build `ValidationSummary` from finding counts + `conforms` bool
6. Build `ValidationReport`, serialise with `report.model_dump_json(indent=2)`
7. Write to `open_output(output)`
8. Exit: `sys.exit(0 if conforms else 1)` — **after** writing output

**Type annotations:** All parameters and return types explicit. Function-level helper
`_strip_shacl_prefix(iri: str) -> str` for prefix stripping.

**Error handling:** Wrap entire body in `try/except Exception as e:` → `click.echo(f"Error: {e}", err=True); sys.exit(1)` (matching lint.py pattern).

**Done when:** `uv run rosetta-validate --help` shows all options; `uv run basedpyright` passes clean.

### Task 3 — Add sample SHACL shapes file rosetta/policies/mapping.shacl.ttl

Create a minimal but realistic shapes file that constrains mapping artifacts produced by
rosetta-ingest and rosetta-rml-gen.

```turtle
@prefix sh:   <http://www.w3.org/ns/shacl#> .
@prefix rose: <http://rosetta.interop/ns/> .
@prefix xsd:  <http://www.w3.org/2001/XMLSchema#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

# Every rose:Field must have rdfs:label (string) and rose:dataType
rose:FieldShape
    a sh:NodeShape ;
    sh:targetClass rose:Field ;
    sh:property [
        sh:path rdfs:label ;
        sh:minCount 1 ;
        sh:datatype xsd:string ;
        sh:message "rose:Field must have an rdfs:label of type xsd:string." ;
    ] ;
    sh:property [
        sh:path rose:dataType ;
        sh:minCount 1 ;
        sh:message "rose:Field must declare rose:dataType." ;
    ] .

# Every rose:Mapping must link source and target
rose:MappingShape
    a sh:NodeShape ;
    sh:targetClass rose:Mapping ;
    sh:property [
        sh:path rose:sourceField ;
        sh:minCount 1 ;
        sh:message "rose:Mapping must have rose:sourceField." ;
    ] ;
    sh:property [
        sh:path rose:targetField ;
        sh:minCount 1 ;
        sh:message "rose:Mapping must have rose:targetField." ;
    ] .
```

**Done when:** File parses as valid Turtle: `python -c "import rdflib; g=rdflib.Graph(); g.parse('rosetta/policies/mapping.shacl.ttl', format='turtle'); print(len(g), 'triples')"`

### Task 4 — Write tests in rosetta/tests/test_validate.py

Create `rosetta/tests/test_validate.py` with the following test cases (use
`click.testing.CliRunner` matching the pattern in `test_lint.py`):

**Fixtures (module-scoped tmp_path via `tmp_path_factory`):**

`conformant_data.ttl` — a graph where all `rose:Field` nodes have `rdfs:label` and `rose:dataType`:
```turtle
@prefix rose: <http://rosetta.interop/ns/> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix xsd:  <http://www.w3.org/2001/XMLSchema#> .
<http://example.org/f1> a rose:Field ;
    rdfs:label "Field One"^^xsd:string ;
    rose:dataType xsd:integer .
```

`violating_data.ttl` — a `rose:Field` missing `rdfs:label`:
```turtle
@prefix rose: <http://rosetta.interop/ns/> .
@prefix xsd:  <http://www.w3.org/2001/XMLSchema#> .
<http://example.org/f2> a rose:Field ;
    rose:dataType xsd:integer .
```

**Test functions:**

```python
def test_validate_conformant(...)         # exit 0, report conforms=true, no findings
def test_validate_violation(...)          # exit 1, report has >=1 finding, severity Violation
def test_validate_missing_shapes_arg(...) # exit != 0, UsageError when neither --shapes nor --shapes-dir
def test_validate_shapes_dir(...)         # --shapes-dir pointing to a tmp dir with mapping.shacl.ttl copy; violation detected
def test_validate_output_file(...)        # --output writes to file, stdout empty
def test_validate_report_schema(...)      # JSON output deserialises into ValidationReport without error
def test_validate_finding_fields(...)     # finding has focus_node, message, constraint, severity populated
```

Each test imports from `rosetta.core.models import ValidationReport` to validate the
output schema.

**Done when:** `uv run pytest rosetta/tests/test_validate.py -v` passes all 7 tests.
