# Phase 22 Plan 01: Ingest Expansion

## Task 1: Multi-schema positional args

**Files:** `rosetta/cli/ingest.py`

1. Change `@click.argument("schema_file")` to `@click.argument("schema_files", nargs=-1, required=True)`
2. Add `-o`/`--output` behavior:
   - 0 inputs → Click error (required=True handles this)
   - 1 input + no `-o` → stdout (current behavior)
   - 1 input + `-o file.yaml` → write to file (current behavior)
   - 1 input + `-o dir/` (trailing slash or existing dir) → write to `dir/{stem}.linkml.yaml`
   - N inputs + no `-o` → write each to `./{stem}.linkml.yaml`
   - N inputs + `-o dir` → write each to `dir/{stem}.linkml.yaml`; mkdir -p
   - N inputs + `-o -` → UsageError
3. Loop over schema_files, calling `normalize_schema` + `stamp_source_format` + `stamp_slot_paths` for each
4. `check_prefix_collision` runs against the output directory for each file
5. Preserve the `--schema-format` flag — when provided with multiple inputs of different formats, raise UsageError ("--schema-format cannot be used with mixed-format inputs; omit it for auto-detection")

**Done when:** `rosetta ingest a.json b.xsd -o out/` produces two files. Single-input backward compatibility preserved. `uv run basedpyright` clean.

## Task 2: Roll translate into ingest

**Files:** `rosetta/cli/ingest.py`

1. Add Click options:
   - `--translate` / `is_flag=True` / `default=False` / help="Translate schema titles to English via DeepL"
   - `--lang` / `default="EN"` / help="Source language code (e.g., DE, FR). EN is a no-op passthrough"
   - `--deepl-key` / `default=None` / help="DeepL API key (or set DEEPL_API_KEY env var)"
2. After `normalize_schema` + stamp calls, if `--translate` is set:
   - If `lang` starts with "EN" → skip translation (passthrough)
   - Else resolve key: `deepl_key or os.environ.get("DEEPL_API_KEY")`
   - If no key → `click.UsageError("DeepL API key required...")`
   - Call `translate_schema(schema_def, source_lang=lang, target_lang="EN-US", deepl_key=resolved_key)`
   - Use the returned schema for output
3. Add imports: `from rosetta.core.translation import translate_schema`, `import os`

**Done when:** `rosetta ingest schema.json --translate --lang de --deepl-key KEY` produces translated output. `--translate` without key errors. `--translate --lang EN` passes through.

## Task 3: Add `--master` flag

**Files:** `rosetta/cli/ingest.py`

1. Add Click option: `--master` / `type=click.Path(exists=True)` / `default=None` / help="Master ontology file (.ttl/.owl) — normalizes and generates SHACL shapes"
2. When `--master` is provided:
   a. Detect format of master file via `detect_format` (must be `ttl`, `owl`, or `rdfs`)
   b. Normalize: `master_def = normalize_schema(Path(master), fmt=master_fmt, schema_name=master_stem)`
   c. If `--translate`: translate master_def too
   d. Write normalized master to output location as `{master_stem}.linkml.yaml` (same output dir logic as source schemas)
   e. Generate SHACL: `turtle = generate_shacl(master_output_path, closed=True)`
   f. Write shapes to `{master_stem}.shacl.ttl` alongside the normalized master
3. Add import: `from rosetta.core.shacl_generator import generate_shacl`
4. --master is processed BEFORE source schemas (master needs to exist for downstream tools)

**Done when:** `rosetta ingest --master ontology.ttl` produces `ontology.linkml.yaml` + `ontology.shacl.ttl`. Shapes are closed-world by default.

## Task 4: Scaffold `rosetta.toml` on first `--master` run

**Files:** `rosetta/cli/ingest.py`

1. After processing --master, check if `rosetta.toml` exists in the current directory
2. If absent, write a minimal config:
   ```toml
   [master]
   schema = "{master_output_path}"
   shapes = "{shapes_output_path}"
   ```
3. If present, do not overwrite (idempotent re-runs don't clobber config)
4. Print info to stderr: `"Created rosetta.toml"` or `"rosetta.toml already exists, skipping"`

**Done when:** First `--master` run creates `rosetta.toml`; second run skips it. File content is valid TOML.
