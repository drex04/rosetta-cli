## Task 5: Update pipeline demo and README

**Files:** `scripts/pipeline-demo.sh`, `README.md`

1. **pipeline-demo.sh** — rewrite to use the 5-command workflow:
   ```bash
   # 1. Ingest source schemas + master ontology
   rosetta ingest schema_a.json schema_b.xsd \
     --master ontology.ttl --translate --lang de -o output/

   # 2. Generate candidate mappings
   rosetta suggest output/schema_a.linkml.yaml output/ontology.linkml.yaml \
     --audit-log output/audit-log.sssom.tsv -o output/candidates.sssom.tsv

   # 3. Analyst selects mappings, then appends to ledger
   rosetta ledger append --role analyst output/candidates.sssom.tsv \
     --source-schema output/schema_a.linkml.yaml \
     --master-schema output/ontology.linkml.yaml \
     --audit-log output/audit-log.sssom.tsv

   # 4. Accreditor reviews and approves
   rosetta ledger review --audit-log output/audit-log.sssom.tsv -o output/review.sssom.tsv
   rosetta ledger append --role accreditor output/review.sssom.tsv \
     --source-schema output/schema_a.linkml.yaml \
     --master-schema output/ontology.linkml.yaml \
     --audit-log output/audit-log.sssom.tsv

   # 5. Compile approved mappings to YARRRML
   rosetta compile output/audit-log.sssom.tsv \
     --source-schema output/schema_a.linkml.yaml \
     --master-schema output/ontology.linkml.yaml \
     -o output/mapping.yarrrml.yaml

   # 6. Transform source data to JSON-LD (validates by default)
   rosetta transform output/mapping.yarrrml.yaml output/source_data.json \
     --master-schema output/ontology.linkml.yaml \
     -o output/result.jsonld
   ```
2. **README.md** — update:
   - Tool listing: 5 commands (ingest, suggest, ledger, compile, transform)
   - Quick-start example using new command shapes
   - Remove all references to translate, embed, lint, validate, shapes as standalone commands
   - Note breaking changes in a "Migration from v3.x" section

**Done when:** `pipeline-demo.sh` runs end-to-end (with mocked DeepL/model). README reflects the 5-command surface.
