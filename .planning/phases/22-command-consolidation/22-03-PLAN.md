# Phase 22 Plan 03: Ledger Append Lint Gate + Transform Default Validation

## Task 1: Add `--role` flag to `ledger append`

**Files:** `rosetta/cli/ledger.py`

1. Add Click option: `--role` / `type=click.Choice(["analyst", "accreditor"])` / `required=True` / help="Role determines which rows are accepted and how they are linted"
2. Role-based row filtering in `append_cmd`:
   - `--role analyst`: filter incoming rows to only `ManualMappingCuration` justification. If any row has `HumanCuration`, it will be caught by lint (Task 2). Non-MMC/HC rows (e.g., CompositeMatching) are silently skipped as before.
   - `--role accreditor`: filter incoming rows to only `HumanCuration` justification. Non-HC rows are silently skipped with a stderr count.
3. Update function signature to accept `role: str` parameter

**Done when:** `--role analyst` appends only MMC rows. `--role accreditor` appends only HC rows. Missing `--role` → Click UsageError.

---

## Task 2: Integrate lint into `ledger append`

**Files:** `rosetta/cli/ledger.py`

1. Add Click options:
   - `--source-schema` / `type=click.Path(exists=True)` / `required=True`
   - `--master-schema` / `type=click.Path(exists=True)` / `required=True`
2. After row filtering (Task 1) but before `append_log`:
   a. Load source and master schemas via `SchemaView`
   b. Call `check_sssom_proposals(filtered_rows, log)` for basic lint checks
   c. Run `_check_units` and `_check_datatype` for each row
   d. For `--role analyst`: add HC-in-candidates BLOCK check (from Phase 21)
   e. Collect all `LintFinding` results
3. If any BLOCK findings: print `LintReport` JSON to stderr, exit 1, do not append
4. If only WARNING/INFO findings: print report to stderr, proceed with append
5. Add imports from lint module: `from rosetta.cli.lint import check_sssom_proposals, _check_units, _check_datatype`
   - Consider moving these functions to `rosetta/core/lint.py` to avoid CLI-to-CLI import (better: refactor check functions into core during this task)

**Refactor note:** The lint check functions (`check_sssom_proposals`, `_check_units`, `_check_datatype`) currently live in `rosetta/cli/lint.py`. Move them to a new `rosetta/core/lint.py` module so both `ledger.py` and the (soon-to-be-removed) `lint.py` CLI can import from core. This avoids a CLI-to-CLI import anti-pattern.

**Done when:** `ledger append` with invalid candidates exits 1 with LintReport on stderr. Valid candidates append successfully. Lint functions live in `rosetta/core/lint.py`.

## Task 3: Add `--dry-run` flag

**Files:** `rosetta/cli/ledger.py`

1. Add Click option: `--dry-run` / `is_flag=True` / `default=False` / help="Run lint checks without appending to the audit log"
2. When `--dry-run`:
   - Run all lint checks (same as Task 2)
   - Print LintReport JSON to stdout (not stderr — this is the primary output)
   - Exit 0 if no BLOCK findings, exit 1 if any BLOCKs
   - Do NOT call `append_log`
3. Without `--dry-run`: current behavior (lint gate + append)

**Done when:** `--dry-run` produces lint report without side effects. Exit code reflects lint result.

---

## Task 4: `transform` validates by default

**Files:** `rosetta/cli/transform.py`

1. Add `--no-validate` flag: `is_flag=True` / `default=False` / help="Skip SHACL validation"
2. Change `--validate` from optional shapes-dir to required default:
   - Current: `--validate <shapes-dir>` is optional (None = no validation)
   - New: `--validate <shapes-dir>` is still the way to specify shapes, but validation runs by default
   - If `--no-validate` is set: skip validation entirely
   - If `--validate` is not provided and `--no-validate` is not set: look for shapes in conventional location (same dir as master schema, or `rosetta/policies/shacl/`)
   - If no shapes found and `--no-validate` not set: warn on stderr but proceed (don't hard-fail on missing shapes — user may not have generated them yet)
3. Mutual exclusion: `--validate` and `--no-validate` together → UsageError

**Done when:** `rosetta transform mapping.yml data.json --master-schema m.yaml` validates by default. `--no-validate` skips. Missing shapes warns but proceeds.
