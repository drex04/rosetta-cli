# Phase 22 Plan 02: Suggest Expansion

## Task 1: Change positional args from embedding JSON to LinkML YAML

**Files:** `rosetta/cli/suggest.py`

1. Replace the two positional args:
   - Old: `@click.argument("source")` + `@click.argument("master")` (embedding JSON paths)
   - New: `@click.argument("source_schema")` + `@click.argument("master_schema")` (LinkML YAML paths)
2. Update help text and docstring to reference schemas, not embeddings
3. Add `--model` Click option: `default="intfloat/e5-large-v2"`, help="Sentence-transformer model for embeddings"

**Done when:** CLI accepts `.linkml.yaml` files as positional args. Old embedding JSON input rejected gracefully.

## Task 2: Internal embedding pipeline

**Files:** `rosetta/cli/suggest.py`

1. Load both schemas via `yaml_loader.load(str(path), target_class=SchemaDefinition)`
2. Extract text inputs: `src_texts = extract_text_inputs_linkml(source_schema)`; same for master
3. Build label/datatype lookups from the text inputs dict (keyed by URI):
   - `src_labels: dict[str, str]` — URI → label
   - `src_datatypes: dict[str, str | None]` — URI → datatype
   - Same for master
4. Load model: `model = EmbeddingModel(model_name)` (lazy — only loaded when suggest runs)
5. Encode: `src_vectors = model.encode(src_text_list)` → numpy array
6. Same for master vectors
7. Add imports:
   - `from rosetta.core.embedding import extract_text_inputs_linkml, EmbeddingModel`
   - `from linkml_runtime.loaders import yaml_loader`
   - `from linkml_runtime.linkml_model import SchemaDefinition`
   - `from typing import cast`

**Done when:** Embedding vectors computed from schemas. No embedding JSON files involved.
