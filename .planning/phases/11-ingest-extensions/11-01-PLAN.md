## Metadata
- phase: 11
- plan: 01

## Must-haves

### Truths
1. `rosetta-ingest schema.xsd --nation NOR` produces valid Turtle with `rose:Field` triples
   for every leaf element and attribute in the XSD; container elements (complexType) emit
   `rose:hasChild` arcs to their child fields.
2. XSD types map correctly: `xs:decimal`/`xs:float`/`xs:double` → `"number"`;
   `xs:integer`/`xs:int` → `"integer"`; `xs:boolean` → `"boolean"`; unknown → `"string"`.
3. Named `xs:complexType` references are fully resolved within the document;
   `xs:extension` merges base-type children with extension children.
4. All `xs:choice` branches are emitted as optional fields (`required=False`).
5. `xs:include`/`xs:import` raises `ValueError` with a clear message.
6. `xs:enumeration` values from `xs:restriction` are collected into `sample_values`.
7. `--input-format xsd` explicit flag works; `.xsd` extension is auto-detected.
8. Exit code 0 on success; exit code 1 with stderr message on parse failure.
9. `uv run pytest -m "not slow"` passes (all existing + new tests green).
10. [review] `xsd_parser.py` uses `defusedxml.ElementTree` instead of stdlib `xml.etree.ElementTree`; `defusedxml` added to `pyproject.toml` (XXE protection for externally-sourced NATO XSD files).
11. [review] `_collect` signature: `_collect(el: ET.Element, registry: dict[str, ET.Element], *, in_choice: bool, visited: set[str] | None = None) -> list[FieldSchema]`; skips (with `stderr` warning) any named type already in `visited` to prevent `RecursionError` on circular `xs:complexType` references.
12. [review] `parse_xsd` raises `ValueError("No xs:element declarations found in XSD")` when `_collect` returns an empty list — prevents silent empty-graph output on stub or empty XSD files.
13. [review] `dispatch_parser` stdin error and unknown-format error messages updated to include `xsd` and `json-sample` alongside the existing three formats.
14. [review] `_collect`, `_map_xsd_type`, `_get_doc` all have explicit parameter and return type annotations (basedpyright strict).

## Tasks

### Task 1 — Shared model: `FieldSchema.children` + `ROSE_HAS_CHILD` + recursive `fields_to_graph`

This task extends shared infrastructure used by both 11-01 (XSD) and 11-02 (JSON sample).
All existing parsers are unaffected — `children` defaults to `[]`.

**File:** `rosetta/core/parsers/_types.py`

Add one field to `FieldSchema` (must come after all non-default fields to preserve dataclass ordering):
```python
children: list["FieldSchema"] = field(default_factory=list)
```

Full updated dataclass:
```python
@dataclass
class FieldSchema:
    name: str
    data_type: str
    description: str = ""
    required: bool = False
    detected_unit: str | None = None
    sample_values: list[str | float | int | bool | None] = field(default_factory=list)
    numeric_stats: dict[str, Any] | None = None
    categorical_stats: dict[str, Any] | None = None
    children: list["FieldSchema"] = field(default_factory=list)
```

**File:** `rosetta/core/ingest_rdf.py`

1. Add constant after the existing `ROSE_DISTINCT_COUNT` line:
   ```python
   ROSE_HAS_CHILD = URIRef(_ROSE_BASE + "hasChild")
   ```

2. Add private helper `_emit_field` before `fields_to_graph`:
   ```python
   def _emit_field(
       g: Graph,
       f: FieldSchema,
       F: Namespace,
       uri_key: str,
   ) -> None:
       """Emit *f* and all its descendants into *g* recursively."""
       field_uri = F[uri_key]
       g.add((field_uri, RDF.type, ROSE_FIELD))
       g.add((field_uri, RDFS.label, Literal(f.name)))
       g.add((field_uri, ROSE_DATA_TYPE, Literal(f.data_type)))

       if f.detected_unit is not None:
           g.add((field_uri, ROSE_DETECTED_UNIT, Literal(f.detected_unit)))

       if f.numeric_stats is not None:
           ns = f.numeric_stats
           stats_bn = BNode()
           g.add((stats_bn, ROSE_COUNT, Literal(ns["count"], datatype=XSD.integer)))
           g.add((stats_bn, ROSE_MIN, Literal(ns["min"], datatype=XSD.double)))
           g.add((stats_bn, ROSE_MAX, Literal(ns["max"], datatype=XSD.double)))
           g.add((stats_bn, ROSE_MEAN, Literal(ns["mean"], datatype=XSD.double)))
           g.add((stats_bn, ROSE_STDDEV, Literal(ns["stddev"], datatype=XSD.double)))
           g.add((stats_bn, ROSE_NULL_RATE, Literal(ns["null_rate"], datatype=XSD.double)))
           g.add((stats_bn, ROSE_CARDINALITY, Literal(ns["cardinality"], datatype=XSD.integer)))
           g.add((stats_bn, ROSE_HISTOGRAM, Literal(ns["histogram"])))
           g.add((stats_bn, ROSE_HISTOGRAM_EDGES, Literal(ns["histogram_edges"])))
           g.add((field_uri, ROSE_STATS, stats_bn))
       elif f.categorical_stats is not None:
           cs = f.categorical_stats
           stats_bn = BNode()
           g.add((stats_bn, ROSE_COUNT, Literal(cs["count"], datatype=XSD.integer)))
           g.add((stats_bn, ROSE_DISTINCT_COUNT, Literal(cs["distinct_count"], datatype=XSD.integer)))
           g.add((stats_bn, ROSE_NULL_RATE, Literal(cs["null_rate"], datatype=XSD.double)))
           g.add((field_uri, ROSE_STATS, stats_bn))

       for child in f.children:
           child_key = f"{uri_key}__{child.name}"
           child_uri = F[child_key]
           g.add((field_uri, ROSE_HAS_CHILD, child_uri))
           _emit_field(g, child, F, child_key)
   ```

3. Replace the loop body in `fields_to_graph` with a single call:
   ```python
   for f in fields:
       _emit_field(g, f, F, f.name)
   ```
   Remove all inline triple-building from `fields_to_graph` — it now delegates entirely
   to `_emit_field`.

**Done when:** `uv run pytest -m "not slow"` green (all existing tests still pass — `children=[]`
is backward-compatible); `uv run basedpyright` clean on `ingest_rdf.py` and `_types.py`.

### Task 3 — Implement `xsd_parser.py`

**File:** `rosetta/core/parsers/xsd_parser.py`

```python
XS = "http://www.w3.org/2001/XMLSchema"
XSD_TYPE_MAP: dict[str, str] = { ... }  # full mapping from CONTEXT.md

def parse_xsd(src: TextIO, path: Path | None, nation: str) -> tuple[list[FieldSchema], str]:
    """Parse an XSD document and return (fields, slug)."""
```

**Implementation steps:**

1. `ET.parse(src)` → root. Raise `ValueError` on `xs:include`/`xs:import` children.

2. **Slug:** `targetNamespace` last segment → `schema_slug()`. Fallback: `path.stem`. Fallback: `"schema"`.

3. **Type registry:** `{name: ET.Element}` from all top-level `{XS}complexType[@name]`.

4. **`_map_xsd_type(xsd_type: str) -> str`** — strip prefix, look up `XSD_TYPE_MAP`, default `"string"`.

5. **`_get_doc(el: ET.Element) -> str`** — text from `xs:annotation/xs:documentation` or `""`.

6. **`_collect(el: ET.Element, registry, *, in_choice: bool) -> list[FieldSchema]`** — returns
   top-level fields (containers have `children` populated, leaves do not):

   - `{XS}element`:
     - Has inline `{XS}complexType` → emit container: `data_type="object"`,
       `children = _collect(complexType, registry, in_choice=False)`,
       `required = minOccurs != "0" and not in_choice`
     - Has `type=` referencing global type → look up in registry,
       emit container with children from that type
     - Has `type=` referencing XSD built-in → emit leaf `FieldSchema`
     - No `type=`, no inline complexType → emit `"string"` leaf
     - Apply `detect_unit(name, description)` on all emitted nodes

   - `{XS}complexType`:
     - `{XS}sequence` / `{XS}all` → `_collect` over children, `in_choice=False`
     - `{XS}choice` → `_collect` over each branch child, `in_choice=True`
     - `{XS}complexContent/{XS}extension` → base type fields first, then
       extension sequence fields; merge into single children list
     - `{XS}attribute` → emit leaf field,
       `required = attr.get("use") == "required"`

   - `{XS}sequence` / `{XS}all` → recurse over element children
   - `{XS}choice` → recurse with `in_choice=True`

   - `xs:enumeration` values: if element has `{XS}simpleType/{XS}restriction/{XS}enumeration`,
     collect all `value` attrs into `sample_values` (strings).

7. **Main:** find top-level `{XS}element` → `_collect` each. Fallback to all global named
   `{XS}complexType` nodes if none found. Deduplicate by name (last-wins).

**URI naming for Turtle output (via `fields_to_graph`):**
```turtle
f:posisjon a rose:Field ;
    rdfs:label "posisjon" ;
    rose:dataType "object" ;
    rose:hasChild f:posisjon__breddegrad, f:posisjon__lengdegrad .

f:posisjon__breddegrad a rose:Field ;
    rdfs:label "breddegrad" ;
    rose:dataType "number" .
```
Child URIs are generated by `_emit_field` using `__` separator — no changes needed in
`xsd_parser.py` itself.
